# CLAUDE.md — 지옥의 길 (Hell Road) 프로젝트 규칙

## 프로젝트 개요

- **게임**: 지옥의 길 — 핵앤슬래시 로그라이크 액션 RPG
- **기반 파일**: `game.html` (단일 HTML, 메인 게임 코드)
- **레거시 파일**: `hell_souls3 (2).html` (구버전, 더 이상 사용하지 않음)
- **기술**: Canvas 2D + Web Audio API + Supabase 인증
- **로컬 서버**: `http://localhost:3333/` (game.html은 index.html 로비에서 진입)
- **기획서**: `지옥의길_통합GDD.md` ← 모든 작업의 기준 문서

## ⚠️ 절대 규칙

### 건드리지 마라 (기존 시스템 보존)

아래 시스템은 이미 완성됐다. 수정/삭제/리팩토링 금지:

- **돌진 시스템** (관통, 체인킬, 피니셔, AoE, 잔상, 스피드라인) — `P.s==='charging'` 관련 전체
- **패링 시스템** (F홀드, 투사체 반사, 근접 카운터, 3초 스턴) — `P.s==='parrying'` 관련 전체
- **마법 시스템** (E탭 투사체, E홀드 빔) — `magicCast`, `beaming` 관련 전체
- **포이즈/그로기** — `e.poise`, `e.stunned`, 그로기 3배 데미지
- **무기 7종** (WTYPES): sword/dagger/hammer/mace/club/axe/spear
- **활 5종** (BOWTYPES): shortbow/longbow/crossbow/mystic/composite
- **속성 5종** (EL): 물리/화염/빙결/암흑/뇌전 + 상성 배율
- **SFX 엔진 v3** — `SFX` 객체 전체 (Web Audio API)
- **저체력 경고** — 심박, 크랙, 비네팅
- **퀵슬롯 물약** — `QSLOTS`, `useQuickslot()`, `POT`
- **키 바인딩** — `BINDS`, 리바인딩 시스템
- **인벤토리** — `INV`, 필터, 비교, 가방 확장
- **데미지 계산** — `hurtE()`, `hurtP()` 의 공식 (DEF 감산, 50% 캡)

### 반드시 지켜라

- **단일 HTML 유지**: 모든 코드는 하나의 HTML 파일 안에. 외부 JS/CSS 분리 금지.
- **외부 라이브러리 금지**: npm, CDN, 프레임워크 사용 금지. 순수 JS + Canvas + Web Audio만.
- **한번에 하나만**: 한 작업에서 하나의 시스템만 추가/변경. 여러 시스템 동시 변경 금지.
- **기존 변수명 유지**: `P`, `G`, `INV`, `ens[]`, `projs[]`, `pProjs[]`, `worldItems[]`, `OPT`, `BINDS`, `STC` 등 기존 변수명/구조체 이름을 바꾸지 마라.
- **기존 함수 시그니처 유지**: `hurtE(e,dmg,ang)`, `hurtP(dmg)`, `addExp(v)`, `addTxt(x,y,t,c,l)`, `addParts(x,y,col,n)` 등의 인자를 바꾸지 마라.
- **테스트 후 커밋**: 변경 후 게임이 정상 실행되는지 브라우저에서 확인.

## Phase별 작업 순서

### Phase 1 — 싱글 코어 (미로 + 무한리스폰)

순서대로 하나씩 진행:

```
1-1. genMaze() 구현
     - 기존 genMap() 교체
     - Recursive Backtracking 미로 알고리즘
     - 층별 크기: 120+fi*20 × 90+fi*15
     - 광장(방) 랜덤 배치 (파밍존, 대장간, 보스방)
     - 위층 통로 배치
     - initStage() → initFloor()로 이름 변경, genMaze() 호출

1-2. 소환굴 시스템
     - spawnHoles[] 배열 추가
     - 4등급: 균열/구멍/심연/나락의입
     - 타이머 기반 몬스터 리스폰
     - checkRooms() 삭제 (클리어 개념 제거)
     - update() 루프에 spawnHoleUpdate() 추가

1-3. 안개 시스템
     - G.fog[][] 2D 배열 (0=안개, 1=탐색됨)
     - 플레이어 주변 반경만 공개
     - draw()에서 안개 렌더링 (검은 오버레이)
     - 미니맵에 안개 반영

1-4. 레벨 1000 시스템
     - addExp() 수정: maxExp 곡선 1.4 → 1.15
     - mkP() 수정: lv상한 1000
     - SP: 레벨당 3 SP (기존 1)

1-5. 5종 스탯 시스템
     - UPG_DEF 교체: hp/atk/def/mp/st/spd → STR/DEX/INT/VIT/LCK
     - 각 max: 600 (600×5=3000)
     - 대장간 SP 탭 UI 수정

1-6. 시네마틱 인트로
     - 기존 doWin() 시네마틱 스타일 참고
     - 프롤로그 → "너는 왜 지옥에 왔느냐?" 전환
     - 게임 시작 시 자동 재생 (스킵 버튼)

1-7. 캐릭터 생성
     - 이름 입력
     - 복수 사연 4택 (빌드 보너스)
     - 초기 무기 선택
     - 완료 후 initFloor(0) 호출
```

2-4. 장비 내구도 + 수리
