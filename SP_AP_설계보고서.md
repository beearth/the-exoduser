## 2. 설계 원칙

1. **선택의 긴장감**: Lv 1000에서도 모든 것을 만렙 찍을 수 없어야 한다
2. **잉여 0에 수렴**: 공급 ≒ 수요의 95~105%
3. **빌드 다양성**: STR빌드 / INT빌드 / LCK빌드 등 차별화
4. **후반부 가치**: 고레벨에서도 포인트 하나가 소중해야 함
5. **점진적 강화감**: 저렙에서 빠른 성장, 고렙에서 느리지만 묵직한 성장

## 3. 제안안

### SP 시스템

| 항목 | 현재 | 제안 |
|------|------|------|
| 레벨당 SP | 3 | **1** |
| Lv 1000 총 SP | 3,000 | **1,000** |
| STAT_MAX | 200 | **150** |
| 전체 스탯 소비 | 1,000 | **750** |

**SP 수요 분석 (1,000 공급):**

| 소비처 | SP | 비고 |
|--------|-----|------|
| 스탯 5종 올캡 | 750 | 150 × 5 |
| 스킬 3종 습득 | 50 | 10+20+20 |
| 스킬 강화 | 200 | 10 SP × 최대 10회 × 2종 |
| **총 수요** | **1,000** | |
| **잉여** | **0** | 이론적 0, 실제 선택 필요 |

**핵심 → 빌드 선택 발생:**
```
┌─ 올라운더: 5스탯 각 150 (750 SP) + 스킬 3종 (50 SP) + 강화 10회 (100 SP) = 900 SP → 나머지로 추가 강화
├─ STR특화: STR 150 + VIT 150 + 나머지 3개 100씩 (600 SP) + 스킬+강화 풀 = 850 SP → 여유
├─ INT특화: INT 150 + DEX 120 + LCK 100 + 나머지 약간 → 스킬 강화 풀투자
└─ LCK도박: LCK 150 + DEX 150 + 나머지 최소 → 크리+드롭 극대화
```

**스탯 캡 150의 의미:**
- 현재 max 200 대비 25% 감소
- 스탯 효과는 그대로 유지 (VIT 150 = HP +150, DEF +30)
- max 200이면 SP가 모자라서 5스탯 올캡 불가 → 강제 선택

### AP 시스템

**현재 문제:** AP 200 vs 수요 86 → 114 잉여

**제안: 점진적 비용 + 패시브 확장**

| 항목 | 현재 | 제안 |
|------|------|------|
| AP 획득 | 5레벨마다 1 | **10레벨마다 1** |
| Lv 1000 총 AP | 200 | **100** |
| 패시브 비용 | 일괄 1 | **점진적 (레벨=비용)** |
| 패시브 최대 레벨 | 8~10 | **5** (비용 증가로 보상) |

**점진적 비용:**
```
패시브 Lv 1 → 1 AP
패시브 Lv 2 → 2 AP
패시브 Lv 3 → 3 AP
패시브 Lv 4 → 4 AP
패시브 Lv 5 → 5 AP (MAX)
───────────────────
1개 만렙 비용: 1+2+3+4+5 = 15 AP
```

**패시브 레벨당 효과 강화 (max 5로 줄이되 효과 2배):**

| 패시브 | max | 올캡 AP | 레벨당 효과 (현재→제안) |
|--------|-----|---------|-------------------------|
| ⚔️ 맹공 | 5 | 15 | 뎀+8% → **뎀+16%** |
| 🛡️ 철벽 | 5 | 15 | 방+3 → **방+6** |
| 💨 돌파 | 5 | 15 | 거리+8%,쿨-5% → **거리+15%,쿨-10%** |
| ⚡ 반격 | 5 | 15 | 판정+8%,뎀+10% → **판정+15%,뎀+20%** |
| 🔮 마력폭주 | 5 | 15 | 뎀+10%,비용-3% → **뎀+20%,비용-6%** |
| 🎯 정밀사격 | 5 | 15 | 뎀+8%,거리+5% → **뎀+16%,거리+10%** |
| 🔰 수호 | 5 | 15 | 뎀+10%,블록+2% → **뎀+20%,블록+4%** |
| 💚 회복력 | 5 | 15 | 리젠+8% → **리젠+16%** |
| 💥 급소 | 5 | 15 | 크리+1.5%,크뎀+5% → **크리+3%,크뎀+10%** |
| 🎁 약탈 | 5 | 15 | 드롭+5% → **드롭+10%** |
| **합계** | | **150** | |

**AP 수급 분석:**
```
공급: 100 AP (10레벨마다 1)
수요: 150 AP (10패시브 올캡)

100 AP으로 가능한 것:
├─ 6~7개 패시브 만렙 (90~105 AP)
├─ 3~4개 만렙 + 나머지 일부 (빌드 특화)
└─ 전부 3레벨씩 (균등) = 60 AP → 나머지로 핵심 만렙

→ 100 < 150: 모든 패시브 만렙 불가 → 빌드 선택 강제 ✓
```

## 5. 레벨 구간별 성장 체감

```
Lv 1~50     : SP 50, AP 5   → 스탯 50 + 패시브 2~3개 (체감 ◎)
Lv 51~100   : SP 50, AP 5   → 스탯 100 + 기폭팔 습득 (강해지기 시작)
Lv 100~200  : SP 100, AP 10 → 스탯 200 + 패시브 본격 투자
Lv 200~400  : SP 200, AP 20 → 빌드 방향 확정, 스킬 강화
Lv 400~700  : SP 300, AP 30 → 주력 스탯 올캡 도달, 서브 스탯 투자
Lv 700~1000 : SP 300, AP 30 → 미세 최적화, 최후의 선택
```

## 6. 코드 변경 사항

### 변경 1: SP 획득 (addExp)
```javascript
// 현재: P.sp+=3;
// 변경: P.sp+=1;
```

### 변경 2: AP 획득 (addExp)
```javascript
// 현재: if(P.lv%5===0){P.ap=(P.ap||0)+1;apGain++;}
// 변경: if(P.lv%10===0){P.ap=(P.ap||0)+1;apGain++;}
```

### 변경 3: STAT_MAX
```javascript
// 현재: const STAT_MAX=200;
// 변경: const STAT_MAX=150;
```

### 변경 4: 패시브 점진적 비용
```javascript
// 현재: row.onclick=()=>{P.ap--;PASSIVES[pd.key]++;...}
// 변경:
const nextLv=PASSIVES[pd.key]+1;
const cost=nextLv; // 레벨=비용 (1,2,3,4,5)
row.onclick=()=>{P.ap-=cost;PASSIVES[pd.key]++;...}
```

### 변경 5: 패시브 max 5 + 효과 2배
```javascript
// 모든 패시브 max: 10/8 → 5

function pAtkMul(){return 1+PASSIVES.pAtk*.16}    // 8% → 16%

```

## 7. 기존 캐릭터 마이그레이션

기존 세이브 데이터와 충돌 방지:
- 스탯이 150 초과인 경우 → 150으로 클램핑, 초과분 SP로 환급
- 패시브가 5 초과인 경우 → 5로 클램핑, 초과분 AP로 환급
- SP/AP 총량 재계산: `P.sp = max(0, 현재SP + 환급SP - (현재레벨×1 배분량과의 차이))`
