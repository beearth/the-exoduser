# 지옥굴 (Hell Souls) — 총 개발 문서

> **파일**: `game.html` (단일 HTML, ~6000줄)
> **장르**: 탑다운 로그라이크 액션 (다크소울 스타일)
> **기술**: Canvas 2D + Web Audio API + Supabase (저장)
> **최종 업데이트**: 2026-02-27

## 1. 핵심 변수

```
P.mhp = 350 + VIT + STR + (장비 bonusHp 합산)

P.stR = 0.28 + DEX×0.002

**배율 함수:**
```
statStr()    = 1 + STR×0.005     (물리 배율)
statDex()    = 1 + DEX×0.003 + 장갑 공속  (공속 배율)
statInt()    = 1 + INT×0.005     (마법 배율)
statCrit()   = DEX×0.05 + LCK×0.1 + pCrit×3 + 반지 크리율  (크리 확률%)
statCritDmg()= 1.5 + LCK×0.005 + pCrit×0.10 + 반지 크리뎀  (크리 배율)

### 4.5 에너지 쉴드 시스템

**해금 조건**: INT ≥ 100

**쉴드 공식 (applyStats)**:
```
P.mshield = 150 + (INT-100)×2.5 + VIT×0.5
P.shield = min(P.shield, P.mshield)
```

**자동 리젠**:
- 피격 후 5초 쿨다운 (`P.shieldRegenT`)
- 리젠 속도: `max(0.3, INT×0.015)` / 프레임
- 물약으로는 회복 불가

**쉴드 리셋 (풀충전)**:
- initStage, 레벨업, restartStage, goToLobby 시 `P.shield = P.mshield`

**HUD**: HP바 위에 파란색 SH 바 (`#shF`), INT≥100일 때만 표시

### 4.6 데미지 타입 시스템

모든 데미지에 물리(phys)/마법(magic)/격참(grab) 타입이 존재.

| 공격 타입 | → HP | → 에너지 쉴드 | → 패링쉴드 |
|-----------|------|---------------|------------|
| 물리 (칼,활,돌진,접촉) | **100%** | 30% | 30% |
| 마법 (투사체,빔,독,뇌전) | 30% | **100%** | 30% |
| 격참 (그랩) | 100% | 100% | **90% 관통** |

**흡수 순서**:
1. 패링쉴드 (패링 중일 때): 격참 90%, 그 외 30%
2. 에너지 쉴드 (shield > 0): 타입별 배율로 흡수, 파괴 시 초과분 HP 전달
3. HP: 쉴드 없으면 직통 (마법은 30%만)

**마법 태그 적용 대상**: 적 투사체(`projs`), 독 데미지, 뇌전, 보스 레이저/AoE

## 5. 전투 시스템

### 5.1 데미지 계산 (`hurtE()`)

```
1. 방패돌격병 정면방어: 공격 각도 차이 ±60° 이내 → 70% 감소
2. 엘리트 모드:
   - M17 전격반사: 20% 반사
   - M14 보호막: 50% 감소
   - M22 동료불사: 근처 M22 아군 시 33% 감소
3. STR 물리 배율: dmg × statStr()
4. 크리티컬: statCrit()% 확률 → dmg × statCritDmg()
5. 그로기 보너스: 스턴 상태 → dmg × 3
6. 한방 캡: max HP의 50%
```

### 5.2 피격 계산 (`hurtP(dmg, opts)`)

`opts` 2번째 파라미터: `{dtype:'phys'|'magic'|'grab'}` (기본 'phys')

```
1. 빔 중이면 beamStop()
2. 방어력 계산: baseDef + pDefAdd() + 전 장비 def 합 + 강화보너스(enh×0.3)
3. 감산: max(1, dmg - totalDef)
4. 돌진 중: 70% 감소
5. 방패 블록: blockRate% 감소 (최대 95%)
6. 패링쉴드: 격참 90%, 그 외 30% 감소 (에너지 쉴드 무시)
7. 에너지 쉴드 (shield > 0):
   - 쉴드 흡수 배율: magic=1.0, phys=0.3, grab=1.0
   - HP 전달 배율: magic=0.3, phys=1.0
   - 쉴드 파괴 시 overflow에 HP배율 적용 후 HP 전달
8. 쉴드 없으면: magic은 HP에 30%만
9. 한방 캡: max HP의 50%
10. 무적 프레임: 35f
11. 경직: stagger 12f (돌진/블록 중 제외)
12. 쉴드 리젠 쿨다운 리셋: P.shieldRegenT = 5초
```

### 5.3 패링 (`doParry()`)

- SFX + "PARRY!" 표시
- 기폭팔 충전 중: 흔들림만 (히트스탑 없음)
- 그 외: hitStop 12f, slowMo 25f, shake 14
- 무적 25f, ST +25 회복
- 카운터 데미지: `(baseAtk + wp.atk + wp.enh) × 4 × pParryDmg()`
- 적 스턴: 180f (기폭팔 습득 시 60f + 흡수)

### 5.4 방패 처내기 (Shield Deflect)

**투사체 반사** (스킬 습득 시):
- 속도 반전 ×2, `p.friendly = true`
- 랜덤 5원소 부여
- 데미지 ×1.5 × (1 + (skillLv-1)×0.1)
- doParry() 호출

**밀리 밀쳐내기** (적이 공격상태일 때):
- 넉백 30, 스턴 120f
- 5원소 중 최대 상성 배율 적용
- 적의 공격상태: windup, eCharge, bossCharge

**히트아크 강화**:
- 데미지: `(sh.atk + combo) × 0.5 × 2 × (1 + (skillLv-1)×0.1)`
- 5원소 최대 배율 적용

**렌더링**: 5등분 무지개 호, 5색 파티클, 히트 프레임 5원소 스파크

### 5.5 화살 유도 + 벽반사

**벽반사** (사정거리 전반 50%):
- 비폭발 화살이 벽에 부딪히면 반사
- 최대 2회 반사, 이후 소멸
- 마법 투사체는 기존대로 3회 반사

**유도** (사정거리 후반 50%+):
- 비폭발 화살이 사정거리 50% 초과 시 유도탄으로 전환
- 300px 반경 내 가장 가까운 적을 추적
- 최대 선회율: 0.12 rad/프레임 (급격한 스냅 방지)
- 각도 보간 방식: `atan2` 기반 점진적 선회

**조건**: 마법 투사체(`p.magic`)와 폭발 투사체(`p.explosive`)는 유도 대상 제외

### 5.6 넉백 시스템 (밸런스 조정)

| 공격 수단 | 넉백 | 비고 |
|-----------|------|------|
| 근접 무기 | **0** (없음) | 적이 밀려나지 않음, 제자리 폭딜 |
| 방패 처내기 | **2.5배** | 강력한 넉백 + 그로기 |
| 돌진 | **1.2배** | 약간 강화 |
| 활/마법 | 기존 유지 | |

**포이즈**: 근접 피격 시 포이즈 감소 **1.5배** (그로기 진입 가속)

### 5.7 빔 시스템

**일반 빔**: E 홀드
- 계수: dmg × 0.04 × statInt() × pMagicMul() × elMul()
- 포이즈 감소: 0.2/프레임

**오원소 빔** (omniBeam 스킬):
- 5줄기 극굵 레이저
- 계수: dmg × 0.04 × 1.2 × (1 + (omniLv-1)×0.1) × statInt() × pMagicMul()
- 포이즈 감소: 0.3/프레임
- 사거리 ×2

## 13. 전투력 (calcCP)

```
ATK 계산: baseAtk × 2 × statStr() + 전 장비 atk × 2
DEF 계산: (baseDef + pDefAdd() + 전 장비 def) × 3
EXTRA 계산:
  - lv × 10
  - mhp × 0.5 + mst × 0.3 + mmp × 0.3
  - 전 스탯 합 × 2
  - 전 패시브 합 × 15
  - 장비 부가스탯 (spd, stagger, range, charge 등)

total = ~~(ATK + DEF + EXTRA)
```
