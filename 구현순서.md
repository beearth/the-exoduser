## Phase 1 — 싱글 코어 (미로 + 무한리스폰)

> 목표: "미로에서 길 잃고 싸우면서 강해지는 게 재미있는가?" 검증

### 1-1. genMaze() — 미로 생성 (최우선)

**변경 대상**: `genMap()` (현재 1057행) → 완전 교체

```
작업:
[1] Recursive Backtracking 미로 알고리즘 구현
    - 층별 크기: (120+fi*20) × (90+fi*15) 타일
    - 2D 배열: 0=벽, 1=통로
[2] 광장(방) 랜덤 배치
    - 파밍존: 소환굴 밀집 막다른길 (3~5개/층)
    - 대장간: 1~2개/층 (forge room)
    - 보스방: 위층 통로 앞 (1개 필수)
[3] 위층 통로 배치 (1~3개)
    - 보스가 지키는 통로 1개 필수
    - 비밀 통로 배치 (벽 파괴로 발견, 선택)
[4] initStage() → initFloor(floorIndex)로 이름 변경
    - genMaze(fi) 호출
    - 기존 ens[], projs[], pProjs[], worldItems[] 초기화 유지
[5] 카메라/충돌 시스템이 새 맵 구조에서 작동하는지 검증
    - canMv(), isW() 함수가 새 맵 배열과 호환되게 수정

의존: 없음 (첫 작업)
영향: 맵 전체, 적 스폰, 이동, 카메라
테스트: 미로 생성 → 플레이어 이동 → 벽 충돌 정상 확인
```

### 1-2. 소환굴 시스템 — 무한 리스폰

**변경 대상**: `initStage()` 적 스폰 + `checkRooms()` 삭제

```
작업:
[1] spawnHoles[] 배열 추가 (G 객체 또는 전역)
    - {x, y, grade:0~3, timer, interval, lvOffset, active:true}
    - 4등급: 균열(60초)/구멍(40초)/심연(25초)/나락의입(15초)
[2] genMaze()에서 소환굴 배치
    - 균열: 통로 벽면 랜덤 (10~15개/층)
    - 구멍: 막다른길 끝 (5~8개/층)
    - 심연: 파밍존 중심 (2~4개/층)
    - 나락의입: 보스방 근처 (1개/층)
[3] spawnHoleUpdate() 함수 구현
    - 매 프레임 타이머 감소 → 0이면 적 스폰 + 리셋
    - 몬스터 레벨 = 층 기본 + lvOffset
    - 화면 밖 소환굴은 스폰 스킵 (성능)
[4] checkRooms() 삭제
    - G.stageCleared 개념 제거
    - 스테이지 클리어 버튼 → 위층 통로 도달로 교체
[5] 게임 루프(update)에 spawnHoleUpdate() 삽입
[6] 소환굴 드로잉 (draw()에 추가)
    - 등급별 색상: 회/빨/보/검
    - 맥동 애니메이션 (리스폰 임박 시 빛남)

의존: 1-1 (맵이 있어야 배치 가능)
영향: 적 스폰 방식 전면 변경
테스트: 소환굴에서 적 리스폰 → 죽여도 다시 나옴 확인
```

### 1-3. 안개 시스템 — 탐색

**변경 대상**: `draw()` 렌더링에 안개 레이어 추가

```
작업:
[1] G.fog[][] 2D 배열 추가 (맵 크기와 동일)
    - 0=안개(미탐색), 1=탐색됨
    - initFloor()에서 전체 0으로 초기화
[2] 매 프레임 플레이어 주변 반경 공개
    - 기본 시야: 8타일 반경
    - (나중에 INT 스탯으로 확대)
[3] draw() 마지막에 안개 오버레이 렌더링
    - fog===0인 타일: 검은색 불투명
    - fog===1이지만 시야 밖: 반투명 어둠 (50%)
    - fog===1이고 시야 내: 완전 공개
[4] 미니맵에 안개 반영
    - 탐색한 곳만 미니맵에 표시
[5] 소환굴 근처 시각 힌트 (안개 속에서 붉은 빛)

의존: 1-1 (맵 배열 필요)
영향: draw() 함수
테스트: 시작 시 주변만 보임 → 이동하면 안개 걷힘 확인
```

### 1-5. 5종 스탯 시스템

**변경 대상**: `UPG_DEF`, 대장간 SP 탭

```
작업:
[1] UPG_DEF 교체
    기존 6종 (hp/atk/def/mp/st/spd) 삭제
    → 5종으로 교체:
    - STR(근력): ATK +1/SP, 벽 파괴력
    - DEX(민첩): Speed +0.02/SP, 패링 윈도우 +1f
    - INT(지능): 마법 DMG +1.5%/SP, 시야 +0.5타일
    - VIT(체력): HP +5/SP, 저항 +0.5%
    - LCK(운):   드랍률 +0.3%/SP
[2] UPGRADES 객체 교체: {str:0, dex:0, int:0, vit:0, lck:0}
    - 각 max: 600 (총 3000 SP)
[3] 각 스탯 apply() 구현
    - STR: P.baseAtk = 8 + UPGRADES.str
    - DEX: P.speed = 2.6 + UPGRADES.dex * 0.02
    - INT: P.magicMul = 1 + UPGRADES.int * 0.015 (신규)
    - VIT: P.mhp = 120 + UPGRADES.vit * 5
    - LCK: P.dropMul = 1 + UPGRADES.lck * 0.003 (신규)
[4] 대장간 SP 탭 UI 수정 (5줄 슬라이더 or +/- 버튼)
[5] 기존 recalcSt() 등 장비 보너스 시스템과 통합

의존: 1-4 (레벨/SP 시스템 먼저)
영향: UPG_DEF, UPGRADES, 대장간 UI, mkP(), 전투 계산
테스트: SP 투자 → 각 스탯 효과 반영 확인
```

```
작업:
```

```
    - +1~+5:  성공률 80%, 재료 소량

[4] 실패 시: 재료만 소멸 (장비 유지)
```

## Phase 4 — 폴리싱

> 목표: 완성도. 분위기. 밸런스.

### 4-1. 보스 대사 시스템

```
- 보스 전투 시작/50% HP/처치 시 대사 출력
- GDD 2장 보스별 대사 구현
- 복수 사연별 대사 분기
```

### 4-2. 층별 고유 기믹

```
- 1층 얼음: 미끄러운 바닥, 얼음벽 파괴
- 2층 곤충: 터널 이동 (벽이 천천히 바뀜)
- 3층 악마: 가짜 출구, 무한루프 복도
- 4층 괴물: 바닥/벽 피해
- 5층 심해: 수위 변화 → 길 개폐
- 6층 용암: 용암 상승 시간 제한
- 7층 고통: 1~6층 기믹 랜덤 혼합
```

### 4-3. 소리 힌트 SFX

```
- 출구 근처: 바람 소리
- 대장간 근처: 망치 소리
- 보스 근처: 심장 박동
- 소환굴 근처: 으르렁/벌레 소리
- 거리에 따른 볼륨 감쇠
```

### 4-4. 보스 출현 방식 다양화

```
- 방황하는 보스: 미로 안 돌아다님
- 둥지 보스: 나락의입 근처 고정
- 조건 보스: 킬 수 달성 시 등장
- 탈출구 수호자: 통로 앞 고정
```

### 4-5. 밸런스 조정

```
- 층별 난이도 곡선
- 스탯별 효율 균형
- 강화 성공률/비용 조정
- 소환굴 리스폰 주기 조정
- 오브 드랍률 조정
```
