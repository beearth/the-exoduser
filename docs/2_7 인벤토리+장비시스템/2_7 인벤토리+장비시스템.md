# 2-7. 인벤토리 + 장비 시스템

## 📌 아이템 bonusHp / bonusShield 5티어 (2026-04-11)
- **공식**: `bonusHp = (tier+1)*100`, `bonusShield = (tier+1)*100`
- **티어 테이블**: T0=100, T1=200, T2=300, T3=400, **T4=500** (최대)
- **적용 슬롯**: 방어구 6종 + 장신구 6종 = **총 12슬롯** (무기 파트 helmet/weapon/bow **제외**)
- **최대 총합**: 12슬롯 × T4 = **HP 6000 + 쉴드 6000** (아이템 베이스 기준)
- **강화 HP**: `_enhHp += _dit.enh * 0.5` → **1강당 +0.5 HP** (1000강 최대 = 아이템당 +500 HP)
- **강화 최대 기여**: 12슬롯 × 1000강 × 0.5 = **+6000 HP** (이론상 풀장착 풀강화)
- **이전 공식 (~2026-04-10)**: `(tier+1)*40` → T4=200 이던 것을 `(tier+1)*100`으로 상향, 강화는 1→0.5로 하향
- **코드 위치**: `game.html` mkItem base 테이블 (라인 ~6723), applyStats _allDefSlots (라인 ~14233)

## 장착창 3파트 레이아웃 (2026-04-10~)
장착창 Paper Doll은 `EQ_POS[]` (game.html)로 5행×3열 그리드에 배치. 3파트 분류:

| Row | top | Col L (16) | Col M (120) | Col R (224) | 파트 |
|-----|-----|------------|-------------|-------------|------|
| 0 | 0 | weapon | helmet | bow | **무기 3종** |
| 1 | 44 | bracelet | necklace | headband | 장신구 상단 |
| 2 | 88 | ring1 | ring2 | belt | 장신구 하단 |
| 3 | 132 | gloves | armor | shield | 방어구 상단 |
| 4 | 176 | pants | boots | cape | 방어구 하단 |

- 최상단 3종 = 무기 (weapon/bow/helmet) → 공격결정 전용
- 중단 6종 = 장신구 (bracelet/ring1/ring2/necklace/headband/belt) → 자원결정 전용
- 하단 6종 = 방어구 (shield/armor/gloves/pants/boots/cape) → 방어결정 전용
- 결정 슬롯 시스템 상세: `docs/2_9 결정슬롯시스템/`

### 정렬 (`↕ 정렬` 버튼 / `sortInvBag()` / `_invAutoPlace()`)
정렬 기준: **calcCP() 가상장착 베니핏** — UI의 "CP +/-" 표시와 완전 동일
1. **즐겨찾기(fav)** → 최상단
2. **쓰레기(junk)** → 최하단
3. **업그레이드(CP+) 먼저** → 이득 큰 순
4. **다운그레이드(CP-) 뒤로** → 손해 적은 순

### 종합지수 (`itemPower`) 산출 공식
**우선순위: 등급 → ATK → 강화 → 방어/HP → 부속 스탯 → 어픽스**

| 카테고리 | 스탯 | 가중치 |
|---|---|---|
| **등급** | sort에서 계층 분리 (itemPower 미포함) | 전설>영웅>희귀>고급>일반 |
| **강화** | enh×80 | +10=800 |
| **아이템Lv** | itemLv×3 | — |
| **ATK (최대)** | 무기: atk×spd×6, 활: atk×bowSpd×6, 기타: atk×6 | — |
| 방어/HP | def×5, eDef×3, bonusHp×2, bonusSt×1.5, spd×30 | — |
| 전투 | wStagger×2, wRange×3, bonusRange×8, bowRange×2, bowPierce×150 | — |
| 전투2 | stagger×2, blockRate×3, charge×1, chargeDist×1, chargeW×0.5, chargeAoE×2, bonusChargeDist×8, chargeCharge×60 | — |
| 크리/속도 | critRate×20, critDmg×120, atkSpd×200, potCd×80 | — |
| 마법/빔 | beamDmg×15, splashR×3, splashMul×60, magicPen×25, 원소dmg×0.6 | — |
| 리젠 | stRegen×150, mpRegen×400 | — |
| 스탯보너스 | bStr×3, bDex×3, bInt×3, **bLck×3, bGrit×3** (장신구 전용) | 장신구(반지/목걸이/팔찌)는 LCK/근성만, 나머지는 STR/DEX/INT |
| 임플리싯 | >1이면 val×2, ≤1이면 val×80 | — |
| **어픽스** | pct/prob: val×150+tier×15, val/dmg/dps: val×1.5+tier×15, cnt등: val×3+tier×15 | 유닛별 분리 |
| 전설특수 | +120 고정 | — |

### 분해 보상 공식 (`salvageVal`)
```
기본값 = [1000, 2000, 4000, 8000, 16000][rarity]   // 등급당 2배
강화값 = enh < 100 ? enh × 3 : 300 + (enh-100) × 1  (최대 1000)
티어값 = tier × 2
총합   = 기본값 + 강화값 + 티어값
```
| 등급 | 기본 악의 |
|------|-----------|
| 일반(0) | 1,000 |
| 고급(1) | 2,000 |
| 희귀(2) | 4,000 |
| 영웅(3) | 8,000 |
| 전설(4) | 16,000 |

## 아이템 레벨 체계
- **itemLv** = `floor(캐릭터레벨 / 10) × 10` (캡 900)
- **reqLv** = `max(0, itemLv - 10)` (착용 요구 레벨)
- 10레벨 단위로 아이템 등급 상승 (0, 10, 20, ... 900)
- 캐릭터 레벨 기반 — 맵/장 무관

## 장비 슬롯 (SLOT_NAMES)
weapon, shield, boots, armor, helmet, bow, gloves, pants, belt, necklace, ring1, ring2, cape, bracelet, headband

### 신규 슬롯
- **bracelet (팔찌)**: 서브타입 2종 (드롭 시 50:50 랜덤)
  - **악마의 팔찌** (`brType:'demon'`): 임플리싯 부활력+5~12%, 전용 어픽스 revivePow/reviveCd
  - **생명의 팔찌** (`brType:'life'`): 임플리싯 최대HP+5~12%, 전용 어픽스 lifeMaxHP/lifeRegen/lifeHealPow
  - 공통 어픽스: antiRevive (부활억제)
- **headband (귀걸이)**: 마법데미지(magicDmg), MP소비감소(mpCostRed) 특화 (maxMPPct 어픽스 제거됨 → ST와 대칭 유지)

## 방어구 기본 HP/쉴드
- 모든 방어구(weapon, bow 제외) 기본 **bonusHp: (tier+1)×40**, **bonusShield: (tier+1)×40** (tier0=40, tier4=200)
- pants, belt은 추가 bonusHp 롤 (base + 티어/레어리티 보너스)
- 어픽스: **maxHPFlat** 최대 +300, **shieldFlat** 최대 +300 (전 방어구 슬롯 장착 가능)

## 강화 효과 (슬롯별)
| 슬롯 분류 | 강화 효과 |
|---|---|
| 무기류 (weapon, bow, helmet) | ATK × (1 + enhMul(enh)) |
| 방어구 (shield~headband 13슬롯) | pDef + enhMul(enh) + **eDef도 동시 상승** (강화 ×0.5) + HP +1/강화레벨 |

- 방어구 슬롯 각각의 enh 합산 → 총 HP 보너스 (예: 갑옷+100, 바지+50 → HP +150)
- 강화 UI 툴팁에 HP 보너스 표시됨

## 장신구 전용 어픽스 (LCK / 근성)
| ID | 슬롯 | 5티어 |
|---|---|---|
| `lckFlatN` | necklace | 20/40/60/80/100 |
| `lckFlatR` | ring/bracelet | 10/20/30/40/50 |
| `gritFlatN` | necklace | 20/40/60/80/100 |
| `gritFlatR` | ring/bracelet | 10/20/30/40/50 |

- `_eqLckTotal()` = STATS.lck + bLck + lckFlatN + lckFlatR
- `_gritTotal()` = _grit + bGrit + gritFlatN + gritFlatR
- statCrit / statCritDmg / statDropBonus 모두 `_eqLckTotal()` 사용
- 근성은 HP/MP/ST 모두에 +1/pt 가산

## 장착 교체 시 자동 전승 (equipItem)
1. **강화 전승**: 기존 아이템 enh → 새 아이템으로 이전 (악의 비용 차감, 부족 시 교체 차단)
2. **결정 전승**: 기존 아이템 결정 → 새 아이템 빈 소켓에 자동 장착
   - 새 아이템 소켓 부족 시 → 결정 주머니(CRYSTAL_BAG)로 자동 탈착
   - 주머니 제한: 9999 (사실상 무제한)
