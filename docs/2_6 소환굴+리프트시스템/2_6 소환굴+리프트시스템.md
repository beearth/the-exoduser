# 2-6. 소환굴 + 리프트 시스템

## 소환굴 (SPAWN_HOLE)
- 4등급: 균열 / 구멍 / 심연 / 나락의입
- 대/소 2종 (50/100 스폰량)
- 방·복도에 자동 배치, 주기적 리스폰
- genFromTemplate에서 배치 (genGauntlet은 폴백)
- 모든 방 최소 1개, 보스 인접 2개, 큰 방 +1
- **스테이지당 최소 20개 보장** — 소환굴 수가 20 미만이면 전투방에 자동 추가 (tileRLE/프로시저럴 모두 적용)
- **트리거 거리 2000px** — 플레이어가 2000px 이내에 접근하면 트리거
- **5초 순차 소환**: 트리거 후 1.5초 연출 → 5초(300f)에 걸쳐 1마리씩 순차 스폰. 간격 = `300 ÷ 총량`프레임
- **바닥 탐색**: 랜덤 30회 → 나선 20타일. 최종 실패 시 소환굴 중심에 강제 스폰 (100% 소환 보장)
- **거리 제한 제거**: 트리거 후에는 플레이어 거리 무관하게 완료까지 스폰 지속
- **스폰 페이드인**: 등장 즉시 `reviveIframes=15` (15프레임 반투명, 렌더 경량)
- **렌더 캡**: 화면 내 적 `_dynERMax`(기본 60) 초과 시 보스 제외 렌더 스킵

## 리프트 시스템
- G.rifts — 적 처치 시 균열 스폰
- 균열 접근 시 보상/이벤트

## 맵 파이프라인 주의
1. FIXED_MAPS 빌드: genSnake/genHoriz/generateGridMap → 템플릿
2. genFromTemplate(tmpl, si) → 실제 맵 (소환굴 배치 여기서). 배치 후 유효성 검증: 벽 위→나선 탐색으로 바닥 이동, 주변 5타일 바닥 30칸 미만→제거 (스폰 공간 부족)
3. genGauntlet은 FIXED_MAPS 없을 때만
4. 맵 변경은 새 게임에서만 적용
