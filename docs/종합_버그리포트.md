# 종합 버그 리포트

발견된 버그와 수정 내역을 누적 기록한다.

## 2026-04-07

### [치명] 몬스터 사라짐 — AI 들러붙기 + MAP_OBJS 충돌 복합 버그
- **증상**: 근접 몬스터가 플레이어 접촉 시 즉사/사라짐. 특히 작은 몬스터, 스킨 적용 몹에서 빈발
- **원인 (복합)**:
  1. AI 들러붙기(`d<400&&d>30`) + 추적해제 거리 80→30 변경으로 몬스터가 플레이어에 과밀착
  2. `isW()`가 MAP_OBJS collision(나무/바위)도 벽으로 판정 → `_pushOutWall`이 적을 먼 곳으로 텔레포트
  3. 벽 끼임 즉사(`isW(e.x,e.y,true)`) 체크가 MAP_OBJS까지 포함 → 오브젝트 근처 몹 즉사
  4. 벽 충돌 데미지가 MAP_OBJS에 반복 적용 → 서서히 사망
  5. 석궁 자동발사가 미장착 상태에서도 작동 (`_xbowEquipped=true` 기본값, `INV.equipped.bow` 미체크)
- **수정**:
  - AI 복구: 어그로 1000→600, 추적해제 30→80, 들러붙기 삭제 (밀어내기도 제거)
  - `_pushOutWall`: `_canMvTile` 도입 (타일벽만 체크, MAP_OBJS 무시)
  - 벽 끼임: 타일벽만 체크 + 즉사→플레이어 방향 80px 밀어내기
  - 벽 충돌 데미지: 타일벽만 적용
  - 석궁: `INV.equipped.bow` 체크 추가
- **교훈**: AI 개선 시 들러붙기 거리/밀착 거리 변경은 반드시 벽 끼임 테스트 병행 필요. MAP_OBJS collision이 `isW()`에 포함되어 있으므로, 적 위치 판정에서 `isW` 대신 타일 전용 체크 사용 권장

### [중] 안개 네모 아티팩트
- **증상**: GPU/WebGL 모드에서 안개가 사각형으로 렌더링
- **원인**: `_useGPU||_useGL` 분기에서 `fillRect`(사각형) 사용
- **수정**: 안개 시스템 제거 → 프리렌더 스탬프 기반 스모그 시스템으로 교체

### [경] 칼등날개 ST 이중소모
- **증상**: 우클릭(sBash) + 칼등날개가 ST를 이중 차감
- **원인**: `useStPct('shield')` 후 `_autoShieldThrow`에서 추가 50/30 ST 차감
- **수정**: `_autoShieldThrow` ST 추가소모 제거 (우클릭에서 이미 차감)

### [경] 레거시 아틀라스 404
- **증상**: `atlas_ch1_south.png` 등 6개 파일 404
- **원인**: 8dir 아틀라스로 전환됐지만 `_loadChAtlas` 레거시 호출 잔존
- **수정**: `_loadChAtlas` 호출 비활성화, `_load8DirAtlas` 사용

## 2026-04-03

### [치명] 보스 게이트 나무 미표시 + 투명 충돌체 차단
- **증상**: 보스 게이트 나무 이미지 안 보임, 투명 충돌체가 플레이어 이동 차단, 포탈 진입 불가
- **원인**: tileRLE 맵에서 맵 캐시 Phase 2 미실행 → 나무 drawImage 누락. exit/충돌/렌더 좌표 불일치
- **수정**: 나무를 실시간 렌더로 이전, 충돌체를 실시간 렌더 나무 중심 `(bossCx+0.5, gateY+0.5)*T`로 3곳 통일
- **상세**: `docs/4.1맵디자인+설정/보스게이트_버그리포트.md`

### [중] MAP_OBJS 투명 오브젝트 벽 현상
- **증상**: `boss_gate_col` 등 스프라이트 없는 충돌 오브젝트가 보이지 않는 벽으로 작용
- **원인**: MAP_OBJS 정의에 `col:1` (충돌 활성)이지만 렌더 스프라이트가 null → 화면에 안 보이지만 충돌 판정 활성
- **영향**: 플레이어가 빈 공간에서 막히는 현상
- **대응**: 충돌 오브젝트 위치를 실시간 렌더 나무 중심 `(bossCx+0.5, gateY+0.5)*T`와 일치시켜 의도된 봉인으로만 작동하게 함
- **향후**: 스프라이트 없는 col 오브젝트에 디버그 모드 시 빨간 윤곽 표시 권장

### [중] 사운드 시스템 — 300마리 동시타격 시 전체 사운드 멈춤
- **증상**: 대량 적 동시타격 시 BGM 포함 모든 사운드 정지
- **원인**: `playTone`/`playNoise`/`playFM`/`playSub`/`playDist` 5개 프로시저럴 사운드 함수에 AudioNode 수 제한 없음 → 수백 개 노드 생성 → AudioContext 스레드 포화
- **수정**: 
  - 5개 함수 전부 `_activeNodeCnt >= _MAX_ACTIVE_NODES` 체크 추가
  - `onended` + `setTimeout` 이중 정리 (onended 미발화 대비)
  - 사운드 우선순위 시스템 (`_SFX_PRI`) 도입: SKILL(10) > VOICE(9) > BOSS(5) > HIT(1) > AMB(0)
  - 채널 포화 시 최저 우선순위 노드 stop() 후 교체
  - HIT 동시 최대 4개 하드캡
  - 30ms 중복 재생 방지
