# 15. 세이브 + 데이터 구조

## 세이브 경로 (3종)

| 경로 | 조건 | 저장 위치 | 함수 위치 |
|---|---|---|---|
| Supabase | `sb` 객체 존재 시 | Supabase DB `characters` 테이블 | `dbSave()` L1160 |
| 로컬 서버 | `?test=1` 파라미터 | `saves/슬롯명.json` (server.cjs) | `dbSave()` L138123 (오버라이드) |
| localStorage | Supabase/서버 모두 실패 시 | `hellsave_web` 키 | `dbSave()` L138243 (오버라이드) |

## 저장/불러오기 함수

| 함수 | 동작 |
|---|---|
| `dbSave()` | 비동기 저장 (경로별 오버라이드) |
| `dbSaveNow()` | 500ms 디바운스 후 dbSave (일반 저장) |
| `dbSaveForce()` | 즉시 dbSave (스킬 전환, 장비 변경 등 핵심 이벤트) |
| `dbRestore(d)` | 세이브 데이터 `d`를 P/G/INV 등에 복원 |

## 복원 시 강제 덮어쓰기 (dbRestore 후)

로드 후 유효성 검증으로 일부 값이 덮어써짐:
- **스킬 미소유 시 기본값 복귀**: activeMagicSk→fireball, activeTechSk→spikeTrap 등
- **합체 관련 LMB 강제**: whirlDet/slamStorm/stormBeam 합체 시 → whirlwind 강제
- **삭제된 스킬 마이그레이션**: fanShot→fireball, galeSlash→charge 등
- **레벨 자동해금**: Lv300→blueShot, Lv700→burstLoop

### 새 게임 전용 (!_loaded)
- `P.activeCtSk='iceOrb'` (iceOrb 보유 시)
- 기본 스킬 슬롯 배정 (giantSlam, holyDome, spikeTrap)

## 저장 타이밍

| 이벤트 | 저장 방식 |
|---|---|
| 30초 자동저장 | `dbSaveNow()` (디바운스) |
| L키 스킬 전환 | `dbSaveForce()` (즉시) |
| 레벨업 | `dbSaveForce()` |
| 아이템 획득/분해 | `dbSaveNow()` |
| 장비 장착/해제 | `dbSaveForce()` |
| 스테이지 전환 | `dbSaveForce()` |
| 합체 완료 | `dbSaveForce()` |

## 주의사항
- 맵 데이터는 세이브에 미포함 → `initStage()`로 재생성
- 쿨다운(P._faCd, P._blueCd 등)은 세이브 미포함 (에페메랄)
- OPT/BINDS는 `localStorage('hellcave_settings')`에 별도 저장
- 세이브 로드 시 마이그레이션 순서 중요 (skills → fused → activeXSk)
