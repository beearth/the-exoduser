# 15. 세이브 + 데이터 구조

## 세이브 경로 (3종)

| 경로 | 조건 | 저장 위치 | 함수 위치 |
|---|---|---|---|
| Supabase | `sb` 객체 존재 시 | Supabase DB `characters` 테이블 | `dbSave()` L1160 |
| 로컬 서버 | `?test=1` 파라미터 | `saves/슬롯명.json` (server.cjs) | `dbSave()` L138123 (오버라이드) |
| localStorage | Supabase/서버 모두 실패 시 | `hellsave_web` 키 | `dbSave()` L138243 (오버라이드) |

## 저장/불러오기 함수

| 함수 | 동작 |
|---|---|
| `dbSave()` | 비동기 저장 (경로별 오버라이드) |
| `dbSaveNow()` | 500ms 디바운스 후 dbSave (일반 저장) |
| `dbSaveForce()` | 즉시 dbSave (스킬 전환, 장비 변경 등 핵심 이벤트) |
| `dbRestore(d)` | 세이브 데이터 `d`를 P/G/INV 등에 복원 |

### 계정 공유 저장소 (localStorage + 서버)
| 키/경로 | 내용 |
|---|---|
| `hellcave_sharedStorage` (localStorage) | 지옥창고 아이템 배열 (모든 캐릭터 공유, 최대 200슬롯) |
| `hellcave_sharedMats` (localStorage) | 악의(mats) 수량 — 런타임 캐시 |
| `saves/_sharedMats.json` (서버) | 악의(mats) 수량 — 서버 원본 (source of truth) |
| `GET /api/mats` | 서버에서 악의 값 조회 |
| `POST /api/mats` `{mats:N}` | 서버에 악의 값 저장 |

- 세이브 파일의 `storage`/`mats`는 레거시 마이그레이션 용도로만 1회 이관 후 비움
- **악의 동기화 흐름**:
  - **저장**: `_saveSharedMats(v)` → localStorage 캐시 + `POST /api/mats` 서버 저장
  - **로드**: 부트 시 `_loadSharedMatsFromServer()` → 서버 값으로 localStorage 덮어쓰기 (서버 실패 시 localStorage 폴백)
  - **런타임**: `_loadSharedMats()` → localStorage 캐시에서 동기적 읽기
- 신규 캐릭터 생성 시 기존 공유 풀에서 로드 (초기 1000 대신 공유 값)
- `mats` 이관은 합산이 아닌 최대값 동기화로 처리 (`Math.max(_sharedMats,_savedMats)`)
  - 목적: 동일 세이브 재로딩 시 `mats`가 반복 누적되는 폭증 회귀 방지

#### 🐞 32비트 오버플로우 수정 (2026-04-11)
**증상**: 캐릭터 삭제 후 신규 시작 시 악의가 음수(예: -1555936894)로 표시
**원인**: `_loadSharedMats`/`_saveSharedMats`가 `~~v`(32bit truncation) 사용 → 21억(2^31) 초과 시 음수 wrap
**수정**:
- `_loadSharedMats`: `parseInt(v,10)` + `isFinite` + `Math.min(n,Number.MAX_SAFE_INTEGER)` + 음수 차단
- `_saveSharedMats`: `Math.max(0, Math.min(Math.floor(+v||0), Number.MAX_SAFE_INTEGER))`
- 세이브 로드 시 `Math.max(0, ...)` 클램프 추가 (game.html 라인 ~1420)

**최대 안전값**: `Number.MAX_SAFE_INTEGER = 9007199254740991` (약 900경)
#### 🛠️ shared mats 누적합 회귀 수정 (2026-04-12)
**증상**: 재접속/재로드를 반복할수록 악의가 비정상적으로 증가
**원인**: 세이브 로드 시 공유 악의와 세이브 악의를 덧셈 병합하던 구로직으로 반복 합산
**수정**:
- `_saveSharedMats(Math.max(_sharedMats,_savedMats))`로 변경 (비가산 동기화)
- 마이그레이션 후 `d.game.mats=0` 유지

#### 🛠️ 로컬/웹 세이브 경로 mats 복원 회귀 수정 (2026-04-12)
**증상**: 악의를 소비해도 로드 시 30억으로 복원 (무한 악의)
**원인**: 3개 세이브 경로 불일치
- **메인 dbSave** (IDB): `mats:0` 저장 ✅ + `_saveSharedMats` 호출 ✅
- **로컬 테스트 dbSave** (server saves/): `mats:G.mats` 저장 ❌ + `_saveSharedMats` 미호출 ❌
- **웹 standalone dbSave** (localStorage): `mats:G.mats` 저장 ❌ + `_saveSharedMats` 미호출 ❌
- 로드 시 `Math.max(_sharedMats, _savedMats)` → 세이브 파일의 높은 값이 항상 복원
**수정**:
- 로컬/웹 세이브 경로 모두 `mats:0` 저장 + `_saveSharedMats(G.mats||0)` 호출 추가
- 모든 세이브 경로가 메인 dbSave와 동일한 공유풀 패턴 준수

#### 🛠️ 악의 경제 전수 감사 + 서버 동기화 강화 (2026-04-12)

**1. 서버 동기화 (source of truth)**
- 서버 엔드포인트: `GET/POST /api/mats` → `saves/_sharedMats.json`
- `_saveSharedMats()` → localStorage + `POST /api/mats` 동시 저장
- `_loadSharedMatsFromServer()` → 부트 시 서버 값 우선 로드 (폴백: localStorage)

**2. 루프 기반 디바운스 동기화**
- 게임 루프 매 프레임에서 `G.mats !== _matsPrev` 감지
- 변경 시 `_matsDirty()` 호출 → 3초 디바운스로 `_saveSharedMats` 실행
- 전투 중 대량 mats 변경(적 처치, 패링, 자동물약 등)도 서버에 주기적 반영

**3. 음수 잔액 방어 추가**

|------|--------|-------------|
| `fireBoneWall` | 12 | `if(G.mats<12)return` |
| `activateGiantSlam` | 20 | `if(G.mats<20)return` |
| `_skClick` 습득 | sk.matCost | `if(G.mats<sk.matCost)return` |
| `_skClick` 강화 | sk.upMat | `if(G.mats<_upMat)return` |

- 기존 안전: maliceHunt(28647), plagueBurst(28873), blastShot, spikeTrap, 상점, 결정 등

| `upgrades` | 강화 레벨 {hp, atk, def, mp, st, spd} |

## 복원 시 강제 덮어쓰기 (dbRestore 후)

로드 후 유효성 검증으로 일부 값이 덮어써짐:
- **스킬 미소유 시 기본값 복귀**: activeMagicSk→fireball, activeTechSk→spikeTrap 등
- **합체 관련 LMB 강제**: whirlDet/slamStorm/stormBeam 합체 시 → whirlwind 강제
- **삭제된 스킬 마이그레이션**: fanShot→fireball, galeSlash→charge 등
- **레벨 자동해금**: Lv300→blueShot, Lv700→burstLoop

### 새 게임 전용 (!_loaded)
- `P.activeCtSk='iceOrb'` (iceOrb 보유 시)
- 기본 스킬 슬롯 배정 (giantSlam, holyDome, spikeTrap)

## 주의사항
- 맵 데이터는 세이브에 미포함 → `initStage()`로 재생성
- 쿨다운(P._faCd, P._blueCd 등)은 세이브 미포함 (에페메랄)
- OPT/BINDS는 `localStorage('hellcave_settings')`에 별도 저장
- 세이브 로드 시 마이그레이션 순서 중요 (skills → fused → activeXSk)
