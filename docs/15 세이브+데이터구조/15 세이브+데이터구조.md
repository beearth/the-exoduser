# 15. 세이브 + 데이터 구조

## 세이브 시스템
- `/api/slots` — server.cjs 세이브 API
- 슬롯 기반 저장 (slot 파라미터)
- 로컬 파일: `saves/` 폴더

## 핵심 데이터 객체
| 객체 | 용도 |
|---|---|
| `P` | 플레이어 (위치, 상태, 스탯, 스킬, 장비 효과) |
| `G` | 게임 전역 (맵, 방, 스테이지, 카메라, fireZones) |
| `INV` | 인벤토리 {bag[], equipped{}} |
| `STATS` | 5종 스탯 {str, dex, int, vit, lck} |
| `OPT` | 설정 (fps, 이펙트, 볼륨) |
| `BINDS/BINDS2` | 키 바인딩 |

## 구버전 마이그레이션
- `_fixDur()` — 내구도 필드 없는 구버전 아이템 패치
- `_fixAffixes()` — affixes 배열 없는 구버전 → 배열화
- `_fixedKick` — 고정스킬 슬롯 강제 정리

## 세이브 데이터에 포함되는 것
- P 전체 (스탯, 스킬 레벨, 합체 상태, 숙련도)
- G 일부 (stage, mats, rifts, 맵 데이터)
- INV 전체 (가방 + 장착)
- OPT, BINDS

## 주의
- 맵은 세이브에 저장됨 → 맵 변경은 새 게임에서만 적용
- 세이브 로드 시 마이그레이션 순서 중요
