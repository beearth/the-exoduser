### GC 특성 (V8)

- **Minor GC (Scavenge)**: 0.5~2ms — 매 프레임 30+개 배열 생성 시 빈번 트리거
- **Major GC (Mark-Compact)**: 5~50ms — 프레임 스파이크 원인
- **핵심**: "단명 객체는 GC 비용이 낮다"지만, **프레임당 100+개 생성은 Scavenge 빈도 증가**

### 핵심 결론

- 현재 아키텍처(Canvas2D + WebGL 후처리 + Worker 충돌) **업계 표준과 일치**
- 드로우콜 수(18개 @ 487적) **매우 우수**
- **병목은 GC 압력(배열/Set/클로저 할당)과 충돌 시스템(8.7ms)**
- 2ms 초과분 = GC Scavenge 1~2회 + 불필요한 할당 오버헤드

## 심각도 [상] — 즉시 수정 대상

### 1. setLineDash([]) 매 프레임 다수 호출

- **위치**: draw() 내 총 25곳
  - 40658~40790: 보스 wind (조건부 8곳)
  - **40935~43229: 조준 UI (매 프레임 17곳)** ← 핵심
- **문제**: 호출마다 빈 배열 `[]` 생성 → GC 대상. 적 30마리+보스 공격 시 프레임당 30~50개
- **수정안**: `const _DASH_NONE=[];` 전역 상수 + `setLineDash(_DASH_NONE)` 교체. dash 패턴도 `const _DASH_8_6=[8,6]` 등 캐싱
- **코드 패턴**:
  ```javascript
  // BEFORE
  X.setLineDash([8,6]); ... X.setLineDash([]);
  // AFTER  
  X.setLineDash(_DASH_8_6); ... X.setLineDash(_DASH_NONE);
  ```

### 2. new Set() 빔 시스템에서 매 프레임 다수 생성

- **위치**: 빔 스킬 핫패스
  - 25267: `const _bpHit=new Set()` — 매 쿼리마다
  - 25278: `const _seen=new Set()` — 매 쿼리마다
  - 25348: `const _branchHit=new Set()` — 프레임당 6갈래
  - 투사체 26294~26498: `if(!p._hitSet)p._hitSet=new Set()` — 투사체당 1회
- **문제**: 빔 활성 중 프레임당 6~12개 Set 생성. Set은 내부 해시테이블이라 할당 비용 높음
- **수정안**: 
  - 빔: 전역 `_bpHitReuse`, `_seenReuse`, `_brHitReuse` Set → `.clear()` 재사용
  - 투사체: `_recyclePProj` 시 `_hitSet.clear()` (new 안 함)
- **코드 패턴**:
  ```javascript
  // BEFORE
  const _bpHit=new Set();
  // AFTER
  _bpHitReuse.clear(); // 전역 Set 재사용
  ```

### 3. _wiPush 매 push시 sort

- **위치**: 12207
- **문제**: `worldItems.sort()` + `splice()` — O(n log n) + 배열 재할당
- **수정안**: O(n) 최소값 스캔 → swap-remove
- **코드 패턴**:
  ```javascript
  // BEFORE
  worldItems.sort((a,b)=>{...}); worldItems.splice(0,worldItems.length-20);
  // AFTER
  let minIdx=0, minR=Infinity;
  for(let i=0;i<worldItems.length;i++){
    const r=worldItems[i].type==='item'?(worldItems[i].item?worldItems[i].item.rarity||0:0):-1;
    if(r<minR){minR=r;minIdx=i}
  }
  worldItems[minIdx]=worldItems[worldItems.length-1]; worldItems.length--;
  ```

### 4. 사슬기동 중 shQuery 5회+ 연속

- **위치**: 23488, 23521, 23594, 23643, 23713
- **문제**: 같은 프레임 같은 좌표에서 공간쿼리 5번. 결과 버퍼 교체 오버헤드
- **수정안**: 최대 반경으로 1회 쿼리 후 결과 배열 복사, 서브쿼리는 거리 필터
- **코드 패턴**:
  ```javascript
  // BEFORE
  const ne1=shQuery(P.x,P.y,40);  // 경로 적
  const ne2=shQuery(P.x,P.y,_csR); // 칼날개
  const ne3=shQuery(P.x,P.y,_csR+20); // 착지
  // AFTER
  const _maxR=Math.max(40,_csR+20);
  const _allNe=shQuery(P.x,P.y,_maxR);
  const _allCopy=[]; for(let i=0;i<_allNe.length;i++)_allCopy[i]=_allNe[i];
  // 서브쿼리: _allCopy에서 dst2 필터
  ```
