## 심각도 [상] — 즉시 수정 대상

### 1. setLineDash([]) 매 적마다 호출

- **위치**: draw() 40658~40991 (약 18곳)
- **문제**: `X.setLineDash([])` 호출마다 빈 배열 리터럴 생성. 적 30마리 = 매 프레임 30개 GC 대상
- **수정안**: 전역 `const _DASH_NONE=[];` 선언 후 재사용
- **상태**: [ ] 미수정

### 2. ens 500 초과 시 전체 정렬

- **위치**: 25515
- **문제**: `_sorted.sort((a,b)=>...)` — 500+ 배열 O(n log n) + 정렬 콜백 클로저 생성
- **수정안**: 보스 우선 파티션 후 nth_element 방식 부분 선택 (정렬 불필요)
- **상태**: [ ] 미수정 (현재 캡 500/400으로 하향 완료, 정렬 방식은 미개선)

### 3. _wiPush 매 push시 sort

- **위치**: 12207
- **문제**: worldItems 20개 초과 시 매번 sort + splice. 드롭 빈번한 구간에서 초당 수십 회
- **수정안**: 최소 rarity 아이템 1개만 찾아서 제거 (O(n) 스캔, 정렬 불필요)
- **상태**: [ ] 미수정

### 4. 사슬기동 중 shQuery 5회+ 연속

- **위치**: 23488, 23521, 23594, 23643, 23713
- **문제**: 같은 프레임 같은 좌표에서 공간쿼리 5번 연속 호출. 각각 버퍼 할당
- **수정안**: 1회 쿼리 후 결과 캐싱, 범위별 필터
- **상태**: [ ] 미수정

### 5. 매 프레임 new Set() 생성

- **위치**: 23704, 24708, 26294+, 26953, 27454, 27789
- **문제**: 투사체 생성마다 `_hitSet:new Set()`. 고빈도 스킬 시 초당 수십 개
- **수정안**: Set 풀링 — 투사체 재활용 시 `_hitSet.clear()` 재사용
- **상태**: [ ] 미수정

### 6. Bloom 후처리 비용

- **위치**: 3656~3679, draw() 42780
- **문제**: 3프레임마다 1/6 해상도 다운샘플 + 4회 블러 = 5회 drawImage. 고해상도에서 누적
- **수정안**: Tier B 이하에서 블러 2회로 축소, 또는 주기 5프레임으로 완화
- **상태**: [ ] 미수정

### 7. 투사체 아틀라스 미사용

- **위치**: 16796+, 26839~26861
- **문제**: 투사체 개별 drawImage 호출. 탄막 스킬 시 100+ 드로우콜
- **수정안**: 투사체 아틀라스 생성 후 배칭
- **상태**: [ ] 미수정
