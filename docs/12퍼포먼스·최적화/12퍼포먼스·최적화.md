### _waitObjSprites(timeout)

`initStage()` 후 맵 오브젝트/VFX 이미지가 전부 로드될 때까지 로딩 화면에서 대기.

**체크 대상:**
- `_OBJ_SPR` — 맵 오브젝트 스프라이트 전체 (pit, tree, deco 등)
- `_bossRoomBg` — CH1 보스룸 배경 이미지
- `_bossGateImg` — CH1 보스문 눈나무 이미지
- `_impSpr` — 기본 타격 임팩트 스프라이트 시트

**동작:** 50ms 간격 폴링, 모든 이미지 `complete` 또는 타임아웃 시 resolve.

### 로딩 UI

- `showBootLoading(msg)` — 로딩 오버레이 표시
- `setBootLoading(pct, msg)` — 진행률 + 메시지 갱신
- `hideBootLoading()` — 로딩 오버레이 숨김
- 최소 표시 시간 5초 (인트로 연출 겸용)

### 핵심 규칙

1. **initStage 후 반드시 _waitObjSprites 호출** — 에셋 미로드 상태로 게임 시작 금지
2. **새 이미지 에셋 추가 시 _waitObjSprites 체크 대상에 포함** — `_extraImgs` 배열에 추가
3. **타임아웃은 8초** — 네트워크 지연 대비, 타임아웃 시 로드된 것만으로 시작

## 맵 캐시 스트리밍 모드 OOM 방지 (2026-04-25)

`_shouldStreamMapCache` 판정 기준 확장. 큰 맵에서 전체 캔버스 할당 시 OutOfMemory 방지.

| 조건 | 변경 전 | 변경 후 |
|------|---------|---------|
| 타일 수 | `mw*mh >= 1M` | 동일 (유지) |
| 캔버스 크기 > texLimit | 미체크 (OOM 가능) | `mw*T > texLimit` 또는 `mh*T > texLimit` 시 강제 스트리밍 |
| 총 픽셀 수 | 미체크 | `mw*T * mh*T > 64M px` 시 강제 스트리밍 |

- texLimit: WebGPU=4096, WebGL=`_glMaxTex`, 소프트웨어=99999
- T=40 기준 texLimit=4096 → 102타일(4080px) 이상에서 스트리밍 강제
- 기존 청크 파티션 코어(`_autoChunkConfig`, `_buildStreamChunk`, `_finalizeBMC`)는 미수정

## 아틀라스 텍스처 워밍업 (2026-04-14)

첫 drawImage 시 GPU 텍스처 바인딩 비용이 집중되어 143~614ms 스파이크 발생.
**오프스크린 + setTimeout 백그라운드 방식**: 게임 루프 밖에서 1장씩 비동기 워밍업.

| 구성 | 설명 |
|------|------|
| `_wqCvs` (1×1) | 오프스크린 캔버스 — 메인 캔버스 상태 오염 없음 |
| `_wqBuf[40]` | 고정 배열, 워밍업 대상 이미지 참조 |
| `_warmupEnsAtlas()` | 대상 수집 → `_wqBuf`에 적재 → `_warmupNext()` 시작 |
| `_warmupNext()` | `setTimeout(fn,0)` 체인, 1장씩 `_wqCtx.drawImage` → 게임 루프 **0ms 영향** |

| 호출 시점 | 설명 |
|-----------|------|
| 부트 완료 (`_dbReady=true` 직후) | 큐 적재 + setTimeout 체인 시작 |
| ens 렌더 루프 진입 | 미완료 시 `_warmupEnsAtlas()` 재시도 (지연 로드 커버) |
| `nextStage()` | `_ensWarmDone=false` 리셋 → 다음 렌더 시 재적재 |

워밍업 대상 (~19장): 8dir idle(8) + walk(8) + 보스 3종 + GL 아틀라스 + 구울 3종

## 500마리 난전 자동 프로파일러 (`_PERF_PROF`, 2026-04-17 추가)

게임 메인 루프(`function loop`) 내부에 주입된 측정 전용 코드. **기존 게임 로직은 일절 수정하지 않음** — 호출부만 감싸고 일부 함수는 외부에서 참조 교체로 후킹.

### 활성화 조건
- 활성 몹 수(`ens`에서 `alive===true`) **300 초과** 시 자동 ON
- 300 이하로 떨어지면 자동 OFF (오버헤드 회피)
- ON/OFF 시 콘솔에 `[PERF] ON (enemies=N)` / `[PERF] OFF (enemies=N)` 로그

### 측정 구간 (4종)
| 키 | 측정 대상 | 후킹 방식 |
|---|---|---|
| `ai` | `update()` while 루프 전체 시간 − `coll`. AI/이동/스킬/플로우필드 등 게임 로직 | 루프 내 호출부 감싸기 (`_aiAcc`) |
| `coll` | `_applyColResult()` + `_dispatchColWorker()` (충돌 워커 디스패치/결과적용) | 함수 참조 교체 (외부 후킹) — update 내부 무수정 |
| `render` | `draw()` 호출 (보간 분기 포함) | 루프 내 호출부 감싸기 (`_renderAcc`) |
| `dom` | `_fpsDraw()` + `_glFlush()` + `_drawBurst()` (포스트렌더 UI/GPU flush) | 루프 내 호출부 감싸기 (`_domAcc`) |

### 출력
- **60프레임마다** 콘솔에 한 줄:
  ```
  [PERF] enemies=487 drawCalls=18 ai(avg/max)=4.2/6.8ms coll=8.7/12.1ms render=3.1/4.5ms dom=2.3/3.0ms heap=245MB
  ```
- 각 구간 `avg/max` 60프레임 윈도우 (롤링 큐)
- `drawCalls` = WebGL `drawArrays`+`drawElements` 호출 횟수 (60프레임 누적, 출력 후 리셋)
- `heap` = `performance.memory.usedJSHeapSize` MB (Chromium 전용)

### 데이터 구조
```js
const _PERF_PROF={
  enabled:false, frameCnt:0,
  ai:[], coll:[], render:[], dom:[], drawCalls:0,        // 윈도우 + 카운터
  _aiAcc:0, _collAcc:0, _renderAcc:0, _domAcc:0,         // 프레임별 누산기 (tick 끝에 리셋)
  _hooked:false, _origACR:null, _origDCW:null            // 후킹 백업 참조
};
```

### 후킹 함수
- `_perfHookGL()` — 1회만 실행. WebGL 드로우콜 카운터 + `_applyColResult`/`_dispatchColWorker` 함수 교체. 원본은 `_origACR`/`_origDCW`에 보관.
- `_perfFrameTick()` — 매 프레임 마지막에 호출. 활성 몹 카운트 → ON/OFF 토글 → 누산값을 윈도우에 push → 60프레임마다 `_perfReport()`.
- `_perfReport()` — 콘솔 출력. 빈 배열 시 `'0.0/0.0'`.

### 주의
- 측정 코드가 활성화되지 않으면 (몹 ≤ 300) 모든 wrapping은 `if(_PERF_PROF.enabled?...:0)` 가드로 zero-cost
- 충돌 후킹은 `_PERF_PROF.enabled` 체크 후에만 누산 (OFF 시 패스스루)
- update 내부의 인라인 충돌 검사(투사체↔적, 플레이어↔적)는 측정 안 됨 — `ai` 항목에 포함됨

### S7 세부 분해: `[S7-BREAKDOWN]` (2026-04-17 추가)

`drawP` 함수의 S7 구간(pProjs+보호막+펫) 내부 세부 측정. 기존 `[S7-DETAIL]` 로그를 보존하고, **300마리 초과 + S7>8ms 동시 충족 시**에만 추가 출력.

**위치**:
- 진입부(라인 ~41224): `_s7t0` 직후 — 카운터 스냅샷 (`_s7DC0`/`_s7Pg0`/`_s7Tx0`/`_s7Bl0`)
- 종료부(라인 ~42026): `[S7-DETAIL]` 직후 — 델타 계산 + 로그
- **drawP 본체 로직 무수정** — 진입/종료 2 곳에만 measurement 코드 주입

**GL 카운터 후킹**:
`_perfHookGL()` 확장 — drawArrays/drawElements 외에 `useProgram`/`bindTexture`/`blendFunc`/`blendFuncSeparate`도 후킹하여 state change 카운트 (`_PERF_PROF._glProg/_glTex/_glBlend`).

**출력 형식**:
```
[S7-BREAKDOWN] iter=1.2ms setup=24.6ms draw=5.8ms post=0.6ms | instances=180 glStates={prog:180, tex:180, blend:12}
```

**필드 의미**:
| 필드 | 의미 | 측정 방식 |
|---|---|---|
| `iter` | pProjs 루프 시간 | `_s7t1 - _s7t0` (실측) |
| `setup` | state change 부하 추정 | `(shield+pet 시간) × (state 변경 비율) × 0.95` (휴리스틱) |
| `draw` | drawcall 부하 추정 | `(shield+pet 시간) × (drawcall 비율) × 0.95` (휴리스틱) |
| `post` | 후처리 잔여 | `(shield+pet 시간) × 0.05` (고정 휴리스틱) |
| `instances` | S7 구간 drawArrays/drawElements 호출 수 | 카운터 델타 (실측) |
| `glStates.prog` | useProgram 호출 수 | 카운터 델타 (실측) |
| `glStates.tex` | bindTexture 호출 수 | 카운터 델타 (실측) |
| `glStates.blend` | blendFunc(+Separate) 호출 수 | 카운터 델타 (실측) |

**제약 — setup/draw/post 분할 정확도**:
drawP 본체 무수정 + 진입/종료 2곳만 수정 제약상 shield+pet 구간 내부에 인라인 마커 추가 불가. 따라서 setup/draw/post는 **GL 호출 수 비율 기반 휴리스틱 분할**. iter와 instances/glStates는 실측이므로 신뢰 가능. 정확한 sub-segment 측정이 필요하면 추후 인라인 마커 단계적 추가 검토.

**검증 방법**: 휠윈드 발동 + 500마리 상태에서 콘솔에 `[S7-BREAKDOWN]` 출력 확인. 출력은 60프레임 무관하게 매 프레임 조건 충족 시 (필요하면 throttle 추가 가능).

### S7 진단 보조 로그 (2026-04-17 추가)

`[S7-BREAKDOWN]`이 안 나올 때 원인 디버깅용으로 두 종류 보조 로그 추가 (drawP 본체 무수정, 기존 S7 종료부 injection 위치 확장).

| 로그 | 조건 | 용도 |
|---|---|---|
| `[S7-SKIP]` | `_dpT7>8 && !_s7Active` | S7 무거운데 PROF 꺼져서 BREAKDOWN 안 나오는 케이스 — gate 값 출력 |
| `[S7-WARN]` | `!_PERF_PROF.enabled && _dpT7>15` | PROF 비활성인데도 S7가 15ms 초과 — 1회 경고 (3초 throttle) |

**`_PERF_PROF._s7WarnedOnce`**: WARN 1회 throttle 플래그. `setTimeout(3000)`으로 자동 false 복귀.

**출력 예시**:
```
[S7-SKIP] BREAKDOWN gate fail | _dpT7=12.34 _s7Active=false
[S7-WARN] 고비용 S7 감지 but PROF 비활성: 18.2ms
```

### S7 Canvas2D 호출 카운터: `[S7-C2D]` (2026-04-17 추가)

S7 구간 내 Canvas2D API 호출 횟수를 실측. GL 직접 호출이 0건인 S7에서 **실제 병목은 Canvas2D 상태 전환**이므로 이 카운터가 핵심 지표.

**후킹 방식** (`_perfHookC2D()`, `_perfHookGL()` 내부에서 1회 호출):
- **메서드 래핑**: `save`, `restore`, `drawImage`, `fillRect`, `arc` — 원본을 `X.__orig_*`에 백업 후 카운터 삽입 래퍼로 교체
- **Setter 래핑**: `globalCompositeOperation`, `globalAlpha` — `Object.defineProperty`로 getter/setter 재정의. 원본 디스크립터를 `_PERF_PROF.__origDesc_*`에 백업 (복구 가능). ProxyX(own descriptor) / native Canvas2D(prototype descriptor) 자동 감지

**카운터 필드** (`_PERF_PROF`):
`_c2dSave`, `_c2dRestore`, `_c2dGco`, `_c2dAlpha`, `_c2dDrawImage`, `_c2dFillRect`, `_c2dArc`

**리셋 타이밍**: S7 진입부(라인 ~41240)에서 모든 카운터 0 초기화
**출력 타이밍**: S7 종료부(라인 ~42072), `[S7-BREAKDOWN]` 직후

**출력 형식**:
```
[S7-C2D] save=30/restore=30 gco=25 alpha=40 drawImage=15 fillRect=3 arc=2
```

**오버헤드**: `_PERF_PROF.enabled` guard — 300마리 이하면 카운터 증가 skip (함수 호출 자체는 래퍼 경유하지만 if 체크 1회로 종료)
