# 12. 퍼포먼스 · 최적화

## AudioContext 안정성 & 노드 폭주 방지

| 항목 | 값 |
|---|---|
| AudioContext 옵션 | `latencyHint:'playback', sampleRate:44100` |
| 워치독 (`_actxWatchdog`) | 500ms 간격 `setInterval`, suspended→resume, closed→전체 재생성 |
| 재생성 시 초기화 | `_actx, _mbus, _busHit, _busSk, _busBoss, _busHitIn, _busSkIn, _busBossIn, _sfxGain` 전부 null/빈객체 |
| 동시 활성 노드 제한 | `_MAX_ACTIVE_NODES=64`, 초과 시 큐 드롭 |
| 노드 카운터 | `_activeNodeCnt` — `_playSampleNow` start 시 +1, `onended` 시 -1 |
| 큐 처리 조건 | `_cnt < _SFX_PER_FRAME(8) && _activeNodeCnt < 128` |
| 시간축 분산 | 같은 키 중복 시 22ms(`_n*0.022`) 간격 딜레이, 다른 키는 즉시 재생 |

## 게임 로딩 시스템

### 부팅 시퀀스 (startGameFromDB / 로컬 테스트)

| 단계 | % | 내용 |
|------|---|------|
| 1 | 25 | 기본 데이터 준비 (mkP, INV, QSLOTS 초기화) |
| 2 | 55 | DB/로컬 세이브 로드 (dbLoad / fetch /api/load) |
| 3 | 75 | 몬스터 아틀라스 대기 (`_mobAtlas.ready`, 최대 5초 타임아웃) |
| 4 | 80 | applyStats → initStage (맵 생성, 몬스터 스폰) |
| 5 | 90 | 맵 에셋 로딩 대기 (`_waitObjSprites`, 최대 8초 타임아웃) |
| 6 | 100 | 완료 → 로딩 화면 페이드아웃 → 게임 시작 |

### _waitObjSprites(timeout)

`initStage()` 후 맵 오브젝트/VFX 이미지가 전부 로드될 때까지 로딩 화면에서 대기.

**체크 대상:**
- `_OBJ_SPR` — 맵 오브젝트 스프라이트 전체 (pit, tree, deco 등)
- `_bossRoomBg` — CH1 보스룸 배경 이미지
- `_bossGateImg` — CH1 보스문 눈나무 이미지
- `_impSpr` — 기본 타격 임팩트 스프라이트 시트

**동작:** 50ms 간격 폴링, 모든 이미지 `complete` 또는 타임아웃 시 resolve.

### 로딩 UI

- `showBootLoading(msg)` — 로딩 오버레이 표시
- `setBootLoading(pct, msg)` — 진행률 + 메시지 갱신
- `hideBootLoading()` — 로딩 오버레이 숨김
- 최소 표시 시간 5초 (인트로 연출 겸용)

### 핵심 규칙

1. **initStage 후 반드시 _waitObjSprites 호출** — 에셋 미로드 상태로 게임 시작 금지
2. **새 이미지 에셋 추가 시 _waitObjSprites 체크 대상에 포함** — `_extraImgs` 배열에 추가
3. **타임아웃은 8초** — 네트워크 지연 대비, 타임아웃 시 로드된 것만으로 시작

## 적(ens) AI 업데이트 최적화

### 뷰포트 컬링 (1차 필터)
카메라 화면+300px 밖 몹은 이동/공격 AI 완전 스킵.
- 보스(`e.ib`), 피격중(`e.hurt>0`), 근접 30px 이내 → 항상 업데이트
- 화면 밖 + alerted 몹 → 8프레임마다 flowfield 추적만 (`_ffMoveE`, sp×4, 0.5)
- hp 감소/상태 업데이트(reviveIframes 등)는 컬링 전에 처리 → 영향 없음

### 4단계 AI 티어링 (2차 필터, 뷰포트 안쪽에서 적용, 2026-04-14 개편)

**보스(`e.ib`)는 항상 T1 강제.**
짝홀 분산 제거 → 인덱스 기반 분산 `(_ei & mask) === (_eTierFrame & mask)`.

| 티어 | 거리² | 거리 | 빈도 | dt 보정 | 비고 |
|------|-------|------|------|---------|------|
| T1 | <22500 | <150px | 매 프레임 | sp | 보스/피격중 포함 |
| T2 | <122500 | <350px | 2프레임 | sp×2 | |
| T3 | <490000 | <700px | 4프레임 | sp×4 | |
| T4 | ≥490000 | 700px+ | 8프레임 | sp×8 (또는 flowfield) | alerted→flowfield |

800마리 기준 매 프레임 풀 AI: ~100~150마리, 물리 업데이트 시간 ~60~70% 절감.

### 타임버짓
32체마다 `performance.now()` 체크, 12ms 초과 시 비보스 스킵.

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|

| Range end 상한 | 요청값 그대로 사용 | `Math.min(end, stat.size-1)` |

| 스트림 에러 핸들링 | 없음 | `createReadStream(...).on('error', ...)`로 500 응답 안전 처리 |

## 아틀라스 텍스처 워밍업 (2026-04-14)

첫 drawImage 시 GPU 텍스처 바인딩 비용이 집중되어 143~614ms 스파이크 발생.
**오프스크린 + setTimeout 백그라운드 방식**: 게임 루프 밖에서 1장씩 비동기 워밍업.

| 구성 | 설명 |
|------|------|
| `_wqCvs` (1×1) | 오프스크린 캔버스 — 메인 캔버스 상태 오염 없음 |
| `_wqBuf[40]` | 고정 배열, 워밍업 대상 이미지 참조 |
| `_warmupEnsAtlas()` | 대상 수집 → `_wqBuf`에 적재 → `_warmupNext()` 시작 |
| `_warmupNext()` | `setTimeout(fn,0)` 체인, 1장씩 `_wqCtx.drawImage` → 게임 루프 **0ms 영향** |

| 호출 시점 | 설명 |
|-----------|------|
| 부트 완료 (`_dbReady=true` 직후) | 큐 적재 + setTimeout 체인 시작 |
| ens 렌더 루프 진입 | 미완료 시 `_warmupEnsAtlas()` 재시도 (지연 로드 커버) |
| `nextStage()` | `_ensWarmDone=false` 리셋 → 다음 렌더 시 재적재 |

워밍업 대상 (~19장): 8dir idle(8) + walk(8) + 보스 3종 + GL 아틀라스 + 구울 3종
