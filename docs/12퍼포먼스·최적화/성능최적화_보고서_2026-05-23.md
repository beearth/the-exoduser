## 1. 요약

성능 측정용 로깅 시스템 강화, 셰이더 프리워밍 코드 47줄 추가, 다수의 옛 가설 검증 및 무효화 작업 진행.
최종 격전 폭타 비용이 **21ms → 17ms (약 4ms 개선)**.

**출시 합격선 도달:**
- 평상시 **140fps 이상**
- 700마리 **60fps**
- 격전 폭타 **55fps**

이후 추가 진단 시도는 **봉인**하며, 실제 게임이 깨지는 명확한 신호가 나타나기 전까지 성능 작업을 중단한다.

## 5. 잔존 이슈

### 5.1 첫 envVig 198ms (1회성)
- **성격**: Canvas2D radialGradient + drawImage 첫 빌드 비용
- **발생 시점**: 게임 첫 프레임에서 1회
- **실제 영향**: 게임플레이 중 절대 미발생. 사용자 체감 0
- **미해결 사유**: boot 시 빌드한 `G._dvgCvs`와 실제 draw에서 참조하는 캐시 객체가 다름. 캐시 키 추적에 추가 진단 필요
- **권장**: 무시

### 5.2 첫 proj 그리기 46ms (1회성)
- **성격**: GPU 셰이더 JIT 컴파일 (linkProgram은 끝났으나 첫 drawElements 시점에 머신코드 생성)
- **발생 시점**: 첫 pProj 발사 시 1회
- **실제 영향**: 1프레임 끊김
- **미해결 사유**: 워밍업으로 부분 개선 (50ms → 46ms). 완전 제거는 `_glMainProg`에 실제 텍스처 바인딩 + 실제 픽셀 출력 필요
- **권장**: 무시

### 5.3 shieldQ setup 19.8ms (반복)
- **성격**: 보호막 Q 발동시 GL 상태 변경 (program 6회, texture 8회)
- **발생 시점**: Q 발동시마다, 보스전 한정
- **실제 영향**: setup 19.8ms + draw 8.5ms + post 1.5ms = 29.6ms 스파이크 1프레임
- **미해결 사유**: GL 상태 변경 자체가 비용. 줄이려면 텍스처 시트 통합 또는 프로그램 통합 필요
- **권장**: 보스전 1-2초 한정이므로 현재 합격선. 진짜 게임 깨지는 신호 나오면 재진단

## 6. 추가 작업 봉인 사유

1. **출시 합격선 도달**: 평상시 140fps 이상, 700마리 60fps, 격전 폭타 55fps
2. **ROI 악화**: 잔존 이슈는 1회성 또는 보스전 한정으로 사용자 체감 영향 미미
3. **도진님 직접 확인**: 보스전에서 렉 없음을 실측 후 명시적으로 확인
4. **기회 비용**: Steam Coming Soon 페이지, ACT1 시네마틱 제작, 마케팅 에셋 작업이 출시 일정상 압도적으로 중요
5. **아키텍처 한계**: 시각 효과 유지 원칙으로 가능한 픽스가 대부분 소진됨

### 6.1 봉인 해제 조건

아래 조건이 충족될 때만 성능 작업 재개:

1. 모든 빌드에서 일관되게 60fps 미만 지속 발생
2. Steam 리뷰에서 다수 사용자가 성능 문제를 명시적으로 보고
3. 새 챕터 / 새 보스 추가로 측정값이 격전 25ms 이상으로 상승

## 7. 운영 원칙 (절대 위반 금지)

- 게임 밸런스 네프, LOD 저하 등 게임성 훼손 해법 **금지**
- 시스템 레벨 순수 렌더링 아키텍처 픽스만 허용
- **어택 티켓 시스템 절대 금지** (몹이 왕창 달려드는 것이 핵슬의 핵심 쾌감)
- 히트스톱은 보스 한정 (메인보스 미니보스 최종보스). 일반몹 엘리트 히트스톱 금지
- **화려함 / 이펙트 축소 금지** (arms 캡, 레이어 축소, spark 캡 전부 거부)
- 게임루프 내 `new`, `splice`, `filter`, `forEach`, `Date.now()` 금지
- `ELC[]`, `ETYPE_COL[]`, `_tseed()`, `StageSeeder`, `CIN_LINES`, 보스 콤보 idx(0-48), 강화 확률 공식 **절대 수정 금지**

## 8. 다음 진단 시 주의사항

향후 누군가 (Claude 포함) 성능 재진단을 시도할 경우 **본 보고서를 먼저 확인할 것**.

1. 옛 메모리 (펫 picker 18ms, xbow+holy 29ms 등)는 **무효**임. 4장 표 참조
2. S3, S7 라벨은 코드 변경 따라 의미가 달라짐. `_PERF_PROF` 라벨-구간 매핑을 먼저 검증
3. 첫 envVig 198ms와 첫 proj 46ms는 **1회성**. 격전 폭타와 혼동 금지
4. `X.drawImage`는 ProxyX를 통해 이미 GL 위에서 도는 중. Canvas2D 비용으로 오해 금지
5. VFX-GL은 additive 고정. blend 모드 다른 효과 합류 불가
