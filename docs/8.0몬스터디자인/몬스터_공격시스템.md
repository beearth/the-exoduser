# 몬스터 공격 시스템

## 핵심 원칙
- **모든 몬스터는 탄막으로만 공격** — 근접 비비기(접촉 데미지) 완전 제거
- 근접 몬스터도 가까이 와서 탄을 발사
- 플레이어와 겹치면 밀어내기 (데미지 없음)
- 탄막은 패링/회피 가능 — **리듬게임식 전투**
- 탄 색상별로 패링 입력이 다름 (빨콩=우클릭, 무지개=Q)

## 투사체 컬링 (despawn)
- **카메라 거리 컬링**: 카메라 중심 기준 **2500px 이상** 떨어진 적 투사체 즉시 제거 (`6,250,000 = 2500²`)
- 정적 장판 제외: mine/trap/poisonZone/iceZone/dimRift/_coralSpike (위치 고정 함정류는 거리 무시)
- **수명 종료(`p.life<=0`)**: 카메라 2500px 안이면 `life=1`로 연장(안 죽음), 밖이면 사망
- **변경 이력**: 2026-04-11 컬링 거리 800px → 2500px 증가 (오픈맵 1_1 등에서 화면 조금만 벗어나도 탄막 사라지는 버그 수정)

## HP 공식 (v9 — 삼차→복리 전환, 500=5000만, 만렙1000=1억)
```
기본커브 = Lv≤500 ? 0.4×Lv³ + 199×Lv + 100 : 50,000,000 × 1.0014^(Lv-500)
최종HP = 기본커브 × (보스?24:1) × dm × et.hpMul × (1 + stage×0.02)
```
- Lv 1~500: **삼차 성장** (0.4×Lv³+199×Lv+100), Lv1=300, +200/렙 시작 → 점진 가속
- Lv 500~1000: **복리 성장** (1.0014^(Lv-500)), 500레벨=5000만, 1000레벨=1억
- 보스: ×24배
- 만렙(1000) 일반몹 HP = **1억**, 보스 = **24억**

### 보스 부활
- 항상 100% 부활 (확률 판정 없음)
- 풀피 부활 (HP 변동 없음) + eShield도 mhp로 갱신
- 부활마다: ATK ×1.5, SPD ×1.25, 포이즈 ×1.5, 패턴쿨 -30f

## 이동속도 공식
```
일반몹: SPD = (ETYPE_SPD[etype] + rand(0~0.08)) × 1.5
보스:   SPD = 0.5 (배율 미적용)
레어:   SPD = (_rd.spd + rand(0~0.05)) × 1.5
```
- M02 모드: ×1.5
- 부활 시: ×1.25 (캡: 일반 1.6, 7장 2.0)

## 포이즈 공식
```
poise = maxPoise = HP × 0.5
```
- 부활 시: maxPoise ×1.5

## 에너지쉴드
```
eShield = eShieldMax = HP (동일값)
```
- HP와 별도 체력바 — 물리/에너지 이원 방어
- 쉴드 리젠: 5초 미피격 후, 쉴드>0일 때 mhp×0.5%/프레임
- 쉴드 완파 시 복구 안 됨

## ATK 공식 (v2 — Lv>500 복리 가속, 2026-04-17)
```
baseLin = (보스?75:50) + monLv×(보스?2.9:1.95)
curve   = monLv≤500 ? baseLin : baseLin × 1.0018^(monLv-500)
ATK     = floor(curve × dm × et.atkMul × 0.5)

monLv = P.lv + 난이도오프셋
dm = DIFF_MUL[난이도]
DIFF_MUL = [1.00, 1.12, 1.25, 1.40, 1.58, 1.78, 2.00, 2.25, 2.52, 2.82, 3.15]
```
- Lv≤500: 기존 선형과 동일
- Lv>500: baseLin 위에 1.0018^(Lv-500) 복리 곱셈
  - Lv600: ×1.197 (~+20%)
  - Lv750: ×1.64 (~+64%)
  - Lv1000: ×2.46 (~×2.5)

## 패링 시스템

### 패링 개념
- 패링 = 탄막을 쳐내서 반사하는 액션
- **리듬게임식**: 탄 색상별 입력이 다르며, 잘못된 입력 시 데미지
- isPWin() = sBash OR (sBlock+패링윈도우) OR (peaceShield+패링윈도우)
- 패링 윈도우: 20프레임(0.33초), 해제 후 12프레임(0.2초)

### 탄 종류별 패링 규칙
| 탄 종류 | 패링 입력 | 패링 성공 시 | 잘못된 입력 시 |

### 반사탄 착탄 폭발
반사탄(projs[].friendly)이 적에게 명중 시 `_aoeDmg` 기반 범위 폭발:

- bomb 반사탄은 별도 경로 (bombR/bombAtk)

### 패링 판정 범위

### 한방캡
- 모든 데미지 경로는 `hurtP()` 통과 → **한방캡(HP 33%)** 적용

### 피격 무적 (iframes)
- DOT 피격: 무적 없음 (P.iframes=0)
- 한방캡 발동: 60f(1초) 무적
- 기절 진입/해제: 60f(1초) 무적
- 패링 성공: 25f 무적
- 사슬기동 착지: 48f(0.8초) 무적

### 방어력 (DR)
- 물리방어: 갑옷/방패/장갑/바지/부츠 → `pDef/(pDef+2500)` (캡 75%)
- 에너지방어: 투구/목걸이/반지 → `eDef/(eDef+2500)` (캡 75%)
- 에너지쉴드: 물리 → 25%만 적용, 에너지 → 풀 적용
- HP: 물리 → 풀 적용, 에너지 → eDR 적용
- 어픽스: barrier(고정 감소), physDR(추가 DR%)
