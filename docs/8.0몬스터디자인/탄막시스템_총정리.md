## 1-B. 탄막 데미지 3-Tier 시스템 (2026-04-17)

### 개요
**이전 시스템의 문제**:
- Caller의 dmgMult(변수: 0.8~1.5) × beanRoll의 배율(1.0~1.5) = 최종배율 0.8~2.25배 (불규칙적 & 예측 불가)

**새 시스템**:
- **탄 종류에만 고정된 3단계 Tier** 적용
- **Caller의 dmgMult 수치는 완전히 무시**
- 최종 데미지 = `floor(ATK × Tier배율 × 엘리트배율 × dmg버프)`

### Tier 정의

| Tier | 분류 | 포함 탄막 | 데미지 배율 | 설계 의도 |
|------|------|----------|----------|---------|
| **Tier 1** | 일반탄/들러리 | 화이트볼(bwBean), 다크볼(gbBean), 노랑탄, 보라탄 | **×1.2** | 약한 기본 공격, 플레이어 유리 |
| **Tier 2** | 중거리 특수탄 | 빨콩(redBean), 근접빨콩(_closeBean) | **×1.5** | 중간 난이도, 패링 필수 |
| **Tier 3** | 강력 추적탄 | 무지개탄(blackBean, 500px 밖만) | **×2.0** | 최강 탄막, 리듬게임 난이도의 핵심 |

### 예시 계산

**보스 Lv500, ATK=1025**

| 탄막 | Tier | 기본배율 | 엘리트배율 | 최종 데미지 |
|-----|------|--------|----------|----------|
| 화이트볼 | T1 | ×1.2 | ×1.0 | 1025×1.2 = **1230** |
| 빨콩 | T2 | ×1.5 | ×1.0 | 1025×1.5 = **1538** |
| 무지개탄 | T3 | ×2.0 | ×1.0 | 1025×2.0 = **2050** |

**보스 부활 1회 이후** (ATK ×1.5 = 1537):
| 탄막 | Tier | 기본배율 | 엘리트배율 | 최종 데미지 |
|-----|------|--------|----------|----------|
| 화이트볼 | T1 | ×1.2 | ×1.0 | 1537×1.2 = **1845** |
| 무지개탄 | T3 | ×2.0 | ×1.0 | 1537×2.0 = **3074** |

### Tier 적용 규칙

1. **특수탄(원소구, 팬텀소드, 함정류)**: Tier 미적용, 각자 고정 배율 사용 (×3.0, ×1.8, ×0.3 등)
2. **엘리트 보너스**: Tier 배율 적용 후, 엘리트 atkMul 곱셈 (예: T1 ×1.2 × 엘리트 ×1.3)
3. **부활 기간**: ATK 상향으로 자동 증가 (Tier 배율 고정)

## 3. _beanRoll — 탄 종류 결정 로직 (2026-04-17 dmgMult 무시)

### 근거리 자동 선택 (dst < 500px)
500px 이내면 무조건 **빨콩(redBean)** → `_closeBean=true`, `spd=max(4,기본)`, `homing=true`

### 교대 카운트 기반 (e._beanCnt)
| 조건 | 결과 |
|------|------|
| `_beanCnt % 4 === 0` | 무지개탄(blackBean) 강제 |
| `_beanCnt % 4 !== 0` | 아래 확률 테이블 |

### 확률 테이블 (교대 아닌 발사)
| 확률 | 탄 종류 | Tier |
|------|--------|------|
| 60% | 무지개탄(blackBean) | T3 (×2.0) |
| 22% | 빨콩(redBean) | T2 (×1.5) |
| 6% | 화이트볼(bwBean) | T1 (×1.2) |
| 6% | 다크볼(gbBean) | T1 (×1.2) |
| 3% | 원본 속성탄 | 고정 |
| 3% | 암흑탄 | 고정 |

### dmgMult 파라미터 (현재 무시됨)
- **이전**: `eProjAt(e, ang, spd, dmgMult, ...)`에서 dmgMult 값이 beanRoll과 곱해짐
- **현재**: dmgMult 파라미터는 함수 시그니처에만 남아있고, **실제 데미지 계산에 미반영**
- **이유**: Tier 고정 시스템으로 전환하여 예측 가능한 밸런싱 달성
- **마이그레이션**: 호출처의 dmgMult 값을 변경해도 최종 데미지 영향 없음

### pParryProjDmg 공식
```javascript
function pParryProjDmg(bulletDmg, isQ){
  const scale = isQ ? magicRef()*statInt()*pMagicMul()    // Q: 마법 기반
                     : meleeRef()*statStr()*pAtkMul();     // 우클릭: 물리 기반
  const _gearBonus = Math.min(1, scale/300);               // 장비보너스 0~100%
  const _base = bulletDmg * (1 + _gearBonus);
  const _afxPB = 1 + _eqAffix('parryBonus');               // 패링보너스 어픽스
  const _legSP = sh()._lShieldParry || 1;                  // 레전더리 패시브
  return ~~(_base * pParryDmg() * _afxPB * _legSP);
}
```

## 11. 회전참 중 탄 흡수

회전참(whirlwind) 상태에서 범위 내 탄을 자동 흡수:
- **흡수 대상**: 무지개탄 제외 일반탄 (blackBean/bwBean/gbBean/poisonZone/mine/elemBall/dimRift 제외)
- **흡수율**: pGuardAbsorb (보호막 흡수율과 동일)
- **parryBank 축적**: 흡수 데미지의 100%
- **무지개탄**: iframes 중에만 범위 내 소멸 (흡수율 적용), **패링 불가**

### 회전기폭(whirlDet) 합체 시 추가
- 흡수한 탄 → 회전참 데미지·범위 성장 (parryBank/500 비례)
- Q키 수동기폭 / 2초 자동폭파 트리거

## 13. eProjAt — 적 탄 발사 함수

```javascript
function eProjAt(e, ang, spd, dmgMult, el, life, col, sz){
  life = ~~(life * 1.5);              // 수명 1.5배
  const b = _beanRoll(el, col, sz, spd, dmgMult, e);  // 탄 종류 결정
  spawnProj({
    x, y,                             // 입 위치(_eMouthXY)
    vx: cos(ang)*b.spd, vy: sin(ang)*b.spd,
    dmg: ~~(e.atk * b.dmgM),
    el: b.el, friendly: false,
    life: b.blackBean ? 400 : life,    // 무지개탄 수명 고정
    col: b.col, sz: b.sz,
    redBean, blackBean, bwBean, gbBean, homing, _closeBean
  });
}
```
