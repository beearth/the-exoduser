# 보스 스켈레탈 애니메이션 시스템

> 최종 수정: 2026-04-07
> 적용 범위: **보스만** (일반 몹은 기존 스프라이트시트 유지)
> 위치: `game.html` — `drawBossBody()` 함수 (최종 폴백 렌더러)

---

## 1. 개요

기존 `drawBossBody()`의 하드코딩 sin 애니메이션을 **파츠별 관절 회전(스켈레탈)** 방식으로 교체.
스프라이트시트 렌더러(`atlas_bosses.png` 등)가 로드된 보스는 영향 없음 — 이 시스템은 **폴백 렌더러(순위 7)**에만 적용.

### 렌더 우선순위 (변경 없음)

| 순위 | 시스템 | 소스 |
|---|---|---|
| 1 | 외부 보스 아틀라스 | `atlas_bosses.png` + `.json` |
| 2 | ch 아틀라스 (mobSkin) | `_chAtlas[ch][dir].img` |
| 3 | 방향별 아틀라스 | `atlas_boss_si0_dir.png` |
| 4 | 걷기 아틀라스 | `atlas_boss_walk.png` |
| 5 | 내부 보스 아틀라스 | 프리렌더 캔버스 |
| 6 | 일반 적 아틀라스 폴백 | `_atlas` UV[10] |
| **7** | **프로시저럴 (스켈레탈)** | **`drawBossBody()` ← 여기 적용** |

## 2. 파츠 구조 (관절 9개)

캐릭터 180cm 기준, 보스 `e.r` = 30~50px.

| 파츠 | 키 | 피벗 위치 (facing 기준) | 드로잉 |
|---|---|---|---|
| 몸통 | `body` | 원점 (0,0) | 타원 arc, `r:` 기울기 + `y:` 상하 오프셋 |
| 왼앞다리 | `legFL` | 몸통 가장자리 뒤쪽 좌 | stroke 선분, `r:` 다리 각도 |
| 오른앞다리 | `legFR` | 몸통 가장자리 뒤쪽 우 | stroke 선분, `r:` 다리 각도 |
| 왼뒷다리 | `legBL` | 몸통 가장자리 뒤쪽 좌 | stroke 선분, `r:` 다리 각도 |
| 오른뒷다리 | `legBR` | 몸통 가장자리 뒤쪽 우 | stroke 선분, `r:` 다리 각도 |
| 꼬리 | `tail` | 몸통 뒤쪽 (facing+π) | quadraticCurveTo + 끝 삼각형, `r:` 흔들림 |
| 무기(곤봉) | `weapon` | 몸통 오른쪽 (facing+0.45) | stroke 선 + 머리 원 + 가시, `r:` 스윙 각도 |
| 왼뿔 | `hornL` | 몸통 앞쪽 좌 (facing-0.35) | 삼각형 fill, `r:` 기울기 |
| 오른뿔 | `hornR` | 몸통 앞쪽 우 (facing+0.35) | 삼각형 fill, `r:` 기울기 |

### 피벗 비율 (e.r 기준)

- 다리 피벗: `r * 0.4` (몸통 가장자리)
- 다리 길이: `r * 0.6 + 10`
- 꼬리 피벗: `r * 0.5`
- 꼬리 길이: `r * 1.5`
- 무기 피벗: `r * 0.6`
- 무기 길이: `r * 1.8 + 6`
- 뿔 시작: `r * 0.8`
- 뿔 끝: `r + 16`

## 3. 포즈 테이블 (`_bPose`)

각 상태는 함수 `(t) → {파츠: {r, y}}` 형태. `t = _now * 0.003`.

| 상태 | body.r | body.y | weapon.r | 다리 | 꼬리 | 설명 |
|---|---|---|---|---|---|---|
| `idle` | 0 | sin(t×2)×0.18 | 0 | ±sin(t×3)×0.4 | sin(t×1.5)×0.5 | 숨쉬기 + 다리 흔들림 |
| `breath` | 0 | sin(t×1.5)×0.15 | -0.4 | ±sin(t×2)×0.2 | sin(t)×0.35 | 회복 중 느린 호흡 |
| `windup` | -0.3 | -0.12 | **-2.0** ± 0.2 | 앞 -0.6, 뒤 +0.25 | -0.6 | 무기 뒤로 크게 들기 |
| `attack` | +0.4 | +0.08 | **+2.5** | 앞 +0.8, 뒤 -0.5 | +0.8, 뿔 ±0.9 | 무기 앞으로 휘두르기 |
| `walk` | 0 | sin(t×4)×0.08 | sin(t×6)×0.25 | ±sin(t×6)×1.0 | sin(t×3)×0.6 | 대각선 교차 보행 |
| `stagger` | sin(t×15)×0.4 | 0 | +1.2 | ±0.3~0.5 | sin(t×12)×0.8 | 그로기 흔들림 |
| `death` | f×1.8 | f×0.2 | f×2.5 | f×1.0~1.3 | f×0.9 | 서서히 쓰러짐 (f=0→1) |
| `rage` | sin(t×5)×0.12 | sin(t×3)×0.06 | sin(t×5)×0.4 | ±sin(t×4)×0.45 | sin(t×2)×0.65 | 분노 진동 |

- `rage_attack` = attack 기반, body.r: 0.8 (더 앞으로 숙임), 뿔 ±1.1 (더 과격)
- `rage_windup` = `windup`과 동일 포즈

## 4. 보간 시스템 (`_bLerpPose`)

### 전역 풀

```
_bCurPose[entityId][partName] = {r: 현재각도, y: 현재Y}
_bPoseTs[entityId] = 상태 진입 시각 (death 타이밍용)
_bPosePrev[entityId] = 이전 상태명
```

### 보간 공식

```
current += (target - current) × min(speed × dt, 1)
```

### 상태별 보간 속도

| 상태 | speed | 의미 |
|---|---|---|
| `attack` / `rage_attack` | **15** | 빠른 전환 → 내려치기 날카로움 |
| `death` | **3** | 천천히 쓰러짐 |
| 그 외 | **8** | 기본 |

### 모션 리딩 핵심

```
windup (spd=8, 0.3초) → attack (spd=15, 즉시)
```

1. `windup` 진입: 무기가 `r = -1.2` (뒤로) → spd=8로 0.3초에 걸쳐 도달
2. `attack` 전환: 무기가 `r = +1.5` (앞으로) → spd=15로 거의 즉시
3. 이 시간차가 플레이어에게 **"지금 피해!"** 시그널 제공

---

## 5. death 타이밍

- `_bGetDeathT(e, now)`: 상태 전환 감지 → death 진입 시각 기록
- `t = (now - 진입시각) × 0.001` (초 단위)
- 포즈 함수에서 `f = min(t × 0.5, 1)` → **2초에 걸쳐 완전히 쓰러짐**
- 보간 속도 3 → 추가 부드러움

## 6. 핫패스 규칙 (성능)

drawBossBody는 보스 1체에만 호출되므로 병목 아님. 그래도 준수:

- **금지**: `new`, `splice`, `filter`, `forEach`, `Date.now()`, `new Audio()`
- **금지**: `includes()` → `indexOf() !== -1` 사용
- 오브젝트 풀링: `_bCurPose`는 전역 캐시, 매 프레임 할당 없음
- `var` 사용 (function scope)
