# 14. 밸런스 + 수치 테이블

## 스탯 5종 (STATS)
| 스탯 | 효과 |
|---|---|
| STR | 물리 공격력 배율 (`statStr()`) |
| DEX | 공격속도/회피 배율 (`statDex()`) |
| INT | 마법 공격력 배율 (`statInt()`), INT 100+에서 에너지쉴드 해금 |
| VIT | 최대 HP/방어력 |
| LCK | 크리티컬/드롭률 |

## 데미지 공식
- `hurtE(e, dmg, ang)` — 적에게 데미지
- `hurtP(dmg)` — 플레이어 피격
- DEF 감산, 50% 캡
- **수정 금지**

## 강화 시스템
- `enhMul()`, `enhCost()` — +1~+20
- 강화 확률: 99 × 0.99^n %
- **수정 금지**

## 레벨
- `P.lv` 상한 없음 (1000+ 초월 시스템)
- `addExp(v)` 경험치 추가
- `_calcMaxExp(lv)`: lv<1000 → `15+lv×2.5+lv²×0.05`, lv≥1000 → `50000+(lv-999)²×1000`

### 초월 레벨 (Lv.1000+)
| 항목 | 값 |
|---|---|
| 경험치 공식 | `50000 + (lv-999)² × 1000` |
| 경험치 고정값 | 일반몹 1, 엘리트 2, 보스 10 (기존 공식 무시) |
| 일일 레벨업 제한 | **1회/일** (`P._transLvDate`, `P._transLvCount`) |
| 보상 | SP +3/렙, 5렙마다 SP+2, 10렙마다 AP+2 (일반과 동일) |
| HUD 표시 | 경험치바 보라색, 텍스트 "⚡ T.{lv}" |

예시 필요 경험치:
| 레벨 | maxExp |
|---|---|
| 1000→1001 | 51,000 |
| 1010→1011 | 171,000 |
| 1050→1051 | 2,651,000 |
| 1100→1101 | 10,251,000 |
