# 14. 밸런스 + 수치 테이블

**주요 변경**: 
- **레벨 직접 보너스 제거**: 레벨은 모든 스탯 +0.5/Lv만 제공 → 간접적으로 HP/ST/MP 증가
- **스탯 기반 시스템으로 통일**: HP/ST/MP/Shield/리젠 모두 스탯 투자의 보상
- **레벨 1000 = 모든 스탯+근성 +500 자동** (STR+500, DEX+500, INT+500, LCK+500, 근성+500)

## 크리티컬 공식
- 크리 발동 시 데미지 = 일반뎀 × `statCritDmg()`
- `statCritDmg() = 1.5 + LCK×0.002 + pCrit×0.08 + 어픽스 + 결정` — **기본 +50%** (1.5배)
- LCK 1000 = 1.5 + 2.0 = **3.5** (크리뎀 +200%)
- 크리뎀 어픽스: 반지 +5~25%, 목걸이 +10~50%
- 크리 확률: `STATS.lck × 0.025 + pCrit×1.5 + 반지/목걸이 critRate + 결정`

## HP/ST/MP/Shield 최대치 공식 (2026-04-17 레벨 성장 통합, bonusHp/bonusShield/bonusSt/bonusMp 슬롯 분리)

**공식 기반**: 모든 스탯은 `s = {str, dex, int, lck}` (레벨 자동 성장 포함)으로 계산

```
s.str = STATS.str + eqStr + floor(Lv × 0.5)
s.dex = STATS.dex + eqDex + floor(Lv × 0.5)
s.int = STATS.int + eqInt + floor(Lv × 0.5)
s.lck = STATS.lck + floor(Lv × 0.5)

mhp = (stageHp + s.str × 1 + bonusHp + affixes) × (1 + hpPct)
mst = 200 + s.dex × 2 + bonusSt + maxSTFlat어픽스 + 결정ST
mmp = 200 + s.int × 2 + bonusMp + maxMPFlat어픽스 + 결정MP
shield = 100 + s.int × 5 + bonusShield + affixes
```

**slot-specific bonus assignments (2026-04-17)**
- **방어구 6슬롯** (shield, armor, boots, gloves, pants, cape): `bonusHp = tier×100 + random(20~100)` + `bonusSt = tier×100 + random(20~100)` — bonusShield 없음
- **장신구 6슬롯** (belt, necklace, ring1, ring2, bracelet, headband): `bonusShield = tier×100 + random(20~100)` + `bonusMp = tier×100 + random(20~100)` — bonusHp 없음
- **무기 3슬롯** (weapon, bow, helmet): bonusHp / bonusShield / bonusSt / bonusMp 모두 없음

**범위 (tier별 random(20~100))**:
| tier | bonusHp/bonusSt/bonusShield/bonusMp 범위 |
|---|---|
| T0 | 20~100 |
| T1 | 120~200 |
| T2 | 220~300 |
| T3 | 320~400 |
| T4 | 420~500 |

**최대 총합** (T4 최대값 기준):
- HP: 방어구 6슬롯(bonusHp) × 500 = **3,000 HP**
- ST: 방어구 6슬롯(bonusSt) × 500 = **3,000 ST**
- Shield: 장신구 6슬롯(bonusShield) × 500 = **3,000 Shield**
- MP: 장신구 6슬롯(bonusMp) × 500 = **3,000 MP**

- **NO P.lv×1 직접 보너스** — 레벨은 스탯만 증가시킴
- HP/ST/MP/Shield 모두 스탯 투자의 보상

## 데미지 공식
- `hurtE(e, dmg, ang)` — 적에게 데미지
- `hurtP(dmg)` — 플레이어 피격
- DR 공식: `DEF / (DEF + 2500)`, 캡 75% (`DR_CAP=0.75`)
  - 물리: `pDR = min(0.75, pDef/(pDef+2500) + physDR어픽스)`
  - 속성: `eDR = min(0.75, eDef/(eDef+2500))` — **전 속성 공통** (화염/냉기/암흑/번개/신성)
  - DEF 2500 → 50%, DEF 7500 → 75%(캡)
- **방어구 베이스 DEF/eDef 대폭 하향** (2026-04-10): 방어도 올리기 극악 난이도. 풀 T4레전더리 장비 DEF ≈ 50, eDef ≈ 20 → 강화/어픽스로 올려야 함
- **eDef 어픽스**: `eDefFlat` SUFFIX 3티어 [50/100/150], weight 40 (희귀), 슬롯 armor/helm/neck/shield/cloak/bracelet/headband/ring
- **eDef 장착 슬롯**: helmet, necklace, ring1/2, cape, shield, armor, boots, gloves, pants, belt, bracelet, headband (어픽스 롤링 슬롯 전부 합산)
- **한방 데미지 캡**: HP의 33% (`_hitCap=~~(P.mhp*.33)`), 발동 시 60f(1초) 무적
- **최소 데미지 1 보장** — DR 중첩 시 `~~()` 절삭으로 0뎀 방지. STR 배율 적용 후 `if(dmg<1)dmg=1`
- **엘리트/챔피언 시스템 비활성화** — `rollElite()` 항상 0 반환. 모든 몹 일반 등급. 모디파이어(M01~M28)/오라(함성/치유/신속/수호) 미적용
- **수정 금지**

### 벽 충돌 데미지
- 공식: `max(넉백속도×4, 적 최대HP × min(8%, 넉백속도×0.4%))`
- 고정뎀 vs %뎀 중 큰 쪽 적용 → 만렙에도 유효
- 넉백속도 2 이하: 데미지 없음 (임계값)
- 예시: 넉백속도 10 → max(40, mhp×4%) / 넉백속도 20 → max(80, mhp×8%캡)

## 강화 시스템
- `enhMul()`, `enhCost()` — 상한 없음 (1000+ 가능)
- 강화 비용: `ceil((1 + n×0.15) × 1.5)` × 등급배율 (50% 상향 적용됨)
- 등급배율: 일반 0.5, 마법 0.7, 희귀 1.0, 영웅 1.3, 전설 1.8
- 강화 확률: max(0.5, 99 × 0.99^n) % — 최소 0.5% 하한 보장 (2026-04-22, 기존 공식 유지 + 하한 추가)
- **강화 성공률 표시 정밀도** (2026-04-17): `game.html:34443` 표시 반올림 수정. ≥10% 정수 / 1~10% 소수점1자리 / 0.01~1% 소수점2자리 / <0.01% 소수점3자리. 이전 `~~(rate*100)` 내림 방식은 <1% 구간에서 전부 "0%"로 표시돼 +500 이상 강화 시 "왜 0%냐" 혼란 유발 (+500=0.65% / +831=0.023% 등). 확률 공식은 유지, 표시만 정밀화.
- **강화 공식 (ATK)** `enhMulAtk(n)`: 1강당 **+0.5** (선형, 2026-04-18 반감)
- **강화 공식 (DEF)** `enhMul(n)`: 1강당 **+0.25** (선형, 2026-04-12 하향)
- **무기류** (weapon, bow, helmet): ATK + enhMulAtk(enh) = ATK + 강화×0.5
- **아이템 ATK 맥스**: `(200 + tier×200 + P.lv) × 0.5` (2026-04-18 반감, T4 Lv600 = 800)
- **방어구** (shield~headband 13슬롯): pDef + enhMul(enh) + **eDef 동시 상승** (강화 ×0.25 전 속성방어 공통) + HP +0.5/강화레벨
- **방어구 강화 ST** (2026-04-17): 방어구 6슬롯(shield/armor/gloves/pants/boots/cape) 1강당 **+0.1 ST** → +100강 = +10 ST
- **악세사리 강화 MP** (2026-04-17): 악세 6슬롯(belt/bracelet/ring1/ring2/necklace/headband) 1강당 **+0.1 MP** → +100강 = +10 MP
- **방어구 기본 HP**: bonusHp = tier×100 + random(20~100) (T4=420~500) — 6슬롯, 풀세트 HP = 6×500(최대) = **3000**
- **장신구 기본 쉴드**: bonusShield = tier×100 + random(20~100) (T4=420~500) — 6슬롯, 풀세트 Shield = 6×500(최대) = **3000**
- **방어구 베이스 DEF** (물리전용): `20 + tier×20` (T0=20, T4=100) — 방어구 6슬롯(shield/boots/armor/gloves/pants/cape) × 100 = **최대 600 pDef**
- **방어구 기본 bonusHp + bonusSt**: `tier×100 + random(20~100)` (T0=20~100, T4=420~500) — 방어구 6슬롯 × 500 = **최대 3,000 HP / 3,000 ST**
- **장신구 베이스 eDef** (속성전용): 전 6슬롯(벨트/목걸이/반지1/반지2/팔찌/귀걸이) `20+tier×20` (T4=100) — 6×100 = **최대 600 eDef**
- **장신구 기본 bonusShield + bonusMp**: `tier×100 + random(20~100)` (T0=20~100, T4=420~500) — 장신구 6슬롯 × 500 = **최대 3,000 Shield / 3,000 MP**
- 방어구: HP + ST, 장신구: Shield + MP — 분리된 자원 구조
- **어픽스**: maxHPFlat [60/100/200/300/420] — 방어구 슬롯만 장착 가능 / shieldFlat [120/200/400/600/840] — 장신구 슬롯만 장착 가능

### SP/AP 저장·복원
- 로드 시 저장된 `P.sp`, `P.ap` 값을 **그대로 복원** (재계산 없음)
- SP 변동은 레벨업·스킬습득·합체·스탯투자·보스클리어 등 이벤트마다 실시간 반영 + 자동저장
- 기본 스킬 (kiSlash, fireball): **완전 무료 보너스** — 습득비·강화비 0, 해제해도 SP 환불 없음

## 장비 ATK 시스템 (디아블로식)

### base ATK 생성
```
_atkMax = 200 + tier × 200 + P.lv    (tier + 레벨 스케일)
_atkFloor = [0, 0.05, 0.15, 0.25, 0.40][rarity]   (전설=40% 최소보장)
_atkRoll = _atkMax × (floor + (1-floor) × random²)  (skew: 고뎀 극악)
```

| tier | Lv1 _atkMax | Lv500 _atkMax | Lv1000 _atkMax |
|---|---|---|---|
| 0 | 201 | 700 | 1200 |
| 1 | 401 | 900 | 1400 |
| 2 | 601 | 1100 | 1600 |
| 3 | 801 | 1300 | 1800 |
| 4 | 1001 | 1500 | 2000 |

- 무기/석궁/투구 동일 공식, wtype/btype의 atkMul 추가 곱셈
- 어픽스 3종(예리한/맹렬한/파멸의) 각 50~330 추가 (skewRoll:3)
- 풀옵 이론 최대: base 1000 + 어픽스 990 = **~1990**

### 공격력 기반 함수
| 함수 | 기반 장비 | 공식 |
|---|---|---|
| `meleeRef()` | 무기(wp) | `(wp.atk × (1+강화배율) + flatAtk어픽스 + baseAtk) × 봉인` |
| `bowRef()` | 석궁(bw) | `(bw.atk × (1+강화배율) + flatAtk어픽스 + baseAtk) × 봉인` |
| `magicRef()` | 투구(hm) | `(hm.atk × (1+강화배율) + flatAtk어픽스 + baseAtk) × 봉인` |
