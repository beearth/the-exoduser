# 스킬 데미지 공식 총정리

> game.html 2026-04-11 기준 (DPS 균등화 + 50% 곡선 재밸런스)

## 기본 공식 (2026-04-17 변경)

```
데미지 = ref() × stat() × passive() × _skMul('skillId') × _fuseMul('skillId')

_skMul(skId) = base + (skLv-1) × growth × 0.5
  → 스킬레벨 효과 절반 (10단계→20단계 쪼개기)
  → 선형 성장 (더 이상 캐릭터 레벨 복리 보너스 없음)

스킬 성장 내용:
- Lv1: base 값 (기본)
- Lv20: base + 19 × growth × 0.5 = base + 9.5 × growth (선형 약 ×1.5~2배)

나머지 레벨 보상은 장비/강화/세팅으로 충당 (장비 의존 게임)

스킬레벨 캡: Lv20 (_skMaxLv=20, 50렙당 1렙 해금), Lv1→Lv20 = 약 ×1.5~2.0 성장
```

## ⚠ 카테고리별 데미지 공식 규칙 (절대 준수)

**스킬의 cat 값에 따라 아래 공식을 반드시 따른다. 새 스킬 추가·기존 스킬 수정 시 카테고리 공식 누락 금지.**

### 물리 근접 (cat:'phys')
```
meleeRef() × statStr() × pAtkMul() × _skMul(skId) × _fuseMul(skId)
```
- **ref**: meleeRef() — 무기ATK + 강화 + STR
- **stat**: statStr() — STR 배율
- **passive**: pAtkMul() = _passDmgSum('melee') — 맹공+광전사+풍요 합연산 (내HP≥70%)
- **해당 스킬**: kiSlash, whirlwind, giantSlam, giantSlam2, maliceHunt, boneWall

### 물리 이동 (cat:'move', STR 스케일)
```
meleeRef() × statStr() × pAtkMul() × _skMul(skId) × _fuseMul(skId)
```
- **phys와 동일 공식** — 이동기라도 물리 데미지면 pAtkMul 필수
- **해당 스킬**: chainSlam(기동파괴), chainSlash(기동칼날개), chargeBoost(사슬기동:충돌)

### 마법 (cat:'magic')
```
magicRef() × statInt() × pMagicMul() × _skMul(skId) × _fuseMul(skId)
```
- **ref**: magicRef() — 마법ATK + INT
- **stat**: statInt() — INT 배율
- **passive**: pMagicMul() = _passDmgSum('magic') — 마력폭주+풍요 합연산 (내HP≥70%)
- **해당 스킬**: fireball, omniBeam, energyShot, fireAura, maliceMortar, maliceStorm, hellRay, elemMissile

### 마법 이동 (cat:'move', INT 스케일)
```
magicRef() × statInt() × pMagicMul() × _skMul(skId) × _fuseMul(skId)
```
- **magic과 동일 공식**
- **해당 스킬**: chainAssault(기동불꽃), magicBlink(사슬기동:화염)

### 석궁 (cat:'bow')
```
bowRef() × statDex() × pBowMul() × _skMul(skId) × _fuseMul(skId)
```
- **ref**: bowRef() — 석궁ATK + DEX
- **stat**: statDex() — DEX 배율
- **passive**: pBowMul() = _passDmgSum('bow') — 정밀사격 합연산 (+15%/Lv)
- **해당 스킬**: blastShot, fanShot, needleShot, fireBeam, blueShot, iceOrb

### 방어/반사 (cat:'def')
```
별도 공식 (패링 기반, pParryProjDmg 등) — _skMul 미적용 가능
```
- **해당 스킬**: detonate, maliceSwipe, guardian, peaceShield

### 유틸 (cat:'tech')
```
데미지 없음 (noDmg:true) 또는 별도 공식
```
- **해당 스킬**: holyDome, holyPrison, iceStorm, spikeTrap

  meleeRef() = (주무기ATK + enhMulAtk + slotFlat + baseAtk + STR) × 봉인

baseAtk = 15 + (stage-1)×2
```

## 통일 관통 시스템 (`_pierceRate`)

모든 투사체는 `_pierceRate` 베이스 값을 가지고, 어픽스/패시브로 추가됨.

### 공식
```
총 관통률 = 베이스 + bowPierce 어픽스 × 1000 + pPierce 패시브 × 100
보장 관통수 = floor(관통률 × 91 / 1000)  ← 1000=91, 2000=182
보장 초과 시: 1회=90%, 2회=80%, ..., 9회=10% (최대 +9회 추가)
최종 최대 = 보장수 + 9
```

### 스킬별 베이스 관통률

| 스킬 | 베이스 _pierceRate | 보장 (Lv1) | 비고 |
|---|---|---|---|
| **기본값 (모든 투사체)** | **50** | **4회** | `_resetPProj`에서 자동 부여 |
| 푸른비 (blueShot) | 200 | 18 | + 어픽스/패시브 |
| 악의 사냥 (maliceHunt) | 800 + Lv×20 + bowPierce×300 + pPierce×30 | 약 74 + 장비/패시브 가산 | 별도 로직 (`_mhMaxPierce`) |

## ref 함수 구조

```
meleeRef() = (주무기ATK + enhMulAtk(강화) + slotFlatAtk + baseAtk + STR) × 봉인배율
bowRef()   = (석궁ATK   + enhMulAtk(강화) + slotFlatAtk + baseAtk + DEX) × 봉인배율
magicRef() = (마법ATK   + enhMulAtk(강화) + slotFlatAtk + baseAtk + INT) × 봉인배율

아이템 ATK = (200 + tier×200 + P.lv) × **0.5** (2026-04-18 반감)
baseAtk = 15 + (stage-1)×2
enhMulAtk(n) = n × 0.5 (1강당 +0.5, 반감)
봉인배율 = weaponSeal 중이면 ×0.3, 아니면 ×1
몬스터 ATK (v2 — Lv>500 복리 가속, ×0.5 제거 2026-04-17):
  baseLin = (보스?75:50) + monLv×(보스?2.9:1.95)
  curve   = Lv≤500 ? baseLin : baseLin × 1.0018^(Lv-500)
  ATK     = curve × 난이도 × 정예배율
  → Lv500=동일, Lv600=×1.2, Lv750=×1.64, Lv1000=×2.46 (2026-04-17)
```
