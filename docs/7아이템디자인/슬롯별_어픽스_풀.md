| 항목 | 값 |

## 문서 읽는 법

이 문서는 동일한 데이터를 7가지 관점에서 본다:

- **0. 슬롯 역할 매트릭스** — 각 슬롯의 설계 의도와 AFFIX 재배치 계획.
- **A. 역할 인덱스** — 공격/방어/도트/자원/유틸 등 기능 축으로 분류. 빌드 컨셉 구상 단계에서 사용.
- **B. 아키타입 인덱스** — `arcMage*/arcRanger*/arcWarrior*/arcAssassin*/arcTank*` 즉 클래스 전용 어픽스 지도.
- **C. 스킬 강화 지도** — `skXxxDmg` 어픽스가 어느 슬롯에서 드랍 가능한지.
- **D. 엔드게임 어픽스 지도** — `end*` 종결 티어. 라이트·한도 설계 확인용.
- **E. 공통 풀** — 동일 id가 여러 슬롯에 등장하는 어픽스. 스케일 통일 확인.
- **F. brType 태그** — `life` / `demon` 특수 태그 어픽스.
- **G. 슬롯별 전체 풀** — 원본 AFFIX_POOL (드랍 계산 시 기준 표).

## 0. 슬롯 역할 매트릭스

UI 장비창 기준 **15슬롯 = 공격 3 + 악세 6 + 방어 6**. 슬롯의 설계 의도는 세 공격축(근접/마법/원거리)을 공격 슬롯에서 **슬롯 단위로 분리**하여 빌드 정체성을 드러내는 것.

### 공격 3슬롯

| UI | slot_key | 역할 | 핵심 어픽스 축 |
|---|---|---|---|
| 무기 | `wpn` | 근접 데미지 | 근접공격·치명·출혈·관통(근접)·스태거 |
| 왕관 | `helm` | 마법 스킬 데미지 | 시전속도·원소강화·도트지속·마법 스킬 강화 |
| 석궁 | `xbow` | 원거리 투사체 데미지 | 사거리·관통횟수·탄속·원거리 스킬 강화 |

### 악세 6슬롯

| UI | slot_key | 역할 | 핵심 어픽스 축 |
|---|---|---|---|
| 팔찌 | `bracelet` | 스탯/특수 | 힘·민첩·치명·생명(life)·악마(demon) |
| 목걸이 | `neck` | 만능/종결 | 치명·행운·만능·종결 계열 |
| 귀걸이 | `headband` | 마법 보조 | 마나·시전속도·지능·주술띠 |
| 반지1 | `ring` | 스탯/조건부 | 힘·민첩·지능·원소반지·콤보 |

### 0.1 AFFIX_POOL 재배치 이력 (2026-04-17 적용 완료)

`_AFSLOT.bow`를 `'wpn'`→`'xbow'`로 변경하고, AFFIX_POOL의 slots 배열을 아래 계획대로 재배치 완료.

**A. wpn 에서 xbow 로 이전 (24종)**

```
arcRangerRange, arcRangerDmg, arcRangerPierce, bowAtkSpd, bowRange,
critRateBow, pierceChance, pierceCount, projDmgPct, projSpeedPct,
rangedDmgPct, multiShot, splashRadius, splashDmg, farDmg, chainTarget,
pierceDmgKeep, skBlastShotDmg, skBlueShotDmg, skChainAssaultDmg,
skMaliceHuntDmg, skMaliceMortarDmg, skMaliceStormDmg, skMissileDmg
```

**B. helm 에서 제거 후 xbow 로 이전 (6종)**

helm 에 잘못 들어가 있던 원거리 어픽스. xbow 로 옮기고 helm 에서 제거.

```
farDmg, skBlueShotDmg, skChainAssaultDmg, skMaliceMortarDmg,
skMaliceStormDmg, skMissileDmg
```

**C. helm 에서 제거 후 wpn 로 이전 (5종)**

마법 전용 슬롯이 돼야 할 helm 에 남아있던 근접 어픽스.

```
brutalAtk, sharpAtk, closeDmg, ruinAtk, skGhostWalkDmg
```

**D. wpn 의 마법 32종 정리**

이 중 26종은 이미 helm 에도 존재 - wpn 에서 단순 제거. 나머지 6종만 helm 으로 신규 이전.

wpn 에서 제거 (helm 중복, 26종):

```
beamDmgPct, castSpeed, critRateSpell, darkDmgPct, darkDuration, elemFocus,
fireDmgPct, fireDuration, flatMagicDmg, iceDmgPct, iceDuration, lightDmgPct,
lightDuration, magicDmg, poisonDmgPct, poisonDuration, spellDmgPct,
arcMageDmg, arcMageElemDmg, skBeamDmg, skDarkPillarDmg, skFireAuraDmg,
skFireballDmg, skFireBeamDmg, skHellRayDmg, skIceOrbDmg
```

wpn 에서 helm 으로 이전 (helm 에 없는 마법, 6종):

```
darkCurse, frostbiteDmg, igniteDmg, shockDmg, skPlagueDmg, skSpikeAuraDmg
```

## H. 이상탐지 / 의사결정 대기

| 유형 | id | 내역 |
|---|---|---|
| `SCALE` | `antiRevive` | bracelet.SUF T1=0.04 T5=0.2, ring.SUF T1=0.02 T5=0.16, neck.SUF T1=0.04 T5=0.2 |
| `I18N` | `lightRes` | 한글명 '암전수호' — 다른 light* 는 '뇌전XX' 로 통일. 오타 가능성. |

### 해석 가이드
- `SCALE` : 동일 id인데 슬롯마다 T1 값이 다름. 의도적 차별화(예: 반지보다 목걸이가 강함)일 수 있으나, 새 슬롯 추가 시 스케일 합의 필요.
- `NAME`  : 동일 id의 한글명이 슬롯별로 다름. 플레이어에겐 같은 어픽스로 보여야 하므로 통일 권장.
- `I18N`  : i18n/네이밍 규칙 위반. 일괄 교체로 즉시 수정 가능한 잔버그.
