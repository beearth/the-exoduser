# 2-5. 부활 + 에너지 쉴드 시스템

## 부활 시스템

### 보스 (e.ib) — 2단계 부활 시스템

#### 1차: 즉시 부활 (`hurtE` 내부, HP=0 순간 처리)
- **풀피(100%) 즉시 부활**, `e.alive=false` 없이 바로 `return` — 보스 절대 사망 안 함
- **확률 공식**: `max(0, 200% + 스테이지×5% - (deaths-1)×15% - 신성력)`
- **부활력 포인트** (`_revPts`): 초기값 `200 + round(si × 5.882)` (si0=200, si34=400). 부활당 기본 10 + 신성력·존·완벽처치 보너스 소모
- 성공 조건: `_revPts > 0` AND 확률 통과 (둘 다 필요)
- 부활마다: ATK ×1.5(H7:×2), SPD ×1.25, maxPoise ×1.5, 패턴쿨 -30f / **현재 포이즈는 maxPoise의 25%로 세팅** (기존 100% 리셋 → 2026-05-13 수정)
- **재소환 연출 시퀀스** (2026-05-08 개선):
  1. 부활 폭발 (구 위치) — `playVFXAng('boss_revive_boom', oldX, oldY, r×7/64, 3f, ...)` (`assets/vfx/boss/boss_revive_boom.png`, 64×64×24f, 8×3 시트)
  2. 보스 투명화 — `e._spawnT=55` (55f 동안 숨김, 이후 20f 페이드인)
  3. 텔레포트 → 플레이어 뒤로 이동
  4. 부활 폭발 (새 위치) — 동일 스케일 (`boss_revive_boom`, r×7/64, 3f/frame)
  5. 40f 후: `_bossReviveGrandVFX` 발동 (`e._bossReviveVFXPending=40` 카운트다운)
- **`_bossReviveGrandVFX(e, vfxR)`**: 백색 플래시 14f + 3중 충격파 파티클 링 + 블래스트 라이트 3층(r×14/9/5) + 히트스톱 16f + 슬로우모 90f + 진동 30 + 크로마 왜곡 18f
- **부활 시 크기 축소**: `e.r = origR × max(0.40, _revPts/_maxRevPts)`. VFX는 축소 전 크기(`_vfxR1`) 기준. "▼ X%" 텍스트 표시
  - 예시(si0, _maxRevPts=200): 부활 1회(190pt)→95%, 7회(130pt)→65%, 13회(70pt)→40%
- deaths≥2: 전방위 분노 폭발탄. 10%확률 동반부활

#### 2차: 폴백 부활 (`_reviveTimer` 3초 카운트다운, 1차 실패 시)
- 1차 실패(포인트 고갈 or 확률 미달) → `e.alive=false` → 3초 대기 → 재판정
- **확률 공식**: `max(0, 150% + 스테이지×5% - deaths×25% - 신성력)`
- 성공 시 **HP 50%** 로 부활 (eShield 삭제됨, 2026-05-10). void 포탈 + `_spawnT=55` + `_bossReviveVFXPending=40` 동일 적용 (크기 축소 포함)
- 포인트 0이면 확률 무관 자동 실패 → 완전 사망

#### 공통
- 신성력(antiRevive) + 구속의 영역(15%~30%)으로 억제 가능
- **무기 3종 기본 신성력**: antiRevive 1~20% 랜덤 롤 (`0.01+0.19×rand()³`). 3개 풀맥스 합산=60%
- `e._origR`: 최초 부활 시 저장되는 원본 r값 (이후 크기 축소 기준)

### 일반몹 (!e.ib)
- 1회만 부활, **구울로 변환** (본모습 아님)
- 부활력: 기본 50% + 100Lv당 +10% (엘리트2+ 기본 60%)
  - Lv100: 60%, Lv500: 100%, Lv1000: 150%(확정+2회부활 가능성)
- 구울 3종: 소형(r≤10), 중형(11~18), 대형(r≥19)
- **벽 끼임 구출 실패**: 플레이어 위치로 강제 이동 (kb 초기화)

### 신성력 (억제 + 언데드 데미지 보너스)
- 목걸이: T1~T3 10~30%
- 무기: 10~30%
- 반지: 0~20%
- 합산 최대 100%
- 구속의 영역 fireZone: 15%~30% (Lv1=15%, Lv10=30%)
- **언데드 데미지 보너스** (2026-04-11 신규): 신성력 1%당 언데드 적에게 데미지 +1% (`hurtE`에서 `dmg×(1+_eqAffix('antiRevive'))` 적용). 대상 etype: `_UNDEAD_ET`(4,10,12,14,23,40,47,82,83,85) + `e._isGhoul`. 보스 제외

### 악마화 부활 (플레이어) — 🔲 부분 구현

#### 핵심 컨셉
- 레벨이 오를수록 악마의 힘이 깨어남
- 사망 시 "악마화 vs 인간 유지" 선택지 (영구 분기, 1회 선택, 되돌릴 수 없음)

#### 부활 규칙 — 악마화 부활 패시브 (`demonRevive`)
- **방어 탭 패시브 스킬** — 부활 강화 (버튼 없음, 자동 발동)
- 미습득이어도 자연부활력+장비로 부활 가능 (스킬은 강화용)

#### 부활력 공식
`부활력% = min(100, max(0, 자연부활력 + 스킬Lv×2 + 장비보너스 - 인간성Lv×10))`

| 요소 | 부활력 | 비고 |
|---|---|---|
| 자연부활력 | `floor(P.lv×0.03)` | Lv100=3%, Lv500=15%, Lv1000=30% |
| 스킬 (`demonRevive`) | Lv당 부활확률+2%, HP+5%, 쿨-30초 | 부활 강화 스킬 (미습득이어도 부활 가능) |
| 장비 어픽스 | revivePow + _iRevPow | 악마의 팔찌 등 |
| 부활의 영역 | +10% + Lv당 1% | 스킬 영역 안에서 |
| **인간성 (`humanity`)** | **Lv당 부활력 -10** | **부활력 감소 패시브 (트레이드오프)** |

#### 쿨다운
`10분(36000f) - Lv당 30초(1800f). 최소 100초(6000f)`

| Lv | 쿨다운 |
|---|---|
| 1 | 9분 30초 |
| 5 | 7분 30초 |
| 10 | 5분 |
| 17+ | 1분 40초 (최소) |

- HP/MP/ST 풀 회복, 2초 무적
- 습득비: SP 15, 악의 150. 강화비: SP 10/Lv

## 에너지 쉴드 (2026-04-17 레벨 성장 변경, bonusShield 슬롯 분리)
- **Lv1부터 기본 제공** (공식: `100 + INT×5 + bonusShield + shieldFlat어픽스`)
  - **NO Lv×1 직접 보너스** — INT 스탯만으로 증가
  - INT = STATS.int + eqInt + floor(Lv×0.5)로 자동 성장
  - **bonusShield**: 장신구 6슬롯 (belt, necklace, ring1, ring2, bracelet, headband)에서만 제공 = `tier*100 + random(20~100)` [T0: 20~100, T1: 120~200, T2: 220~300, T3: 320~400, T4: 420~500]
  - **shieldFlat 어픽스**: 장신구 슬롯 전용
- **쉴드 = 추가 HP바** — HP와 완전 동일한 방어 계산, 색만 다름
  - **모든 DR(기본DR + barrier 어픽스 + 조건부DR)** 적용 후 쉴드→HP 순서로 분배 (2026-05-13 수정)
  - 쉴드가 먼저 흡수 → 오버플로우만 HP로 (추가 cap 없음)
  - 쉴드 있는 한 HP 절대 안 깎임 (`_shFullBlock`)
- ~~이원 방어 (pPhysPen/pMagPen): 삭제됨~~ — 물리10%/마법100% 비대칭 방어는 pPhysPen/pMagPen 패시브 삭제로 설계 붕괴 → 2026-05-13 단순화

### hurtP 쉴드 처리 구현 (2026-05-13 최종)

#### 흡수 공식
1. DR 적용: 물리→`a=max(1,~~(dmg×(1-pDR)))`, 마법→`a=max(1,~~(dmg×(1-eDR)))`
2. barrier 어픽스: `a=max(0,a-_barV)` (고정 데미지 감소)
3. 조건부 DR 어픽스: healthyDef/injuredDR/dotDR/bossDR/closeDR/farDR (합산 최대 50%)
4. 최소 데미지: `a=max(1,~~(P.mhp×0.005))`
5. **쉴드가 먼저 흡수**: `P.shield -= a`, 오버플로우→HP (`a=overflow`)
6. 쉴드 없으면 `a` 그대로 HP로

- 핵심: 쉴드와 HP는 **완전히 동일한 모든 방어 계산**을 공유 → 쉴드 방어 횟수 = HP 방어 횟수
- 설계 변경 이유: 이전 구현은 barrier/조건부DR이 HP에만 적용되어 쉴드가 HP보다 훨씬 약했음

#### 버그 수정 (2026-05-12 → 2026-05-13)
- 2026-05-12: 쉴드 완전흡수 시 `_minDmg`가 HP를 깎던 버그 → `_shFullBlock` 플래그로 수정
- 2026-05-13: barrier/조건부DR이 쉴드 적용 이후에 계산되어 HP에만 유리하던 버그 → 쉴드 체크를 모든 DR 이후로 이동

### 쉴드 자동 리젠
- 피격 시 3초(180f) 쿨다운 (`shieldRegenT=180`), 쿨다운 소진 후 리젠 시작
- **레벨업 시 쉴드 회복 없음** (HP/ST/MP만 20% 회복)
- 쿨다운 감소 속도: `pShieldCdMul()` = `1 - min(0.3, pMRegen×0.03)` (마력회복 패시브 -3%/lv, 최대 -30%)
- 리젠량/f: `(0.3 + INT×0.003) × pMRegenMul()`
- `pMRegenMul()` = `1 + pMRegen×0.03` (+3%/lv, MP/쉴드 리젠 배율)

## 몬스터 에너지쉴드 (`hurtE` 구현)

### 흡수 공식 (2026-05-13 최종 단순화)
- **쉴드 = 추가 HP바** — 물리/마법 구분 없이 동일 흡수 (비대칭 공식 폐기)
- `_shAbs = min(e.eShield, dmg)` → `e.eShield -= dmg` → `dmg -= _shAbs`
- 쉴드가 완전 흡수 시 `_shBlocked=true` → HP 안 깎임
- 쉴드 일부만 남은 경우 `dmg = overflow`가 HP로 전달됨
- **DOT 제외** — `isDot=true`이면 쉴드 처리 건너뜀
- **eShieldMax>0** 조건 확인 (쉴드 없는 몹은 처리 건너뜀)

### 구현 순서 (hurtE 내)
1. 특수 몬스터 처리 (타락기사 방어, 방패돌격병 등)
2. **에너지쉴드 차감** — `_shBlocked` 플래그 설정 (2026-05-13: 물리/마법 비대칭 → 단순 추가HP바 방식으로 변경)
3. 원소 저항 / M14 보호막 / 수호 오라 / M22 불사
4. 최소 데미지 1 보장 — `_shBlocked=true`이면 건너뜀
5. HP 차감 (`if(dmg>0){ e.hp-=dmg }`)

### 몬스터 eShield 초기화값
- 일반몹: `eShield:_hp, eShieldMax:_hp` (레벨스케일 HP와 동일, 2026-05-13 버그수정)
  - ⚠ 기존 `hp` (= `ETYPE_HP[etype]`, 미스케일 base값)로 초기화하던 버그 수정
  - `hp` = base 100~500, `_hp` = 레벨스케일 수천만 → 기존엔 쉴드가 1방에 깨짐
- 보스: `eShield:baseHp, eShieldMax:baseHp` (baseHp = 실제 HP와 동일, 정상)
