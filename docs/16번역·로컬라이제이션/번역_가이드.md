## 번역 시스템 구조 (현재)

- `game.html` 내부 `const _EN = { ... }` 오브젝트 (line 4528~5282)
- `_T(s)` 함수 (line 5296): `OPT.lang==='en'`일 때만 번역 적용
- 장비명 조합 번역: `_EN_PFX` + `_EN_BASE` (line 5283~5295)
- 펫 대사: `_petSayCD()` → 내부에서 `_T(txt)` 호출 (line 5586)

## 번역 시스템 확장 방향 (제안)

현재 `_EN` 단일 오브젝트 → 다국어 확장 시:
```js
const _LANG = {
  en: { ... },
  zh: { ... },
  ja: { ... },
  es: { ... }
};
function _T(s) {
  if (OPT.lang === 'ko') return s;
  const tbl = _LANG[OPT.lang];
  return (tbl && tbl[s]) || s;
}
```

## 번역 규칙

1. **캐릭터 성격 유지**: 까마귀=냉소적·간결, 고양이=활발·직설적
2. **키 바인딩은 번역하지 않음**: Shift, TAB, Q, E, G, K, L, C, Z, ESC 등 그대로
3. **게임 용어 통일**: Malice(악의), Parry(패링), Groggy(그로기), ST(스태미나) 등
4. **이모지는 그대로 유지**: 💀, ⚡, 🔥 등 번역하지 않음
5. **번호(No.)는 코드 내 고유 키**: 번역 시 한국어 원문 키 변경 금지
