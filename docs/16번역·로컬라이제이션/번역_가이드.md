## 번역 시스템 구조 (구현 완료)

### game.html (인게임)
```
game.html  → const _EN={...}, _EN_PFX, _EN_BASE (인라인)
lang_zh.js → const _ZH={...}, _ZH_PFX, _ZH_BASE
lang_ja.js → const _JA={...}, _JA_PFX, _JA_BASE
lang_es.js → const _ES={...}, _ES_PFX, _ES_BASE
```

- `_T(s)` 함수: `OPT.lang==='ko'`이면 원문 반환, 아니면 해당 언어 테이블 조회
- `_LANG_TBL`, `_LANG_PFX`, `_LANG_BASE` 매핑 오브젝트로 언어별 테이블 자동 선택
- 장비명 조합: 접두사+기본형 분리 번역 (각 언어별 PFX/BASE 테이블)
- 설정 UI 드롭다운: ko/en/zh/ja/es 5개 옵션
- 하드코딩 166곳: `OPT.lang!=='ko'?` 패턴으로 통일 (비한국어→영문 fallback)

### index.html (인트로 시네마틱 + 로비)
```
index.html → _LOBBY_EN, _LOBBY_ZH, _LOBBY_JA, _LOBBY_ES (인라인)
           → CIN_LINES[].en / .zh / .ja / .es (인라인)
```

- `_TL(s)` 함수: `_lobbyLang()==='ko'`이면 원문, 아니면 해당 언어 테이블 → EN fallback → 원문
- `_LOBBY_TABLES = {en, zh, ja, es}` 매핑으로 언어별 테이블 자동 선택
- 인트로 시네마틱: `CIN_LINES` 각 라인에 `zh`, `ja`, `es` 필드 추가 (2026-04-30)
- 드롭다운: `cinLang` + `langSelect` 모두 ko/en/zh/ja/es 5개 옵션
- 로비 UI 번역 항목: ~30개 (메시지, 버튼, placeholder 등)

## 번역 규칙

1. **캐릭터 성격 유지**: 까마귀=냉소적·간결, 고양이=활발·직설적
2. **키 바인딩은 번역하지 않음**: Shift, TAB, Q, E, G, K, L, C, Z, ESC 등 그대로
3. **게임 용어 통일**: Malice(악의), Parry(패링), Groggy(그로기), ST(스태미나) 등
4. **이모지는 그대로 유지**: 💀, ⚡, 🔥 등 번역하지 않음
5. **번호(No.)는 코드 내 고유 키**: 번역 시 한국어 원문 키 변경 금지
