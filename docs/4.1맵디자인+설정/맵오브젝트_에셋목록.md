# 맵 장식 오브젝트 에셋 목록

## 에셋 경로 규칙

### 원칙: 챕터 폴더 = 해당 챕터의 유일한 에셋 소스
- **CH1**: `assets/map/ch1/` — CH1에서 사용하는 모든 오브젝트 에셋은 이 폴더에서 로드
  - `floor_objects/` — 비충돌 장식 에셋
  - `collision/` — 충돌 에셋 (저주나무, 구덩이, 썩은나무, 애니나무, 보스게이트)
- **CH2~CH7**: `assets/map/ch2/` ~ `assets/map/ch7/` (추후 각 챕터별 폴더에 에셋 배치)
- **`assets/objects/`는 창고(아카이브)**. 게임 런타임에서 직접 로드하지 않음. 원본 보관·백업 용도

### 코드 적용
- `_OBJ_SPR_MAP`: 모든 경로가 `assets/map/ch1/...`로 지정됨
- `_CH_DECO`: `assets/map/ch{N}/` + `(dir||'floor_objects')` + `file`로 자동 합성. `dir:'collision'` 등으로 서브폴더 지정 가능
- 벽 나무/눈깔 등 프로시저럴 장식도 `assets/map/ch1/` 경로 사용
- 새 챕터 추가 시: 해당 챕터 폴더에 에셋 넣고 `_OBJ_SPR_MAP` 경로를 챕터별로 분기하거나, `_CH_DECO` 시스템 활용

### 에셋 추가/교체 절차
1. 새 에셋 파일을 해당 챕터 폴더에 저장 (예: `assets/map/ch1/new_object.png`)
2. `_OBJ_SPR_MAP`에 경로 등록 또는 `_CH_DECO` 배열에 추가
3. `_OBJ_META`에 sz/collision/light/anim 등 메타 설정
4. `assets/objects/`에도 백업 복사 (선택)

- 에디터 폴더: `docs/4.1맵디자인+설정/objects/`
- 카테고리: 에디터는 `장식-대형`/`장식-중형`/`장식-소형` 3분류, 자동 배치는 챕터별
- 크기 배율(scale): 에디터에서 0.3~3.0x 슬라이더로 조절 가능, MAP_OBJS에 `scale` 프로퍼티로 저장

### 벽 타일 렌더링 흐름 (tileRLE 맵 = `buildMapCache()`)
1. **해골 패턴 생성**: `_skullWallImg` 로드 → 256×256 축소 캔버스 → `createPattern('repeat')` → `_skullWallPat`
2. **tileRLE 경로**: `G.stage===1`이면 `_skullWallPat`으로 벽(map===1) 전체를 클립+fillRect
3. **경계 에셋**: `G.stage===1`이면 `_wallEdgeImg`로 바닥 인접 벽 타일에 경계 장식 (기둥 대체)
4. **폴백**: 해골 패턴 없으면 `_hasVoidWall` → 검정 순서
5. **프로시저럴 경로(Pass 3)**: tileRLE 맵에서는 실행 안 됨 (return)

### 프리로드
- `_skullWallImg`, `_wallEdgeImg` 모두 `_waitObjSprites()`의 `_extraImgs` 배열에 포함 → 맵 빌드 전 로드 보장
- `_skullWallImg`: `onload` 콜백에서 패턴 생성, 맵 빌드 시점에 미생성이면 즉석 생성 폴백

## 자동 배치 시스템 (`_CH_DECO`)
- `initMapObjects()`에서 해당 챕터 에셋을 바닥 타일에 자동 배치
- 시드 기반 난수로 동일 스테이지 = 동일 배치

### 대형 에셋 자동 배치 (1차 패스)
- `_CH_DECO`에서 `large:1` 플래그가 있는 에셋만 대상
- 맵당 최대 60개 (바닥 80타일당 1개), 사이드(벽 인접) 80% + 중앙 20% 비율
- 벽에서 2~4타일 범위가 사이드 판정, 대형끼리 **500px** 이상 간격
- 충돌 판정: `isW()`에서 `col:1` 또는 `collision:true` 모두 벽으로 처리 (sz×0.4 반경). 코드: `_pm.collision||_pm.col`
- 대형 에셋: m_ctree1~12 (저주나무 12종, sz:400, col:1)

### 일반(소/중형) 에셋 자동 배치 (2차 패스)
- 바닥 40타일당 1개, 최소 12개 최대 100개
- 벽에서 2타일 이상, 기존 오브젝트와 sz 기반 동적 거리 유지 (최소 60px, 대형 에셋은 (sz+32)/2, sz≥150 scatter 에셋은 500px)
- 시작방/보스방/대장간 근처 제외
- CH1 벽 눈깔: 바닥 인접 벽 타일에 2% 확률로 eyeball/eye1_bloodshot/eye2_terror 배치 (sz:72)
- 자동 배치 에셋 크기: 나무 100, 뼈 48, 꽃/벌레 40, 눈깔/유충 56
- 추가 에셋 (PixelLab): flesh_pile 64, spider_web 56, tombstone 56, weapon_pile 64, meat_stake 64, acid_pool 56

## 보스 게이트 시스템 (7장 공통)

### 구조 개요
```
[게이트 이미지] — 보스방 경계에 정적 렌더 (나무/문 등 대형 에셋)
      │
      ▼ (gateY+7 = mouth 위치)
[포탈 애니메이션] — 투명 webp 회전+맥동, 출구 타일 위에 렌더
      │
      ▼ (플레이어가 출구 타일 밟으면)
[보스 아레나 진입] — 별도 맵 생성, 아레나 6시에도 포탈 렌더
```
챕터별 **게이트 에셋**(나무/문) + **포탈 이미지**(투명 webp) 2종 세트.
배열 인덱스 = hell(챕터) 번호. 에셋 경로만 배열에 추가하면 자동 적용.

### 2. 게이트 배치 로직 (맵 생성 시)

#### 게이트 방향 결정
```
genFromTemplate: 보스방 4방향 스캔 → 복도 연결이 가장 많은 방향 = 입구
  → _entryDir = 'top'|'bot'|'left'|'right'
  → 해당 방향 경계에 게이트 타일 3칸 배치 (_gateTiles)
  → _gateY = 게이트 타일 Y좌표
```

#### 출구 타일 배치
```
위치: gateY + 7 (= 게이트 이미지 하단 blood trail/mouth 위치)
폭: 3칸 (게이트 타일과 동일 X좌표)
타일값: map[ey][ex] = 2 (출구 타일)
바닥 확보: gateY ~ gateY+8까지 벽(1) → 바닥(0) 자동 카빙
```
- genFromTemplate: `_mouthOff=7`, 게이트 타일별 X좌표 유지, Y = gateY+7
- genGauntlet: `_mouthOff=7`, 3칸(iCx-1, iCx, iCx+1), Y = gateY+7

#### 충돌 오브젝트
```
type: 'boss_gate_col'
위치: (게이트 중심 X, (gateY+2)*T)  — 나무 몸통 중심
크기: sz=500, 충돌반경 = 500×0.4 = 200px
효과: 나무 본체 통과 차단, mouth(gateY+7, 200px 밖)은 접근 가능
```

#### 게이트 이미지 렌더 (맵 캐시 — buildMapCache)

**3곳에서 렌더됨** (스트리밍/풀빌드/아레나):

```
[1] 스트리밍 맵 캐시 (변경 없이 맵 유지 시)
    조건: G._bossArena → 아레나 12시 출구에 게이트 이미지
          else G.bossGate → 던전 게이트에 이미지
    크기: 600px 폭, 비율 유지 높이
    오프셋: gateY*T - bgH + 350 (하단이 gateY*T+350)

[2] 풀 맵 캐시 (buildMapCache 풀 리빌드)
    ── Pass 3.5: 보스방 대문 (tile=3) ──
    - tile=3 타일 찾기 → 게이트 영역 계산
    - _bossGateImgs[hell] 있으면: 챕터별 이미지 렌더
    - 없으면: 프로시저럴 지옥문 (철문+리벳+균열+보라빛)

    ── Pass 3.7: 보스문 앞 장식 ──
    - 범위: bossCx 중심, gateY+1 ~ gateY+9 (5×8 타일)
    - 1) 바닥 핏자국/균열 (40% 확률 타원, 15% 룬)
    - 2) 양쪽 악마 동상 (삼각형 실루엣 + 뿔 + 눈)
    - 3) 보스문 앞 경고 룬 원 (큰 마법진)

[3] 아레나 맵 캐시
    조건: G._bossArena && G._bossArenaGate
    위치: 아레나 12시 최상단 (exitX, exitY)
    오프셋: gateY*T - bgH + 40
```

**게이트 이미지 크기/위치 공식:**
```javascript
const _bgW = 600;
const _bgH = ~~(600 * img.naturalHeight / img.naturalWidth);
// 렌더 좌표 (던전):
x = gateCenterX - _bgW/2
y = gateY*T - _bgH + 350
// 이미지 상단: gateY*T - bgH + 350
// 이미지 하단: gateY*T + 350
// mouth(입) 위치: 대략 gateY*T + 280 (= gateY+7 타일)
```

### 3. 포탈 렌더 로직

#### _drawPortal(ctx, cx, cy, sz, t, hell)
```
인자:
  ctx  — 캔버스 2D 컨텍스트 (X)
  cx,cy — 포탈 중심 월드좌표
  sz   — 기본 크기 (px)
  t    — 현재 시간 (_now, ms)
  hell — 챕터 인덱스 (SI_TO_HELL[G.stage])

렌더 레이어:
  1) 메인 소용돌이: _portalImgs[hell], 느린 회전(t/2000), 알파 0.9
     크기: sz ± 12% 사인파 맥동
  2) 내부 코어: 같은 이미지, 역회전(t/800), 55% 크기, 알파 0.45
```

#### 던전 포탈 (출구 타일 루프 뒤)
```
조건: !G._bossArena && G.exits.length>0 && G.bossGate && G.bossAlive
위치: 출구 타일 중심 좌표 (자동 계산)
크기: sz=250
```

#### 보스 아레나 포탈 (맵 캐시 직후)
```
조건: G._bossArena && G._bossArenaEntry
위치: (entryX+0.5)*T, (entryY+2.5)*T  — 6시 입구
크기: sz=400
```

#### 출구 타일 바닥 처리
```
포탈 이미지 로드 완료 시: 출구 타일 바닥색 fill 생략 (포탈이 덮음)
포탈 미로드 시: 기존 바닥색으로 fill
```

### 4. 진입/리스폰 로직

#### 보스방 진입
```
플레이어 타일 == 출구 타일 → G._bossLoadPhase=1 (로딩 시작)
  → _enterBossArena() 호출
  → _preArenaBackup에 현재 맵/출구/게이트/소환굴 백업
  → genBossArena(si) 새 아레나 생성
  → 플레이어를 아레나 entryY에 배치
```

#### 보스방 사망 → 리스폰
```
G._bossArena && _preArenaBackup 존재 시:
  1) MAP_OBJS=[] 초기화
  2) 백업에서 맵/출구/게이트 복원
  3) boss_gate_col 충돌 오브젝트 재생성 (MAP_OBJS.push)
  4) 리스폰 위치: (bossCx, gateY+8) — mouth 바로 아래
  5) buildMapCache + initMapObjects 재호출
  6) G.bossAlive=true, G._bossArena=false
```

#### 진입 텍스트
```
"▼ 보스방 ▼" — 출구 가운데 타일(ei===exits.length/2)에서만 표시
플레이어 5타일 이내 접근 시 알파 페이드인
```

### 5. 전체 파이프라인 순서 (한 프레임 기준)

```
맵 생성 (initStage)
 ├── genFromTemplate / genGauntlet
 │    ├── 보스방 생성 + 격리 (tile=4)
 │    ├── 게이트 방향 결정 (4방향 스캔)
 │    ├── 게이트 타일 3칸 배치 (_gateTiles, _gateY)
 │    ├── 출구 타일 3칸 배치 (gateY+7, tile=2)
 │    ├── 바닥 카빙 (gateY ~ gateY+8)
 │    └── boss_gate_col 충돌 오브젝트 생성
 └── buildMapCache
      ├── 게이트 이미지 렌더 (스트리밍/풀빌드)
      ├── Pass 3.5: tile=3 대문 렌더
      └── Pass 3.7: 장식 (핏자국, 동상, 룬)

매 프레임 렌더 (메인 루프)
 ├── 맵 캐시 청크 그리기
 ├── [아레나] _drawPortal() — 6시 입구, sz=400
 ├── 출구 타일 루프
 │    ├── 포탈 로드 시: 바닥 fill 생략
 │    ├── 보스 생존: 진입 텍스트 표시
 │    └── 보스 사망: 초록 맥동 출구
 ├── [던전] _drawPortal() — 출구 중심, sz=250
 └── MAP_OBJS 렌더 (boss_gate_col은 스프라이트 없어서 무시)

보스방 진입
 ├── 출구 타일 밟음 → _bossLoadPhase=1
 ├── _preArenaBackup 백업
 ├── genBossArena(si) → 새 아레나 맵
 └── 플레이어 entryY 배치

보스방 사망 → 리스폰
 ├── MAP_OBJS=[] 초기화
 ├── 백업 복원 (맵/출구/게이트)
 ├── boss_gate_col 재생성
 ├── 리스폰: gateY+8 (mouth 아래)
 └── buildMapCache + initMapObjects
```
