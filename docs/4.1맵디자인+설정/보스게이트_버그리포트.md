# 보스 게이트 나무+포탈 배치 버그 리포트 (2026-04-03)

## 증상
1. 보스 게이트 나무 이미지가 맵에 안 보임
2. 포탈과 나무가 따로 놀고, exit 타일이 엉뚱한 위치에 생성
3. 투명 충돌체(boss_gate_col sz:250)가 플레이어 이동 차단
4. 포탈은 보이는데 진입 불가 (exit 타일과 포탈 위치 불일치)

## 근본 원인

### 1. 맵 캐시 Phase 2에서 나무가 안 그려짐
- tileRLE 맵(세이브 로드)에서는 `_bmcPhase`가 스트리밍 청크 경로로 빠짐
- Phase 2 블록(`_bmcPhase===2`)이 실행되지 않아 나무 drawImage가 호출 안 됨
- 스트리밍 청크(`_buildStreamChunk`)에서도 나무를 그리지만, 600px 나무가 청크 경계에 걸치면 잘리거나 좌표 변환 오류로 안 보임
- **해결**: 맵 캐시가 아닌 실시간 렌더(draw 함수)에서 매 프레임 직접 그리도록 이전

### 2. exit 타일/충돌체/렌더 위치 분리
- 원본: exit=`_gateY+7`, 충돌=`_gateY+2`, 렌더=`_gateY` 기준 → 모두 보스방 하단(6시)
- 보스방 상단(12시)으로 옮기려 했으나, 각각 다른 좌표계 사용:
  - exit: `_iCx`/`_gateY` 기준 (맵 생성 함수 내 로컬 변수)
  - 충돌: `_gateTiles` 중앙 X / `_gateY` 기준
  - 렌더: `G._bossCx` / `G._gateY` 기준 (전역)
  - `_iCx`와 `G._bossCx`가 다를 수 있음
- **해결**: 전부 `bossCx`/`_innerT+1` 기준으로 통일

### 3. boss_gate_col 투명 충돌체
- sz:500 (500px = ~10타일)짜리 거대 충돌 오브젝트
- 나무 위치를 옮겨도 충돌체가 이전 위치에 남아서 이동 차단
- 세이브 로드 시 충돌체 재생성 코드도 별도로 있어서 3곳 동기화 필요
- **해결**: 충돌체를 실시간 렌더 나무 중심 `(bossCx+0.5, gateY-1)*T`로 3곳 통일 (genFromTemplate, genGauntlet, 리스폰)

### 4. PNG 배경 오해
- boss_gate.png는 이미 57% 투명 (알파 채널 있음)
- 이미지 뷰어에서 투명=흰색 표시 → 흰색 배경으로 오인
- `_removeWhiteBg` 함수로 코드 내 투명 처리 시도 → 나무 본체까지 투명화
- multiply 블렌딩 시도 → 검은 배경에서 나무 완전 안 보임
- **해결**: PNG 원본 유지, 코드 처리 함수 삭제

## 최종 구조 (수정 후)

```
[보스방 하단 _innerB = _gateY]
  ├─ exit 타일 (tile=2) : gateY+2 (mouth), bossCx ← 진입 트리거
  ├─ boss_gate_col 충돌  : (bossCx+0.5, gateY-1)*T ← 나무 중심 봉인
  ├─ 나무 렌더 (실시간)  : (gateY+0.5)*T 중심, _bossCx ← 매 프레임
  └─ 포탈 렌더 (실시간)  : exit 타일 중심 (gateY+2+0.5)*T
[보스방 중앙 bossCy]
  └─ 보스 스폰
[메인 던전]
```
