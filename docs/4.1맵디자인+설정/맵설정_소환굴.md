# 맵 설정 — 소환굴 시스템

## 소환굴 종류 (S/M/L)

| 종류 | 에디터 오브젝트 | size값 | 스폰 수 | 게임 비주얼 크기 | 에디터 반지름 |
|---|---|---|---|---|---|
| **S(소)** | hole_s | 0.3 | 50마리 | 420px (10.5타일) | 5.25T |
| **M(중)** | hole_m | 0.5 | 100마리 | 500px (12.5타일) | 6.25T |
| **L(대)** | hole_l | 0.8 | 200마리 | 620px (15.5타일) | 7.75T |

## 트리거 동작

- 소환굴이 **화면에 보이면** 트리거 (뷰포트 + 50px 여유, 2026-04-14 거리→뷰포트 변경)
- 1.5초 연출 (소환굴 빛남 + 스케일업)
- 몹 스폰 반경: **120 + rand(0~300) + size×150 px** (최소 120px, 코앞 스폰 방지 — 2026-04-14)
- 트리거된 소환굴은 **완료까지 유지** (2500px 밖이면 스폰 지연, 중단 안 함 — 2026-04-14)
- 몹 스폰 → 소환굴 소멸
- 안전구역: 시작점 200px 내 소환굴 자동 제거

## 코드 정의 (game.html)

```js
const SPAWN_HOLE = {
  name: '소환굴',
  col: '#aa4444',
  cnt: { S: 50, M: 100, L: 200 }
};
```

### size → 타입 매핑
- h.size ≤ 0.3 → S
- h.size ≤ 0.5 → M
- h.size > 0.5 → L

## 배치 방법

### 수동 배치 (에디터)
1. 에디터에서 hole_s / hole_m / hole_l 오브젝트 선택
2. 맵에 클릭하여 배치 (빨간 원으로 실제 크기 미리보기)
3. 내보내기 → FIXED_MAPS에 `spawnHoles` 배열로 포함

### 자동 배치 (폴백)
에디터 spawnHoles가 없으면 방 크기 기반 자동 배치:

| 방 면적 | 소환굴 개수 | size |
|---|---|---|
| ≤150 타일² (≈10×10) | 5개 | 0.3 (S) |
| ≤500 타일² (≈20×20) | 10개 | 0.5 (M) |
| >500 타일² (≈30×30) | 30개 | 0.8 (L) |

골든 앵글 분산 배치 (균등 간격)

## 에디터 → 게임 파이프라인

1. **에디터** (`g:\hell\docs\4.1맵디자인+설정\tilemap-editor.html`)
   - 타일 페인팅 (바닥/벽)
   - 오브젝트 배치 (시작점, 보스, 소환굴)
   - 📐 맵 크기 조정 (20~2000 타일)

2. **내보내기** (📤 버튼)
   - tileRLE: 에디터 타일 → 게임 맵 (바닥=0, 벽=1)
   - spawnHoles: hole_s/m/l → [{x, y, type:'S'|'M'|'L'}]
   - rooms: player_start → start 방 좌표 동기화, boss_pos → boss 방 생성

3. **적용** (node 스크립트)
   - 내보내기 파일의 FIXED_MAPS → game.html 교체
   - genFromTemplate: tileRLE 있으면 타일 직접 로드 (카빙/스무딩 건너뜀)

## 주의사항

- 소환굴은 일회성 (트리거 후 소멸, 주기적 리스폰 없음)
- 게임 T=40px, 에디터 T=8px (비율 5:1)
- 보스방(type:'boss') 없으면 보스 게이트/출구 생성 안 됨
