# 2-1. 스킬 관리 + 합체 시스템

## 두 개의 스킬 창

| 키 | 창 이름 | 역할 | 함수 |
|---|---|---|---|
| **K** | 전투스킬 관리창 | 스킬 습득/강화/합체/리셋 | `renderSkillPanel()` → `_renderSkillRow()` |
| **L** | 스킬 선택 관리창 | 슬롯별 활성 스킬 선택 (좌클/우클/SH/E/T/Q/CT 탭) | `openSkSlotPop()` |

### K ↔ L 동기화 흐름

```
배운다(K) → L에 등록
    ↓
합체한다(K) → L에서 흡수스킬 사라짐, 합체명 표시
    ↓
해제한다(K) → 안배운 상태로 복귀 → L에서 전부 사라짐
    ↓
다시 배운다(K) → L에 다시 등록
    ↓
다시 합체(K) → L에서 다시 사라짐
```

### 동기화 함수

| 시점 | 호출 함수 |
|---|---|
| 습득/강화 후 | `renderSkillPanel()` + `updateSkSlot()` + `updateQS()` + `dbSaveForce()` |
| 합체 후 | `_syncActiveSkAfterReset(absorbed)` + `updateSkSlot()` + `updateQS()` + `renderSkillPanel()` |
| 리셋 후 | `_syncActiveSkAfterReset(resetIds)` + `updateSkSlot()` + `updateQS()` + `renderSkillPanel()` |
| 세이브 로드 | `_syncActiveSkAfterReset(_getAllAbsorbed())` + 슬롯 유효성 검사 |

### `_syncActiveSkAfterReset(ids)` — 활성 스킬 변수 폴백

리셋/합체/로드 시 안배운/흡수된 스킬이 활성 변수에 남지 않도록 기본값으로 복귀:

| 변수 | 폴백 기본값 |
|---|---|
| `P.activeLMBSk` | `'kiSlash'` |
| `P.activeChargeSk` | `'charge'` (작살) |
| `P.activeMagicSk` | `'fireball'` (악의구) |
| `P.activeQSk` | `'parry'` (보호막) |
| `P.activeCtSk` | `'ghostWalk'` (뇌전걸음) |
| `P.activeBowSk` | `'normal'` (일반사격) |

## 시작 스킬 (캐릭터 생성 시 자동 습득)

| id | 이름 | 슬롯 | 키 | 난이도 제한 |
|---|---|---|---|---|
| kiSlash | 기검참 | LMB | 좌클 | 항상 |
| fireball | 악의구 | E(마법) | E | 항상 |
| iceOrb | 얼음보주 | CT슬롯 | Ctrl | 항상 |
| giantSlam | 대왕치기 | SKILL_SLOTS[4] | Space | 난이도 0만 |
| holyDome | 신성한 땅 | SKILL_SLOTS[5] | F | 난이도 0~1 |
| spikeTrap | 가시덫 | SKILL_SLOTS[0] | 1번 | 난이도 0~3 |

- `mkP()` 기본값: `skills:{kiSlash:1}` (나머지는 강제 장착으로 부여)
- 기검참 + 얼음보주는 난이도 무관 항상 지급
- 난이도 선택지: 0(일반) / 1단 / 3단 / 5단 / 10단
- 난이도 0: 5종 전부 / 1: -대왕치기(4종) / 3: -신성한땅(3종) / 5: -가시덫(2종) / 10: 기검참+얼음보주만
- 세이브 로드 시 마이그레이션: 기존 캐릭은 스킬 유지 (없으면 자동 부여)
- 고정슬롯 강제 배정(Space=대왕치기, F=신성한땅, 1번=가시덫)은 **캐릭터 생성(1렙) 시에만** 실행. 이후 로드에서는 세이브된 슬롯 배치를 그대로 복원

### 처형 (execution) 상세

- **id**: `execution`
- **카테고리**: ult (필살기)
- **키**: X키 (`_chkJust('KeyX')`)
- **발동 조건**: `P.skills.execution >= 1` + `P._execCd <= 0` + 그로기 보스 존재 (`G._bossRef.stunned > 0`)
- **비용**: HP/MP/ST 각 최대치의 10%
- **효과**: 보스 최대HP 30% 순수 데미지 (방어 무시)
- **쿨다운**: 20초 (1200프레임), `P._execCd`로 관리
- **연출**: 섬광 관통 돌진 (보스 관통→뒤쪽 벽까지), 궤적 잔상, 임팩트 파티클 30개
- **습득**: 스킬 패널(K)에서 SP 10 + 소재 100으로 배워야 사용 가능
- **강화**: 불가 (`up:false`)

#### magicBlink 이동 효과 — dimBreach 이상 합체에서도 유지
사슬기동(magicBlink) 이동 중 **3프레임마다 화염길 DOT 존(`fireTrail`)** 생성:
- **발동 조건**: `_harpSk==='magicBlink'` 또는 `_harpSk==='dimBreach'` (합체 후에도 유지)
- **범위**: 40+Lv×3 (magicBlink Lv 기준)
- **데미지**: `magicRef()×0.5×(1+(Lv-1)×0.1)×INT×pMagicMul()` (화염 속성)
- **지속시간**: 720프레임 (12초)
- **DOT 간격**: 15프레임 (0.25초)
- **추가 효과**: 화상(3초) + 감속(20프레임)
- **적 접촉 히트**: 이동 중 적과 접촉 시 화염 데미지 + 화상 3초 (dimBreach 합체에서도 INT 경로 발동, `EL.F`)
- **⚠ 중요**: dimBreach 합체 시 `_harpSk`가 `'dimBreach'`로 세팅되므로, 반드시 `'dimBreach'` 조건도 포함해야 합니다

#### 기동칼날개 (chainSlash) 상세 — [히든: 5성 dimRush 합체 전용]
사슬기동 시 **자동 발동** — 이동하면서 경로상 광역 베기 (착지 스킬이 아님).
- **해금**: dimRush(기동:칼날개) 5성 합체 + **Lv200** 필요. SP/악의 비용 없음
- **발동**: 사슬기동 이동 시작 시 자동 활성화 (U키 AUTO 토글). 좌클릭 수동 발동도 가능
- **비용**: ST 15+Lv×2
- **히트 범위**: 반경 160+Lv×5 (이동 중 전방위)
- **데미지**: meleeRef×STR×(2.0+(Lv-1)×0.12), 무기 속성
- **다단히트**: 4프레임마다 히트 리셋 → 같은 적 재히트 가능
- **포이즈**: 1.2배, 넉백 없음
- **출혈**: 히트 시 출혈 2초 부여
- **키**: 사슬기동 중 좌클릭 (비행+이동 모두 가능) / U키로 AUTO 토글 시 사슬기동 시작과 동시에 자동 발동

#### 기동:강화전폭 (dimBlast) 상세
사슬기동 중 **Shift+좌클릭** → 착지 시 전방 파워웨이브 + 착지 AOE.
- **발동**: 사슬기동(작살/대시) 중 Shift+좌클릭으로 준비 → 착지 시 발동
- **비용**: ST 30+Lv×3, 악의 20
- **착지 AOE**: 범위 200+Lv×15, baseDmg×0.5, 일반몹 기절(3초), 보스 체간 삭감
- **파워웨이브**: 전방 직선 투사체 (폭 120+Lv×8, 사거리 300+Lv×20)
  - 데미지: meleeRef×8×(1+(Lv-1)×0.15)×STR×이동거리보너스(최대×6)
  - 관통, 넉백 30+slv×3, 일반몹 그로기, 보스 체간 30%

## 합체 흡수 규칙

합체 시 하위 스킬은 **흡수**되어 스킬창에서 완전 제거 + 독립 발동 불가.
호스트 스킬만 남아 합체 이름/설명으로 표시됨.

### 호스트 → 흡수 테이블

| 합체 | 호스트 (남는 스킬) | 흡수 (사라지는 스킬) |
|---|---|---|
| dimBreach (사슬이동:강화) | chargeBoost | magicBlink |
| dimSlam (사슬이동:강화기습) | chargeBoost | magicBlink, chainAssault |
| dimThunder (사슬기동:기습충돌) | chargeBoost | magicBlink, chainAssault, chainSlam |
| dimRush (기동:칼날개) | chargeBoost | magicBlink, chainAssault, chainSlash, chainSlam |
| dimBlast (기동:강화전폭) | chargeBoost | magicBlink, chainAssault, chainSlash, chainSlam, giantSlam |
| whirlDet (회전기폭) | whirlwind | detonate |
| slamStorm (지진폭풍) | whirlwind | detonate, giantSlam |

### 호스트 승격 체인

상위 합체가 활성되면 하위 합체의 흡수 스킬이 호스트로 승격될 수 있음:
- `sixFuse` → fanShot 호스트, omniBeam 흡수
- `elemFuse` → **omniBeam 호스트 승격**, fanShot 흡수 (sixFuse 흡수 오버라이드)
- `stormBeam` → whirlwind 호스트, omniBeam 다시 흡수

## 스킬 슬롯 전환 규칙 (L키 팝업)

### 핵심 원칙
1. **배운 스킬만 표시** — `P.skills[id] >= 1`인 것만 팝업에 노출
2. **합체 시**: 흡수+호스트 스킬은 목록에서 숨기고, **합체 엔트리**(합체명)로 대체 표시
3. **합체 엔트리 클릭** → 해당 슬롯의 합체 호스트 스킬로 세팅 (아래 표 참조)
4. **비합체 스킬 클릭** → 해당 스킬로 세팅 (합체 엔트리와 자유 전환 가능)

### E슬롯 발동 라우팅 (`_execMagicE`)

E키 누르면 `P.activeMagicSk` 값에 따라 분기:

| 우선순위 | 조건 | 발동 |
|---|---|---|
| 1 | `_amsk === 'blueShot'` | 푸른비 발사 |
| 3 | `_amsk === 'elemMissile'` (단독, 합체 아닐 때) | 원소추적탄 단독 발사 |
| 4 | `_amsk !== 'hellRay'` + 빔합체 활성 (sixFuse/elemFuse) + stormBeam 아님 | 만화광선/추적뇌전 빔 모드 |
| 5 | 그 외 (beam, hellRay 등) | 기본 파이어볼 or hellRay(stormBeam 시 별도 경로) |

**핵심**: `hellRay`/`blueShot` 선택 시 빔합체 경로를 **스킵** → 선택한 스킬이 정확히 발동됨.
hellRay는 stormBeam 합체 전용 — stormBeam 없으면 기본 파이어볼 폴백.

## 레이저(빔) 스킬 MP 소비량

회전참(ST)과 동일 구조: `최대MP × 틱% × 레벨배율 × pMagicCost()`

| 스킬 | 틱% | 레벨배율 공식 | 1렙 실소비 | 10렙 실소비 | 비고 |
|---|---|---|---|---|---|
| 멸살광선 (단일) | 0.0015 | 0.5+(lv-1)×0.078 | mmp×0.075%/f | mmp×0.18%/f | 회전참과 동일 |
| 뇌전 (sixFuse) | 0.002 | 0.5+(lv-1)×0.078 | mmp×0.1%/f | mmp×0.24%/f | 6가닥이라 33% 증가 |
| 추적뇌전 (elemFuse) | 0.002 | ×0.9 적용 | mmp×0.09%/f | mmp×0.22%/f | 유도라 약간 할인 |
| 뇌전나선 (stormBeam) | 0.001 | 없음 (고정) | mmp×0.1%/f | mmp×0.1%/f | 회전참이 메인이므로 66% 할인 |

**참고**: 회전참 ST 소비 = `mst × 0.0015 × (0.5+(lv-1)×0.078)`

### 빔 관통

빔은 투사체 관통 패시브(`pPierce`) + 장비 `bowPierce`에 의해 관통 확률이 결정됨.
- 패시브/장비 없으면 **0% 관통** → 첫 적에서 끊김
- 패시브 1렙 = 5%, 10렙 = 50%
- 캡 95%

## 마력폭발 (magicBurst)

- **id**: `magicBurst`
- **카테고리**: magic (마법)
- **키**: E 홀드
- **스케일링**: INT (`statInt()` × `pMagicMul()` × `DPS_BAL.magic`)
- **MP 소모**: `mpCost('magicBurst')` — base 30, Lv 스케일

### 차징 메커니즘
| 프레임 | 단계 | 폭발 반경 | 데미지 배율 |
|---|---|---|---|
| 10f | 1단 | 300 × (1+(Lv-1)×0.05) | ×1.5 × (1+(Lv-1)×0.12) |
| 20f | 2단 | 500 × (1+(Lv-1)×0.05) | ×2.5 × (1+(Lv-1)×0.12) |
| 30f | 3단 | 700 × (1+(Lv-1)×0.05) | ×4.0 × (1+(Lv-1)×0.12) |

- 3단계 도달 시 **자동 폭발**
- 홀드 유지 시 MP 있으면 **반복 차징+폭발** (30f마다)
- 릴리즈 시 현재 단계로 즉시 폭발 (0단계면 불발)
- 넉백 배율: 0.3 (약한 넉백)
- 폭발 VFX: 충격파 링 (`G._gSlamWave`) + 보라 파티클

### 데미지 공식
```
baseDmg = magicRef() × statInt() × pMagicMul() × DPS_BAL.magic × stageMul × lvMul
stageMul = [1.5, 2.5, 4.0][stage]
lvMul = 1 + (Lv-1) × 0.12
```

## 성급 규칙

- **스킬 고유 성급** = 처음 등장하는 최소 합체 그룹의 스킬 수 - 1 (`_skInnateStar`)
  - 기본 스킬(chargeBoost, magicBlink 등 2스킬 그룹 소속) = 1★
  - chainAssault(dimSlam 3스킬에서 추가) = 2★
  - chainSlam(dimThunder 4스킬에서 추가) = 3★
  - chainSlash(dimRush 5스킬에서 추가) = 4★
  - 합체 가능한 모든 스킬에 별(★) 표시
- **합체 성급** = 합체 그룹의 구성 스킬 수 (`_fuseStar`)
  - dimBreach=2★, dimSlam=3★, dimThunder=4★, dimRush=5★, dimBlast=6★
  - 합체 후 카드에는 max(고유성급, 합체성급) 표시 → 합체하면 성급 상승

## 합체 비용 공식

- **합체 수행**: 성급 × 3 SP
- **합체 강화**: `_fuseUpSpCost(curLv, gemCnt)` — 기본테이블 + (성급-1)
- **스킬 습득**: 모든 스킬 10 SP 통일

### 합체 리셋 환불 공식

합체 해제 시 투입한 SP를 정확히 복구:

| 항목 | 환불액 | 비고 |
|---|---|---|
| 구성 스킬 습득비 | Σ spCost (스킬별) | 스킬 개수만큼 |
| 합체 생성비 | 성급 × 3 | 합체 수행 시 지불한 비용 |
| 합체 강화비 | Σ `_fuseUpSpCost(lv, gemCnt)` (lv=1..fuseLv-1) | 합체 후 강화한 레벨만큼 |
| **악의(mats)** | **환불 안 됨** | 일반 스킬 리셋과 동일 |

- 일반 스킬 리셋: `_skillSpentSp(sk, slv)` = 습득비 + 개별강화비 합산 (악의 미환불)

- **적용 방식**: `G._fireZones`에 등록, 6프레임마다 `shQuery`로 범위 내 적 탐색

- **적용 방식**: 플레이어 사망 시 `_revPowBonus`에 영역 보너스 가산

- **체크 시점**: `_onPlayerFallen()`에서 `dst(P, zone) < zone.r`로 범위 판정
- 적 디버프가 아닌 **플레이어 버프** — fireZone 틱에서 파티클만 처리
