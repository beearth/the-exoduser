# 2-1. 스킬 관리 + 합체 시스템

## 두 개의 스킬 창

| 키 | 창 이름 | 역할 | 함수 |
|---|---|---|---|
| **K** | 전투스킬 관리창 | 스킬 습득/강화/합체/리셋 | `renderSkillPanel()` → `_renderSkillRow()` |
| **L** | 스킬 선택 관리창 | 슬롯별 활성 스킬 선택 (좌클/우클/SH/E/T/Q/CT 탭) | `openSkSlotPop()` |

### K ↔ L 동기화 흐름

```
배운다(K) → L에 등록
    ↓
합체한다(K) → L에서 흡수스킬 사라짐, 합체명 표시
    ↓
해제한다(K) → 안배운 상태로 복귀 → L에서 전부 사라짐
    ↓
다시 배운다(K) → L에 다시 등록
    ↓
다시 합체(K) → L에서 다시 사라짐
```

### 동기화 함수

| 시점 | 호출 함수 |
|---|---|
| 습득/강화 후 | `renderSkillPanel()` + `updateSkSlot()` + `updateQS()` + `dbSaveForce()` |
| 합체 후 | `_syncActiveSkAfterReset(absorbed)` + `updateSkSlot()` + `updateQS()` + `renderSkillPanel()` |
| 리셋 후 | `_syncActiveSkAfterReset(resetIds)` + `updateSkSlot()` + `updateQS()` + `renderSkillPanel()` |
| 세이브 로드 | `_syncActiveSkAfterReset(_getAllAbsorbed())` + 슬롯 유효성 검사 |

### `_syncActiveSkAfterReset(ids)` — 활성 스킬 변수 폴백

리셋/합체/로드 시 안배운/흡수된 스킬이 활성 변수에 남지 않도록 기본값으로 복귀:

| 변수 | 폴백 기본값 |
|---|---|
| `P.activeLMBSk` | `'kiSlash'` |
| `P.activeChargeSk` | `'charge'` (작살) |
| `P.activeMagicSk` | `'beam'` (마법탄) |
| `P.activeQSk` | `'parry'` (보호막) |
| `P.activeCtSk` | `'ghostWalk'` (뇌전걸음) |
| `P.activeBowSk` | `'normal'` (일반사격) |

### 처형 (execution) 상세

- **id**: `execution`
- **카테고리**: ult (필살기)

- **발동 조건**: `P.skills.execution >= 1` + `P._execCd <= 0` + 그로기 보스 존재 (`G._bossRef.stunned > 0`)
- **비용**: HP/MP/ST 각 최대치의 10%

- **습득**: 스킬 패널(K)에서 SP 10 + 소재 100으로 배워야 사용 가능
- **강화**: 불가 (`up:false`)

## 합체 흡수 규칙

합체 시 하위 스킬은 **흡수**되어 스킬창에서 완전 제거 + 독립 발동 불가.
호스트 스킬만 남아 합체 이름/설명으로 표시됨.

### 호스트 → 흡수 테이블

| 합체 | 호스트 (남는 스킬) | 흡수 (사라지는 스킬) |
|---|---|---|
| dimBreach (사슬이동:강화) | chargeBoost | magicBlink |
| dimSlam (사슬이동:강화기습) | chargeBoost | magicBlink, chainAssault |
| dimThunder (사슬기동:기습파괴) | chargeBoost | magicBlink, chainAssault, chainSlam |
| whirlDet (회전기폭) | whirlwind | detonate |
| slamStorm (지진폭풍) | whirlwind | detonate, giantSlam |
| stormBeam (뇌전나선) | whirlwind | detonate, giantSlam, fanShot, omniBeam, elemMissile |
| sixFuse (뇌전) | fanShot | omniBeam |
| elemFuse (추적뇌전) | **omniBeam** ※ | fanShot, elemMissile |
| bladeFuse (악의 해방) | maliceHunt | guardian |
| plagueFuse (독혈해방) | maliceHunt | guardian, plagueBurst |
| iceMortar (얼음소용돌이) | maliceMortar | iceOrb |
| boneStorm (해골번개) | maliceStorm | boneWall |
| elecRepent (참회 귀환) | maliceStorm | hellRay, boneWall |
| shieldFuse (둔격참) | maliceSwipe | shieldThrow |
| holyFuse (신성한 결계) | holyDome | holyPrison |
| holyIce (신성한 빙결) | holyDome | holyPrison, iceStorm |

※ elemFuse의 omniBeam은 sixFuse에서는 흡수 대상이지만, elemFuse 활성 시 호스트로 승격 (stormBeam 활성 시 다시 흡수)

### 호스트 승격 체인

상위 합체가 활성되면 하위 합체의 흡수 스킬이 호스트로 승격될 수 있음:
- `sixFuse` → fanShot 호스트, omniBeam 흡수
- `elemFuse` → **omniBeam 호스트 승격**, fanShot 흡수 (sixFuse 흡수 오버라이드)
- `stormBeam` → whirlwind 호스트, omniBeam 다시 흡수

| 함수/변수 | 역할 |

| `_isAbsorbed(skId)` | 스킬이 현재 흡수 상태인지 판정 (호스트 오버라이드 포함) |
| `_getAllAbsorbed()` | 현재 활성된 모든 흡수 스킬 목록 반환 |
| `_syncActiveSkAfterReset(ids)` | 리셋/합체/로드 후 활성 스킬 변수 폴백 |
| `_renderSkillRow` (K키) | 스킬패널: `_isAbsorbed` → return으로 완전 제거 |
| `openSkSlotPop` (L키) | 합체 팝업: 호스트+흡수 제외 / 비합체: 배운 스킬만 |

## 레이저(빔) 스킬 MP 소비량

회전참(ST)과 동일 구조: `최대MP × 틱% × 레벨배율 × pMagicCost()`

| 스킬 | 틱% | 레벨배율 공식 | 1렙 실소비 | 10렙 실소비 | 비고 |
|---|---|---|---|---|---|
| 멸살광선 (단일) | 0.0015 | 0.5+(lv-1)×0.078 | mmp×0.075%/f | mmp×0.18%/f | 회전참과 동일 |
| 뇌전 (sixFuse) | 0.002 | 0.5+(lv-1)×0.078 | mmp×0.1%/f | mmp×0.24%/f | 6가닥이라 33% 증가 |
| 추적뇌전 (elemFuse) | 0.002 | ×0.9 적용 | mmp×0.09%/f | mmp×0.22%/f | 유도라 약간 할인 |
| 뇌전나선 (stormBeam) | 0.001 | 없음 (고정) | mmp×0.1%/f | mmp×0.1%/f | 회전참이 메인이므로 66% 할인 |

**참고**: 회전참 ST 소비 = `mst × 0.0015 × (0.5+(lv-1)×0.078)`

### 빔 관통

빔은 투사체 관통 패시브(`pPierce`) + 장비 `bowPierce`에 의해 관통 확률이 결정됨.
- 패시브/장비 없으면 **0% 관통** → 첫 적에서 끊김
- 패시브 1렙 = 5%, 10렙 = 50%
- 캡 95%

## 마력폭발 (magicBurst)

- **id**: `magicBurst`
- **카테고리**: magic (마법)
- **키**: E 홀드
- **스케일링**: INT (`statInt()` × `pMagicMul()` × `DPS_BAL.magic`)
- **MP 소모**: `mpCost('magicBurst')` — base 30, Lv 스케일

### 차징 메커니즘
| 프레임 | 단계 | 폭발 반경 | 데미지 배율 |
|---|---|---|---|
| 10f | 1단 | 300 × (1+(Lv-1)×0.05) | ×1.5 × (1+(Lv-1)×0.12) |
| 20f | 2단 | 500 × (1+(Lv-1)×0.05) | ×2.5 × (1+(Lv-1)×0.12) |
| 30f | 3단 | 700 × (1+(Lv-1)×0.05) | ×4.0 × (1+(Lv-1)×0.12) |

- 3단계 도달 시 **자동 폭발**
- 홀드 유지 시 MP 있으면 **반복 차징+폭발** (30f마다)
- 릴리즈 시 현재 단계로 즉시 폭발 (0단계면 불발)
- 넉백 배율: 0.3 (약한 넉백)
- 폭발 VFX: 충격파 링 (`G._gSlamWave`) + 보라 파티클

### 데미지 공식
```
baseDmg = meleeRef() × statInt() × pMagicMul() × DPS_BAL.magic × stageMul × lvMul
stageMul = [1.5, 2.5, 4.0][stage]
lvMul = 1 + (Lv-1) × 0.12
```

## 합체 비용 공식

- **합체 수행**: 성급 × 3 SP
- **합체 강화**: `_fuseUpSpCost(curLv, gemCnt)` — 기본테이블 + (성급-1)
- **스킬 습득**: 모든 스킬 10 SP 통일

### 합체 리셋 환불 공식

합체 해제 시 투입한 SP를 정확히 복구:

| 항목 | 환불액 | 비고 |
|---|---|---|
| 구성 스킬 습득비 | Σ spCost (스킬별) | 스킬 개수만큼 |
| 합체 생성비 | 성급 × 3 | 합체 수행 시 지불한 비용 |
| 합체 강화비 | Σ `_fuseUpSpCost(lv, gemCnt)` (lv=1..fuseLv-1) | 합체 후 강화한 레벨만큼 |
| **악의(mats)** | **환불 안 됨** | 일반 스킬 리셋과 동일 |

- 일반 스킬 리셋: `_skillSpentSp(sk, slv)` = 습득비 + 개별강화비 합산 (악의 미환불)

### 디버프 영역 구현 상세
- **적용 방식**: `G._fireZones`에 등록, 6프레임마다 `shQuery`로 범위 내 적 탐색
- **디버프 태그**: `e._weakPhys`, `e._weakMag`, `e._weakPj` (0~1 비율값)
- **디버프 지속**: `e._weakT=12` (0.2초), 영역 밖으로 나가면 자연 해제
- **데미지 적용**: `hurtE()`에서 이원 판정 직후, 크리티컬 전에 `dmg *= (1 + e._weakXxx)` 적용
  - `_weakPhys`: 물리(비마법·비투사체) 피격 증가
  - `_weakMag`: 마법(`_isMagicDmg`) 피격 증가
  - `_weakPj`: 투사체(`_isArrow`) 피격 증가

### 부활의 영역 구현 상세
- **적용 방식**: 플레이어 사망 시 `_revPowBonus`에 영역 보너스 가산
- **보너스**: `0.10 + (Lv-1) × 0.01` (Lv1=10%, Lv10=19%)
- **체크 시점**: `_onPlayerFallen()`에서 `dst(P, zone) < zone.r`로 범위 판정
- 적 디버프가 아닌 **플레이어 버프** — fireZone 틱에서 파티클만 처리
