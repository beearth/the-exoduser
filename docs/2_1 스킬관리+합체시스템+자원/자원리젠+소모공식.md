# 자원 리젠 & 소모 총정리

## 1. 자원 리젠 공식

### 기본 리젠 (recalcStats)

| 자원 | 공식 | 비고 |
|------|------|------|
| **ST리젠** | `0.05 + DEX×0.002 + VIT×0.001 + Lv×0.0005` | |
| **HP리젠** | `0.05 + VIT×0.002 + STR×0.002 + Lv×0.0005` | |
| **MP리젠** | `0.05 + INT×0.003 + Lv×0.0005` | |

- 기본치 0.05 = 스탯 0일 때 초당 약 0.375 (÷8 적용)
- 레벨 계수 0.0005 = Lv100에서 +0.05, Lv1000에서 +0.5
- 스탯 투자 없이는 리젠이 거의 없음 (의도된 설계)

### 상태별 리젠 배율

| 상태 | ST | HP | MP |
|------|----|----|-----|
| **대기 (idle)** | ÷8 | ÷8 | ÷8 |
| **행동 중** | ÷16 | ÷16 | ÷16 |
| **빔 사용 중** | ÷16 | ÷16 | ÷16 × 0.5 (50% 감소) |

### 추가 배율
- **신성한 땅 (holyDome)**: 리젠 ×2
- **장비 어픽스**: stRegen, mpRegen 별도 가산
- **패시브 (회복력 등)**: pRegenMul(), pMRegenMul() 곱연산

## 2. 소모 시스템 구조

### 실제 소모 경로: _COST_BASE + stCost()/mpCost()

```
stCost(key) = _COST_BASE[key] × _skLv(key) × _stDisc(key)
mpCost(key) = _COST_BASE[key] × _skLv(key) × pMagicCost()
```

- `_skLv(key)`: 연동 스킬 레벨 (레벨 올리면 비용도 증가)
- `_stDisc(key)`: pAtkCost() 또는 pBowCost() 자동 선택

### 패시브 할인 함수

| 함수 | 대상 | 할인 | 캡 |
|------|------|------|----|
| pAtkCost() | 무기/방패/돌진/회전참/거인강타 | -4%/lv | 최소 60% |
| pBowCost() | 석궁/칼날살/유탄산탄/만화방창 | -4%/lv | 최소 60% |
| pMagicCost() | 마법 전체 + 빔 틱 | -4%/lv | 최소 60% |

## 6. 채널링 — 틱당 소모

### 회전참 (ST)

```
소모 = mst × 0.0015 × (0.5 + (lv-1) × 0.078) × sp
```

| 레벨 | 배율 | 실효 소모율 |
|------|------|------------|
| Lv1 | ×0.5 | 0.075% /틱 |
| Lv5 | ×0.81 | 0.122% /틱 |
| Lv10 | ×1.2 | 0.180% /틱 |

### 빔 (MP)

```
소모 = mmp × (기본값 + (빔레벨-1) × 0.00015) × pMagicCost() × sp
```

| 빔 종류 | 기본값 | Lv1 | Lv5 | Lv10 |
|---------|--------|-----|-----|------|
| 멸살광선 | 0.001 | 0.001 | 0.0016 | 0.00235 |
| 뇌전빔 | 0.0013 | 0.0013 | 0.0019 | 0.00265 |
| 추적뇌전 | 0.0012 | 0.0012 | 0.0018 | 0.00255 |

### 뇌전나선 (회전참 중 자동 빔, MP)

```
소모 = mmp × 0.001 × pMagicCost() × sp
```
- 고정 비율 (레벨 스케일링 없음)

### 방패막기 유지 (MP)

| 단계 | 소모 |
|------|------|
| 진입 시 | mmp × 8% (1회) |
| 유지 틱 | mmp × 0.000833 × sp (~5%/초) |

### 회전기폭 (HP 드레인)

| 상태 | 소모 |
|------|------|
| 회전참 중 기폭 차지 | mhp × 0.0005 × sp |
| 방패막기 중 기폭 차지 | mhp × 0.0003 × sp |
- HP 1 이하로 안 떨어짐

## 11. 설계 의도

- **리젠은 스탯 투자의 보상**: 기본치를 극도로 낮춰 스탯 없이는 회복 불가
- **총량 비례(%) 소모**: 자원풀을 키워도 스킬 사용 횟수가 무한히 늘지 않음
- **채널링은 레벨 트레이드오프**: 회전참/빔 레벨이 올라가면 위력도 오르지만 소모도 증가
- **패시브 할인 캡 60%**: 아무리 투자해도 소모를 0으로 만들 수 없음
- **리젠 감소**: 빔 사용 중 MP리젠 반감 → 소모와 회복의 균형

## 12. 특이사항

- **COST_PCT vs _COST_BASE 이원화**: COST_PCT는 HUD 표시용, 실제 소모는 _COST_BASE + stCost()/mpCost() 경로. 예외로 whirlTick, bladeDash는 COST_PCT 직접 사용.
- **폭풍소환(maliceMortar)**: 악의 소모 없음, MP만 소모.
- **방패투척(shieldThrow)**: COST_PCT에 0.20 등록되어 있으나 실제는 인라인 고정값 50/30 ST.
- **beamCost() 함수**: 레거시, 현재 미사용 (인라인 _bcost로 대체됨).
