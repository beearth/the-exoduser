# 자원 리젠 & 소모 총정리

## 0. 자원 최대치 공식

| 자원 | 공식 | 비고 |
|------|------|------|
| **최대ST (mst)** | `100 + Lv×0.5 + DEX×1 + 장비bonusSt + maxSTFlat어픽스 + 결정ST + 성장` | DEX 스케일 + flat 어픽스 |
| **최대MP (mmp)** | `100 + Lv×0.5 + INT×1 + maxMPFlat어픽스 + 결정MP + 성장` | INT 스케일 + flat 어픽스 (ST와 대칭) |

### maxSTFlat / maxMPFlat 어픽스
- 5티어: T0=10, T1=20, T2=30, T3=40, T4=50
- 장착 슬롯: 방어구 10종 (armor, helm, shield, boots, gloves, pants, belt, cloak, bracelet, headband)
- 최대치: 10슬롯 × T4(50) = +500 (ST/MP 각각)
| **최대HP (mhp)** | 별도 공식 (STR/VIT/장비 등) | recalcStats 참조 |

## 1. 자원 리젠 공식

### 기본 리젠 (recalcStats)

| 자원 | 공식 | 비고 |
|------|------|------|
| **ST리젠** | `0.05 + DEX×0.005 + Lv×0.0015` | DEX 계수 0.005 유지, Lv 계수 0.0015 (약간 증가) |
| **HP리젠** | `0.05 + STR×0.002 + Lv×0.001` | |
| **MP리젠** | `0.05 + INT×0.005 + Lv×0.0015` | INT 계수 0.005 유지, Lv 계수 0.0015 (약간 증가) |

- 기본치 0.05 = 스탯 0일 때 초당 약 0.375 (÷8 적용)
- **주스탯 계수 통일 (v4)**: DEX/INT 모두 0.005로 통일. 기존 DEX 0.008 / INT 0.003으로 ST가 MP보다 3배 빠른 불균형 해소
  - 예시: Lv1000, DEX500, VIT300 → stR≈4.15 → idle 초당 ~31 회복

### 상태별 리젠 배율

| 상태 | ST | HP | MP |
|------|----|----|-----|
| **대기 (idle)** | ÷8 | ÷8 | ÷8 |
| **행동 중** | ÷16 | ÷16 | ÷16 |
| **빔 사용 중** | ÷16 | ÷16 | ÷16 × 0.5 (50% 감소) |

### 추가 배율
- **신성한 영역 (holyDome)**: 리젠 ×2
- **장비 어픽스**: stRegen, mpRegen 별도 가산
- **패시브 (회복력 등)**: pRegenMul(), pMRegenMul() 곱연산

## 2. 소모 시스템 구조

### 실제 소모 경로: _COST_BASE + stCost()/mpCost()

```
stCost(key) = min( _COST_BASE[key] × (1 + (skLv-1) × _COST_DPS[key]) × _stDisc(key), P.mst × 0.90 )
// 최대ST 90% 캡 — ST 풀이면 어떤 스킬이든 항상 발동 가능
mpCost(key) = _COST_BASE[key] × (1 + (skLv-1) × _COST_DPS[key]) × pMagicCost()
```

- `_COST_DPS[key]`: 해당 스킬의 DPS 성장률 (비용 성장 = DPS 성장, 효율 불변)
- `_stDisc(key)`: pAtkCost() 또는 pBowCost() 자동 선택

### 패시브 할인 함수

| 함수 | 대상 | 할인 | 캡 |
|------|------|------|----|
| pAtkCost() | 무기/방패/돌진/회전참/거인강타 | -4%/lv | 최소 60% |
| pBowCost() | 석궁/칼날살/만화방창 | -4%/lv | 최소 60% |
| pMagicCost() | 마법 전체 + 빔 틱 | -4%/lv | 최소 60% |

## 6. 채널링 — 고정값/초 소모 (2등급)

### 공통 규칙
- **레벨 스케일 소비**: 초당 `(10 + (Lv-1) × 10)` — COST_GLOBAL 레거시 삭제됨
- **소비량 보너스**: `1 + 초당소비 × 0.001` (Lv1=+1%, Lv10=+5.5%)
- 패시브 할인 **미적용**
- 레벨 올리면 비용↑ + 데미지↑ (소비량 보너스)

### 레벨별 소비량

| Lv | 초당 소비 | 소비량 보너스 | 자원500 지속 | 자원1000 지속 |
|----|----------|-------------|------------|-------------|
| 1 | 12/초 | ×1.012 (+1.2%) | 41.7초 | 83.3초 |
| 3 | 36/초 | ×1.036 (+3.6%) | 13.9초 | 27.8초 |
| 5 | 60/초 | ×1.06 (+6%) | 8.3초 | 16.7초 |
| 7 | 84/초 | ×1.084 (+8.4%) | 6.0초 | 11.9초 |
| 10 | 120/초 | ×1.12 (+12%) | 4.2초 | 8.3초 |

### 회전참 (ST)

```
costSec = (10 + (whirlwindLv - 1) × 10) × COST_GLOBAL
틱당 소모 = costSec / 60 × sp
소비량 보너스 = 1 + costSec × 0.001
```

### 빔 — 멸살광선/암전빔/추적암전 (MP)

```
costSec = 30 × (1 + (omniBeamLv - 1) × 0.18)
틱당 소모 = costSec / 60 × sp
소비량 보너스 = 1 + costSec × 0.001
sixFuse / elemFuse도 MP 비용은 동일
```

### 방패막기 유지 (MP)

| 단계 | 소모 |
|------|------|
| 진입 시 | mmp × 8% (1회) |
| 유지 틱 | (10 + (detonateLv - 1) × 10) / 60 × sp |

### 회전기폭 (HP 드레인)

| 상태 | 소모 |
|------|------|
| 회전참 중 기폭 차지 | **없음** (HP 드레인 제거) |
| 방패막기 중 기폭 차지 | mhp × 0.0003 × sp |
- HP 1 이하로 안 떨어짐

### 작살 게이지 (사슬기동)

```
리젠 = MAX × 0.025 / 60 × (1 + chargeBoostLv × 0.05) × pChargeGauge()
```

| 항목 | 값 |
|---|---|
| 기본 리젠 | MAX × 2.5%/초 |
| chargeBoost Lv10 | ×1.5 |
| pCharge 패시브 10 | ×1.3 |
| 풀스펙 | MAX × 4.875%/초 |
| MAX 300 기본 풀충전 | ~20초 |
| MAX 300 풀스펙 풀충전 | ~10.3초 |
| 게이지 코스트 | [0, 30, 65, 100] (1~3단) |

## 11. 데미지 카테고리 & DoT 체계

### 3대 카테고리

| 카테고리 | 주 스탯 | 패시브 | 자원 | DoT | 스케일링 |
|---------|--------|--------|------|-----|---------|
| **근접(물리)** | STR | 맹공(pAtkMul) | ST | 🩸 출혈(bleed) | `풀+=히트뎀×50%`, 5초, 중첩 | EL.P (물리판정) |
| **석궁(민첩)** | DEX | 정밀(pBowMul) | 악의 | ☠ 중독(poison) | `풀+=히트뎀×40%`, 5초, 중첩 | EL.P (물리판정) |
| **마법** | INT | 마력(pMagicMul) | MP | 🔥 화상(burn) / ❄ 냉기(ice) | `풀+=히트뎀×60%`, 3초, 중첩 | EL.F / EL.I (마법판정) |

### 카테고리별 대표 스킬

| 근접(STR) | 석궁(DEX) | 마법(INT) |
|----------|----------|----------|
| 기본공격(weapon) | 석궁(bow) | 멸살광선(omniBeam) |
| 회전참(whirlwind) | 만화방창(fanShot) | 추적암전(elemBeam) |
| 기검참(kiSlash) | 칼날살(bladeShot) | 암전나선(stormBeam) |
| 대왕치기(giantSlam) | ~~유탄산탄(삭제됨)~~ | 폭풍소환(maliceMortar) |
| ~~진동보행(삭제됨)~~ | 폭산탄(blastShot) | 폭독칼날(plagueBurst) |
| 칼등치기(maliceSwipe) | 만화방창 II(needleShot) | 블링크(blink) |
| 악의사냥(maliceHunt) | | 푸른비(blueShot) |
| 기동파괴(chainSlam) | | 기동불꽃(chainAssault) |
| 칼등날개(shieldThrow) | | 뇌전걸음(ghostWalk) |
| 기폭팔(detonate) | | 얼음보주(iceOrb) |

### DoT 설계 원칙
- 각 카테고리의 스킬은 **해당 카테고리의 DoT만** 부여해야 함
- 화상과 전기도트는 모두 INT 스케일 (마법 카테고리 내 서브타입)
- **D4식 DOT 풀 시스템**: 히트 데미지의 %를 풀에 축적, 틱마다 풀/잔여초 드레인. 크리=강한 DOT (자연 스케일)
- 스택(1~10)은 시각 표시용, 적용마다 +1 (최대10). 시간 종료 시 풀+스택 리셋
- **출혈**: 히트뎀 50% 풀 축적, 5초(300f), 중첩 (풀 누적). bleedPool += dmg×0.50
- **중독**: 히트뎀 40% 풀 축적, 5초(300f), 중첩 (풀 누적). poisonPool += dmg×0.40
- **화상**: 히트뎀 60% 풀 축적, 3초(180f), 중첩 (풀 누적). burnPool += dmg×0.60
- 틱 데미지 = pool / 잔여초 (매초 1회). 풀에서 틱뎀만큼 차감

## 12. 설계 의도

- **리젠은 스탯 투자의 보상**: 기본치를 극도로 낮춰 스탯 없이는 회복 불가. ST리젠은 DEX 주력(0.008)으로 민첩 투자 보상
- **채널링은 고정값/초 (2등급)**: `(10+(Lv-1)×10)×COST_GLOBAL` 공식. 자원풀 투자 → 지속시간 증가
- **채널링 패시브 할인 삭제**: 채널링 스킬은 고정비용, 할인 미적용. 1회성 스킬은 여전히 패시브 할인 캡 60% 적용
- **리젠 감소**: 빔 사용 중 MP리젠 반감 → 소모와 회복의 균형

## 12. 특이사항

- ~~COST_GLOBAL/COST_PCT/\_COST\_RAW~~ — **삭제됨** (미사용 레거시). 실제 소비는 `_COST_BASE × (1+(skLv-1)×DPS스케일) × 할인계수` DPS비례 방식.
- **채널링(회전참/빔/암전나선)은 COST_PCT 미사용**: 고정값/초 인라인 (`30/60*sp` 또는 `70/60*sp`). COST_PCT.whirlTick은 레거시.
- **~~진동보행(quakeStride)~~**: 삭제됨.
- **폭풍소환(maliceMortar)**: 악의 소모 없음, MP만 소모.
- **칼등날개(shieldThrow)**: ST 추가소모 0 (우클릭 useStPct('shield')에서 이미 차감). 제자리 처내기+충격파. 쿨 5초. **넉백 1.5**. 날개치기 발동 중 패링 시 반사뎀 +50% (빨콩·일반탄 모두). 검기 색상 항상 검정.
- **beamCost() 함수**: 레거시, 현재 미사용 (인라인 _bcost로 대체됨).
