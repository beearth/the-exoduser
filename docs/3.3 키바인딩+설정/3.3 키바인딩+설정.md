# 3.3 키바인딩 + 설정

## 키바인딩 (BINDS / BINDS2)
- BINDS: 주요 키 (이동, 공격, 스킬, 인벤, 설정 등)
- BINDS2: 보조 키 (같은 액션의 대체 키)
- 리바인딩 시스템: 설정 화면에서 변경 가능
- **키 설정 초기화**: 키바인딩 목록 하단 "⟲ 키 설정 초기화" 버튼 클릭, 또는 설정 패널(ESC) 열린 상태에서 **F5 키** → BINDS/BINDS2 전부 기본값 복원 (마우스 클릭 불가 상황 대비)

## 키바인딩 표시명 (BIND_NAMES / BIND_NAMES_EN)
- `BIND_NAMES`: 한국어 표시명 (위로 이동, 무기 공격, 칼등 처내기 등)
- `BIND_NAMES_EN`: 영문 표시명 (Move Up, Weapon Attack, Blade Parry 등)
- `_bindName(key)`: OPT.lang에 따라 한/영 자동 선택 헬퍼 함수
- 설정 화면 키바인딩 목록에서 사용

### 키 토글 설정창 미러 (자동저장)
- 설정창(ESC) → 게임 옵션 섹션의 기동:칼날개 위쪽에 키 토글 체크박스 2종 미러:
  - **석궁 자동발사 (T)** (`#optXbowAuto`, 색 #ffe066): T키와 양방향 동기화
  - **버스트루프 당기기 (H)** (`#optBlPullMode`, 색 #44ccff): H키와 양방향 동기화
- 체크박스 클릭 = 키 누른 것과 동일 동작 (showPH/addTxt 안내 포함)
- 키 입력 시 체크박스 상태 자동 갱신 (T→optXbowAuto, H→optBlPullMode)
- **자동저장**: 토글 변경 시 `OPT.xbowAuto`/`OPT.blPullMode` → `saveSettings()` 즉시 호출
  - 키 누름/체크박스 클릭 둘 다 OPT 갱신 + localStorage 저장
  - 게임 재시작 시 `_xbowEquipped=OPT.xbowAuto` 로 복원
  - `P._blPullMode`는 P가 있으면 P 값 우선, 없으면 `OPT.blPullMode` 폴백
- 설정창 열 때 현재 상태 반영 (체크박스 = P 우선 → OPT 폴백)

## 스킬패널 (K키) — 아이콘 모드 + 디아블로식 상세 툴팁 (2026-04-11)

### 아이콘 그리드
- **기본 모드**: 모든 스킬이 74×118 아이콘으로 표시 (`_skCompact=true` 기본값)
- **아이콘 이미지**: `img/skillskin_upscaled/{skill.id}.png` 사용 (48×48, pixelated)
  - 폴백: 이미지 없을 시 `onerror` 핸들러 → 이모지(`sk.emoji`) 표시
- **아이콘 표시 (3단)**:
  1. **★ 성급 배지** (좌상단 absolute, font 9px, #ffcc00, max 6개) — `_skStar = Math.max(_skInnateStar, _fuseStar)`로 산출
  2. PNG (48×48) — 중앙
  3. **스킬 이름** (font 11px, ★/HTML 제거 후 ellipsis 잘림, 폭 70px, text-shadow로 가독성)
  4. Lv.X (학습 시 #44ff66) 또는 ⭐SP 비용 (미학습 #ff8844, font 10px)
- **카테고리 컬럼**: compact 모드에선 아이콘들이 flex-wrap 그리드로 배치, 펼친 스킬만 컬럼 전체 폭 차지

### 🔗 합체 라인 연결 SVG 오버레이 (2026-04-11)
- `#skillPanel` 내부에 `#skillGridWrap` 래퍼 + `#skillFuseLines` SVG 오버레이 추가 (`position:absolute; pointer-events:none; z-index:5`)
- `_renderSkillRow`에서 각 스킬 카드에 `d.dataset.skId = sk.id` 설정
- `renderSkillPanel` 끝에서 `_drawSkillFuseLines()` 호출

**`_drawSkillFuseLines()` 동작 (2026-04-11 개정):**
1. `_skExpandedId` 없으면 SVG 비움
2. **`_canFuse(grp.key)` 통과한 합체 그룹만** 후보 (선행 합체 미완 = 라인 안 그림 → 사용자 혼란 방지)
   - `_allFuseGemGroups(_skExpandedId)`로 모든 그룹 후보 수집
   - `_isFused(g.key)===false && _canFuse(g.key)===true` 필터
   - 폴백: `_fuseGemGroup(_skExpandedId)` 단일 체크
3. 후보 그룹의 `_FUSE_PAIRS[g.key]`에서 펼친 스킬 외 파트너 ID 수집
4. `data-sk-id` 카드 순회 → 펼친 스킬은 소스, 파트너 ID 매칭은 타겟 push (그룹 색상 함께)
5. **박스 가장자리 좌표 계산 (`_edgePoint`)**:
   - 박스 중심에서 타겟 방향으로 박스 경계까지 t값 계산
   - `t = min(hw/|dx|, hh/|dy|)` → 박스 경계 교점 반환
   - 라인이 박스 중심 X, **가장자리에서 시작/종료**
6. 각 타겟마다 베지어 곡선 (`Q midX midY` 컨트롤 위로 휨) 그리기
7. 색상: 합체 그룹 색 (`_grp.color`), 점선 (`stroke-dasharray:6 4`), drop-shadow glow
8. 양 끝 마커: 시작점 5px 그룹 색 원, 끝점 4px 그룹 색 원

**자동 스크롤 (2026-04-11)**:
- `renderSkillPanel` 끝에서 `_skExpandedId`가 있으면 `requestAnimationFrame`으로 다음 프레임에 스크롤
- `document.querySelector('#skillGrid [data-sk-id="..."]')`로 펼친 카드 찾아 `scrollIntoView({behavior:'smooth',block:'center',inline:'nearest'})` 호출
- 클릭한 스킬이 카드 펼침 후 즉시 화면 중앙으로 스크롤되어 사용자 시야에 들어옴

**선행 합체 시퀀스 예**:
- 악의사냥(maliceHunt) 펼침 → `bladeFuse`(악의보호자) 합체 가능 → 라인 1개
- bladeFuse 합체 후 → `plagueFuse`(폭독칼날) 해금 → 라인 2개
- 그 전엔 plagueFuse 라인 안 보임 (`_canFuse('plagueFuse')===false`)

- 1+1 합체 방식이라 펼친 스킬 → 각 파트너로 1쌍씩 곡선 연결
- viewBox는 `skillGridWrap` 크기로 동적 갱신

### 합체 선택 모드 (`_skExpandedId` 기반 페이드)
스킬 아이콘 클릭으로 `_skExpandedId`가 설정되면 (상세 펼침 모드 트리거) 해당 스킬과 합체 가능한 스킬만 강조, 나머지는 페이드 처리:
- **펼친 스킬 본인** (`_skExpandedId === sk.id`): 다른 분기로 이동 → 카테고리 컬럼 안에서 디아블로식 상세 카드 렌더
- **합체 가능 파트너** (같은 `_fuseGemGroup` or `_FUSE_PAIRS` 매칭): outline 2px dashed `_fGrp.color` + box-shadow 16px 그룹 색상
- **그 외 모든 스킬**: `opacity:0.15; filter:grayscale(1) brightness(0.4); pointer-events:none` → 회색/거의 사라짐
- 합체 가능 판정: `_selGrp.key === _fGrp.key` OR `_FUSE_PAIRS[_selGrp.key]?.includes(sk.id)` OR `_FUSE_PAIRS[_fGrp.key]?.includes(_skExpandedId)`
- 펼친 카드의 ✕ 닫기 → `_skExpandedId=null` (선택 해제, 모두 정상 표시)
- 패널 닫기 → `closeAllPanels` → `_skExpandedId=null`

### 펼친 상세보기 (디아블로식 D2/D3 스타일)
- **클릭**: 아이콘 클릭 → 해당 스킬만 카테고리 컬럼 안에서 펼쳐서 상세보기
- **헤더**: 48px PNG + 스킬명 + 숙련도 + ✕ 닫기 버튼
- **`_skDetailHTML(sk, slv)` 함수**가 동적 스탯 섹션 생성:
  1. **스킬 정보**: 타입(물리/활/마법) / 레벨 / SP비용 / 악의비용 / 쿨다운 / 요구 레벨
  2. **데미지** (학습 시): 기본 참조뎀(meleeRef/bowRef/magicRef) / 스탯 배율 / 패시브 배율 / 레벨 배율 / 예상 평균뎀
  3. **치명타** (학습 시): 치명타 확률 / 치명타 배율 / 유효 배율
  4. **스킬 고유** (`_skSpecificDetails(sk, slv)`): id별 분기로 컴보 배율/관통/범위/사거리 등
  5. **설명** (원본 desc): 카테고리 섹션 아래 italic
- **동적 계산**: `meleeRef()`, `statStr()`, `pAtkMul()`, `statCrit()`, `statCritDmg()` 등을 try/catch로 안전 호출 → P 없을 때 폴백
- **고유 수치 분기 (`_skSpecificDetails`)**: kiSlash/whirlwind/fireball/detonate/maliceHunt/giantSlam/chargeBoost/magicBlink/darkPillar/spikeTrap 등 주요 스킬은 각자 고유 표 반환

### 🔄 펼침 모드는 기존 기능 fall-through (2026-04-11)
- 펼친 상세보기 카드는 디아블로식 디테일 + +/- 버튼 + **기존 모든 기능을 같은 카드에 fall-through**
- 펼침 모드 분기 끝에서 `return`을 제거 → `_renderSkillRow`의 다음 로직이 펼침 카드 `d`에 그대로 append됨:
  - **클릭 합체 시스템** (`_hasGem` 블록): 합체 그룹 버튼들 + `_fuseSelId` 클릭 토글 + 파트너 선택 → 합체 발동
  - **슬롯 배정 버튼** (1/2/3/4/SP/F): 액티브 스킬을 단축키 슬롯에 배정/해제
  - **리셋 버튼** (🔄 리셋): 합체 그룹 자동 탐색 → 그룹 전체 리셋
  - **드래그 시스템**: 마우스 드래그로 슬롯 재배치
- 아이콘 모드(`!_isExpanded`)는 단순 클릭 → `_skExpandedId=sk.id` 세팅 후 `return`으로 즉시 종료 (다른 로직 미실행)

### + / - 버튼 (학습/강화/취소/레벨다운/합체해제)
펼친 상세보기 하단에 두 개의 버튼이 나타남:

| 버튼 | 조건 (우선순위) | 동작 |
|---|---|---|
| **+ 습득** | 미습득 + canLearn | `_skClick()` — **SP/악의 선검증**(부족 시 `자원 부족!` 후 중단) → `P.skills[sk.id]=1`, SP/악의 차감, 슬롯 자동 배정 |
| **+ 레벨업** | 학습됨 + canUp | `_skClick()` — **SP/악의 선검증**(부족 시 `자원 부족!` 후 중단) → `P.skills[sk.id]++`, SP/악의 차감 |
| **− 합체 해제** | 학습 + 합체 완료 (`_isFused(_fGrp.key)===true`) | `_unfuseAll()` — 해당 스킬이 속한 모든 `P._fused[fk]` 삭제, 합체 강화 누적 SP 복구 추정 환불 (`_fuseUpSpCost`) |
| **− 레벨** | 학습 + slv>1 | `_skUnclick()` — `P.skills[sk.id]--`, 직전 강화 비용(`_skillUpSpCost(slv-1)`) **환불**, `sk.upMat` 환불 |
| **− 리셋** | 학습 + slv===1 | `_skUnclick()` — `delete P.skills[sk.id]`, **학습 비용 전액 환불** (`sk.spCost` + `sk.matCost`), 모든 슬롯에서 자동 해제 (LMB/charge/bow/magic/Q/tech/ULT) |

- 우선순위: **합체 해제 > 레벨다운 > 리셋** (합체 상태가 가장 먼저 풀림)
- 환불 시 `SFX.pickup()` + `addTxt(...) '합체 해제 (+NSP)' / 'Lv.X (-)' / '리셋!'` 알림
- 패널 닫기 시 `_skExpandedId=null` 리셋 (`closeAllPanels`)
- `_unfuseAll` 헬퍼: `P._fused`를 순회하며 해당 스킬이 `_FUSE_PAIRS[fk]` 또는 `_FUSE_GEM_GROUPS[fk].skills`에 포함되면 `delete P._fused[fk]` + 누적 강화 SP 추정 환불
- **전역 상태**: `_skExpandedId` (한 번에 하나만 펼침)
- **코드 위치**: game.html `_skDetailHTML`/`_skSpecificDetails` (라인 ~28049), `_renderSkillRow` (라인 ~29097), `_skUnclick` (라인 ~29503 인라인), `_skClick` (라인 ~29720)
- **아이콘 폴더**: `img/skillskin_upscaled/` (50+ 스킬 PNG, kiSlash/whirlwind/fireball 등 skill.id 매칭)

### BGM 선택 시스템
- `OPT.bgmTrack='auto'`: 스테이지별 자동 BGM (기본)
- 카테고리 선택 시: 해당 카테고리 내 랜덤 셔플, death/victory/forge만 자동전환 허용

- 선택 가능: 12카테고리 랜덤 + 27곡 개별 트랙 (optgroup으로 장별 그룹화)
