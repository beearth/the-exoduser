# 게임패드 호버 상호작용 리스트

> 패드 포커스 이동 시 `.gp-hover` 클래스가 자동 부여/제거됨. CSS에서 `:hover`와 `.gp-hover`를 병기하면 패드에서도 호버 애니메이션 작동.

## 적용 패턴

```css
/* 예시 — :hover 옆에 .gp-hover 추가 */
.enter-game-btn:hover:not(:disabled), .enter-game-btn.gp-hover { ... }
```

JS 로직 (`index.html` 로비 게임패드 네비게이션):
```js
// 이전 포커스 hover 제거
if(_prevGpIdx !== undefined && _navItems[_prevGpIdx]) _navItems[_prevGpIdx].el.classList.remove('gp-hover');
// 현재 포커스 hover 추가
if(cur) cur.el.classList.add('gp-hover');
```

## TODO — gp-hover 전역 적용 방안

인게임 `_gpUINav()`에서도 포커스 항목에 `.gp-hover` 추가/제거하면 위 항목 전부 자동 적용됨:

```js
// _gpUINav() 하이라이트 루프에서:
items[i].classList.add('gp-hover');    // 현재 포커스
items[prev].classList.remove('gp-hover'); // 이전 포커스
```

CSS에서 `:hover`를 `.gp-hover`와 병기하면 완료:
```css
.sk-opt:hover, .sk-opt.gp-hover { ... }
```

> **규칙**: 새 UI 요소에 `:hover` 추가 시 반드시 `.gp-hover`도 병기할 것.
