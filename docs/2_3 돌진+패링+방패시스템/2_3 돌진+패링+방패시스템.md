## 패링 시스템 (isPWin)

### 패링 개념
패링 = 무기/방어구로 상대 공격을 쳐내는 액션. 캐릭터 무기에 따라 모션만 다르고 판정은 동일.
| 캐릭터 | 패링 수단 | 상태 |
|--------|----------|------|
| 대검 | 칼등치기 | sBash |
| 도끼 | 옆날 | sBash |
| 방패 | 방패 | sBlock+패링윈도우 |

- 패링 윈도우: 20프레임(0.33초), 해제 후 12프레임(0.2초)
- isPWin() = sBash OR (sBlock+패링윈도우) OR (peaceShield+패링윈도우) OR (비회전/빔/전기참회+패링윈도우)
- **패링 성공 = 데미지 0 + 무적 + 상대 포이즈 대량 삭감** — 리스크 없이 그로기를 유도하는 핵심 수단
- **리듬게임식 패링**: 탄 색상별로 입력 방법이 다르며, 잘못된 입력 시 데미지를 받음

### 탄 종류별 패링 규칙 (리듬게임식 — 입력 구분 필수)

> **버그 수정 (2026-04-17)**: 빨콩에 대한 Q 오패링/Q 보호막 흡수 시 `hurtP` 호출에 `redBean:true` 플래그가 누락돼 `hurtP` 내부 sBlock/peaceShield PARRY 분기 (`!(opts&&opts.redBean)`)에 잡혀 `dmg=0` + 노란 `PARRY!` 텍스트가 잘못 발동하던 문제. 빨콩 오패링/흡수 경로 2곳에 `redBean:!!p.redBean` 전달하도록 수정. 이제 Q로 빨콩 맞으면 실제로 오패링 데미지가 들어감.
> - 수정 위치: `game.html:21021` (보호막 흡수), `game.html:22000` (빨콩오패링)
> - `hurtP` 가드: `game.html:29447 / 29453 / 29467`

| 탄 종류 | 패링 입력 | 패링 시 처리 | 잘못된 입력 시 |
|---|---|---|---|
| 빨콩(redBean) | **우클릭(sBash)만** | 블루콩 변환, pParryProjDmg(탄뎀,false)**×1.5**(근접+50%), **속도7**(2배), AoE×2, life 240 | Q 시도 → 탄뎀 100% + "❌ 우클릭으로!" (좁은범위 _wrongR) |
| 무지개탄(blackBean) | **Q(sBlock + 해제패링)** | 블루콩 유도반사, pParryProjDmg(탄뎀,true)×3, AoE×2, life 300 | 우클릭 시도 → 탄뎀 100% + "❌ Q키로!" (좁은범위 _wrongR) |
| 화이트볼(bwBean) | isPWin() + _rbDeflR 범위 | 유도 반사, pParryProjDmg(탄뎀,_sbPW)×1.5, life 180, homing | 피격 (dmg×1.3) |
| 다크볼(gbBean) | isPWin() + _normalR 범위 | 유도 반사, max(탄뎀,무기뎀)×2×pParryDmg(), life 200, homing | 피격 (dmg×1.3) |
| 일반탄 | isPWin() + _wwAbsR 범위 | 유도 반사, max(탄뎀,무기뎀)×1.5×pParryDmg(), life 180, homing | 피격 |
| 칼등처내기 | sBash 상태 | 반사, max(탄뎀,무기뎀)×1.5×Lv배율×_msReflect×pParryDmg()×날개, life 80 | — |
| 평화의보호 | peaceShield+패링윈도우 | 반사, max(탄뎀,무기뎀)×1.2×pParryDmg(), life 80, **doParry() 보상 동일** | 흡수 |

> **무기뎀** = meleeRef() × statStr() × pAtkMul() — 후반 레벨에서 적 탄환보다 무기뎀이 높으면 자동으로 무기 기반 반사

### 반사탄 착탄 시 폭발 공식 (projs[].friendly 히트)
반사탄이 적에게 명중하면 `_aoeDmg` 기반 범위 폭발 발동:
- **폭발 범위**: `_aoeR` (패링 변환 시 세팅, 타입별 120~180px)
- **폭발 데미지**: `_aoeDmg × (0.8 + 거리감쇠 × 0.2)` — 중심 100%, 가장자리 80%
- `_aoeDmg`는 패링 변환 시 `p.dmg × AoE배율`로 계산됨
- bomb 반사탄은 별도 경로 (`bombR`, `bombAtk` 기반)

| 타입 | 변환 p.dmg | _aoeDmg | _aoeR | 최종 착탄 배율(pPD 기준) |
|------|-----------|---------|-------|----------------------|
| 빨콩 | pPD×1.5 | p.dmg×2 | 160 | pPD×3 |
| 무지개탄 | pPD×3 | p.dmg×2 | 180 | pPD×6 |
| 화이트볼 | pPD×1.5 | p.dmg×1.5 | 140 | pPD×2.25 |
| 다크볼 | pPD×2 | p.dmg×2 | 160 | pPD×4 |
| Q일반탄 | pPD×1.2 | p.dmg×1.5 | 120 | pPD×1.8 |

### 패링 판정 범위 (`_rbDeflR`)
| 상태 | 범위 (px) |
|---|---|
| 칼등치기/Q일반/해제패링 (`_parryR`) | 기본 70 + bonusRange×4 + pParry×2 + parryBonus×20, **최대 100** |
| Q 기폭팔 (`_parryRDet`) | 기본 120 + bonusRange×6 + pParry×3 + parryBonus×30, **최대 180** |
| 공격 상태(wSwing 등) | (wRange+bonusRange)×2+60 |
| 기본 | _pCollR+20 |
| 반대 패링 패널티(_wrongR) | **100px 고정** |

### 공통 패링 보상 (doParry) — 모든 패링 유형 통일
- **적용 범위**: 우클릭(sBash) 패링, Q(sBlock) 패링, 해제 패링, 평화보호 패링, 투사체 패링, 무지개탄 패링 — 전부 doParry() 호출
- iframes 25프레임
- **HP/ST/MP 회복**: 우클릭(sBash) `30`, Q(sBlock/peaceShield/해제) `50` × (1+parryBonus어픽스)
- **악의 +1000**
- parryBank 축적 (기폭용)
- parryExplosion 어픽스 발동 (장착 시)
- hitStop/slowMo/shake 연출

## 방패 시스템
| 모드 | 키 | 효과 |
|---|---|---|
| 견갑막기 (sBlock) | 우클 | 데미지 감소만 |
| 견갑치기 (sBash) | 우클 (타이밍) | 패링/반사 가능 |
- 보호막 쿨다운: 30프레임(0.5초)

### 기폭팔 (detonate) — 2초 자동폭파 (전 모드 통일)
- **2초(120f)마다 자동폭파** — parryBank 유무 무관, 홀딩 안 풀어도 자동 발동 후 parryT 리셋
- 보호막(sBlock) 홀드 중: parryT 차지 + HP 드레인(0.03%/f) + 2초 자동폭파
- 보호막 해제 시: **2초(120f) 이상 홀드했을 때만** 폭발 (탭 해제 시 폭발 안 함)
- 해제 패링(hurtP) 시에도 동일: parryT>=120일 때만 해제기폭 발동
- 회전기폭(whirlDet) 합체: 회전참 중 2초 자동폭파 + Q수동기폭 + 종료 시 자동폭발
- 지진폭풍(slamStorm), 암전나선(stormBeam), 암전폭류(stormBurst) 체인 합체: 동일 규칙
- parryBank 없으면 기본 데미지(`magicRef()×statInt()×차지단계`) 사용

## 피격 무적 (iframes)
- 일반 피격 후 무적: 30f(0.5초) — 연속 피격(비비기) 방지
- DOT 피격: 무적 없음 (P.iframes=0)
- 한방캡(HP 33% 초과): 60f(1초) 무적
- 기절 진입: 무적 없음 (피격 계속 받음)

## 플레이어 기절 (pStun)
- 포이즈 0 → **즉시 기절(pStun)**: 480f(8초, ccRes 어픽스로 최대 30% 감소=5.6초). 포이즈 풀 복원
- **스턴 분류**: 미니스턴 `_MINI_STUN=60`(1초), 대왕치기 풀스턴 `min(300, 150+(Lv-1)*15)` (1렙=2.5초, 10렙=5초)
- **기절 진입 시 무적 없음** — 기절 중에도 데미지를 받아 위험하므로 빠른 탈출 필요
- 탈출: **Shift(작살)/Ctrl(탈출기)/Space(스킬) 즉시 해제**(poise 4칸 복원, 무적30f). Q 즉시 탈출은 제거됨 (2026-04-20)

## 포이즈/그로기 (통합 — 구 posture 시스템 제거됨)
- `e.poise` — 몬스터 체간 (보스: `HP*0.5`, 일반: `HP*0.5`)
- `e.stunned` — 그로기 상태
- `e._maxStunned` — 그로기 최대 시간 (게이지 비율 표시용)
- 그로기 시 데미지 1.5배
- **보스 그로기**: 1.5초(90f), 1회 발동 시 즉시 5초(300f) 면역(`_pImmune`) — **설계 의도: 연속 그로기(무한 CC) 방지**
- **일반몹 그로기**: 3초(180f)
- 포이즈 자연회복: 보스 5초(300f), 일반 3.3초(200f) 후 maxPoise의 15%씩
- 그로기 게이지: `stunned / _maxStunned` 비율로 실시간 감소
- 패링 시 보스 poise 직접 감소 (패턴별 10~60)
- **색상 코딩**: 빨강=우클릭 패링(PARRY), 무지개=Q 패링(Q!), 잘못된 입력=최대HP 10% 고정 페널티 (2026-04-21)
