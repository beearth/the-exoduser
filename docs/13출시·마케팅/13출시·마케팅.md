## 데모 배포 파이프라인 (2026-05-13 정리)

### 파일 구조

| 파일 | 용도 | 비고 |
|------|------|------|
| `G:\hell\game.html` | 본게임 (소스 원본) | 모든 코드 변경은 여기 |
| `G:\hell\gamedemo.html` | 데모 게임 (`_DEMO_MODE=true`) | game.html과 거의 동일, DEMO 제한만 다름 |
| `G:\hell\indexdemo.html` | 데모 로비 | gamedemo.html로 이동 |
| `G:\hell-build\` | 데모 빌드 폴더 (NW.js + Vercel 배포용) | |

### 핵심 규칙

1. **코드 변경은 game.html에만** — gamedemo.html은 자동 생성, 절대 직접 편집 금지
2. **gamedemo.html = game.html + `_DEMO_MODE=true`** — deploy-demo.bat이 자동 치환
3. **두 파일의 유일한 차이**: `const _DEMO_MODE=false` → `const _DEMO_MODE=true`
4. **hell-build sync는 deploy-demo.bat으로** — 개별 cp 금지

### 배포 명령 (`deploy-demo.bat`)

```bash
# G:\hell\deploy-demo.bat 실행
# 또는 터미널에서:
cd G:/hell
cp game.html ../hell-build/game.html
cp gamedemo.html ../hell-build/gamedemo.html
cp index.html ../hell-build/index.html
cp indexdemo.html ../hell-build/indexdemo.html
cp lang_*.js ../hell-build/
cp lobby_i18n.js maps_data.js ../hell-build/
cd ../hell-build && vercel --prod --yes
```

### 주의사항
- `hell-build`에는 NW.js 바이너리(dll, exe 등)가 있어 Vercel 100MB 제한에 걸림 → `.vercelignore`로 제외
- bgm/sfx/assets/sprites/img 폴더는 hell과 hell-build가 동일 (별도 sync 불필요, 파일 직접 복사)
- **gamedemo.html은 직접 편집 금지** — deploy-demo.bat 실행 시 항상 덮어씌워짐

## 코드 보호 전략 (출시 필수)

### 단계별 적용 계획

| 단계 | 방법 | 시점 | 보호 수준 | 비고 |
|------|------|------|----------|------|
| 1 | **Minify** | BIC 데모 | ★☆☆☆ | Terser로 공백/주석 제거, 파일 크기 60%↓ |
| 2 | **Obfuscation** | EA 출시 | ★★☆☆ | javascript-obfuscator로 변수명 치환, 문자열 암호화, 제어흐름 변환 |
| 3 | **WASM 변환** | 정식 출시 | ★★★★ | 핵심 로직(데미지 공식, 아이템 생성, 세이브 검증)을 C/Rust → WASM 컴파일 |
| 4 | **서버 권위** | 온라인 모드 | ★★★★★ | 드롭 테이블, 강화 확률, 경제 밸런스를 서버에서 처리 |

### WASM 변환 대상 (정식 출시 목표)

핵심 연산을 WASM으로 옮기면 디컴파일이 극도로 어려워짐:

| 모듈 | 현재 위치 | WASM 이관 시 효과 |
|------|----------|-----------------|
| 데미지 공식 | `hurtE()`, `_skMul()`, `elMul()` | 수치 해킹 차단 |
| 아이템/어픽스 생성 | `rollAffix()`, `rollItem()` | 드롭 조작 차단 |
| 세이브 직렬화+검증 | `dbSave()`, `dbLoad()` | 세이브 변조 차단 (HMAC 서명) |
| 경험치/레벨업 | `addExp()`, `_calcMaxExp()` | 레벨 해킹 차단 |
| 강화 확률 | 제련/결정/어픽스 리롤 | 확률 조작 차단 |
| 난수 생성기 | `Math.random()` 대체 | 시드 예측 차단 (xorshift128+ in WASM) |

### 구현 방법 (Rust → WASM)
```
[Rust 소스] → wasm-pack build → game_core.wasm + JS 바인딩
game.html에서 import → 핵심 함수 호출만 JS, 연산은 WASM 내부
```
- **언어**: Rust (wasm-bindgen) — 메모리 안전 + 최적 WASM 크기
- **빌드**: `wasm-pack build --target web --release`
- **크기 예상**: 핵심 로직만 ~50-100KB .wasm
- **성능**: JS 대비 동등~2배 (숫자 연산은 WASM이 유리)
