# 2-1. 스킬 관리 + 합체 시스템

## 스킬 슬롯 구조

### 고정스킬 (전용 키, 선택슬롯 배정 불가)
| 키 | 스킬 | 전환 변수 |
|---|---|---|
| Shift | 사슬기동 (돌진/블링크) | `P.activeChargeSk` |
| E탭/홀드 | 마법 (빔/육선포/추적탄) | `P.activeMagicSk` |
| T(자동) | 석궁 (일반/유탄/칼날/산탄) | `P.activeBowSk` |
| 좌클(홀드) | 회전참 | `P.activeLMBSk` |
| 우클 | 칼등치기/방패 | `P.activeRMBSk` |
| Q | 피보호막 | `P.activeQSk` |
| Ctrl | 뇌전걸음/얼음보주 | `P.activeCtSk` |

### 선택스킬 (SKILL_SLOTS[0~5])
| 키 | 슬롯 |
|---|---|
| 1~4 | SKILL_SLOTS[0~3] |
| Space | SKILL_SLOTS[4] |

## 합체 비용 공식

- **합체 수행**: 성급 × 3 SP
- **합체 강화**: `_fuseUpSpCost(curLv, gemCnt)` — 기본테이블 + (성급-1)
- **스킬 습득**: 모든 스킬 10 SP 통일
