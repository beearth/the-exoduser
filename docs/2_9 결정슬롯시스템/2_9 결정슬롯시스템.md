# 2-9. 결정 슬롯 시스템 (악의결정)

## 개요
장비에 **결정 슬롯**(소켓)을 두고, **악의결정**(보석)을 장착하여 추가 스탯을 부여하는 시스템.
탈착 자유, 강화 가능.

---

## 결정 슬롯

### 기본 규칙
- 장비 드롭 시 **소켓 수 랜덤** (1~4개) — 소켓 수 = 결정 티어
- weapon, bow 포함 전 장비 대상
- 빈 슬롯 = 효과 없음, 결정 장착 시 즉시 스탯 반영
- 탈착 자유 (비용 없음, 파괴 없음)

### 소켓 수 확률 (드롭 시 결정)
| 소켓 수 | 티어 | 확률 |
|---|---|---|
| 1개 | 1티어 | 50% |
| 2개 | 2티어 | 30% |
| 3개 | 3티어 | 15% |
| 4개 | 4티어 | 5% |

### 데이터 구조
```js
item.socketCount = 1~4          // 드롭 시 랜덤 결정
item.crystals = [null, ...]     // socketCount 길이 배열, null=빈칸
```

## 등급 (성급)

| 성급 | 이름 | 스탯 배율 | 색상 |
|---|---|---|---|
| 1성 | 조각 | ×1.0 | #888888 (회색) |
| 2성 | 파편 | ×2.0 | #44cc44 (초록) |
| 3성 | 결정 | ×3.5 | #4488ff (파랑) |
| 4성 | 순수결정 | ×6.0 | #cc44ff (보라) |
| 5성 | 완전결정 | ×10.0 | #ffaa00 (금) |

- 최종 수치 = 기본값 × 성급 배율 × (1 + 강화레벨 × 0.1)
- 예: HP결정 3성 +10강 = 100 × 3.5 × 2.0 = **700 HP**

---

## 강화

### 공식
- 결정 강화 비용: `악의 = 결정기본가 × (1 + 강화레벨 × 0.5)`
- 성공률: 100% (실패 없음, 순수 악의 소모)
- 강화 효과: 강화레벨당 **+10%** (1강=+10%, 10강=+100%=2배, 20강=+200%=3배)
- 강화 상한: **+20강**

### 강화 비용 테이블 (1성 기준)
| 강화 | 비용 (악의) | 누적 |
|---|---|---|
| 0→1 | 500 | 500 |
| 1→2 | 750 | 1,250 |
| 5→6 | 1,750 | 6,250 |
| 9→10 | 2,750 | 16,250 |
| 14→15 | 4,000 | 36,250 |
| 19→20 | 5,250 | 60,250 |

- 성급 보정: 2성 ×1.5, 3성 ×2.5, 4성 ×4, 5성 ×7

## 획득

### 드롭
- 일반몹: 1~2성 결정 (랜덤 종류)
- 엘리트: 2~3성
- 보스: 3~4성
- 장보스(챕터 최종): 4~5성

### 분해
- 결정 분해 → **결정 가루** 획득
- 결정 가루로 원하는 종류의 결정 제작 가능

### 제작
- **결정 가루 × N** → 원하는 종류 + 성급의 결정 제작
- 비용: 1성=50가루, 2성=200가루, 3성=500가루, 4성=1500가루, 5성=5000가루
- 분해 환급: 제작비의 50%

### 합성 (성급 업그레이드)
- **같은 종류 3개** 합성 → 1성급 상승
- 예: 1성 HP결정 × 3 → 2성 HP결정 × 1
- 강화 수치는 리셋 (0강으로)

## 인벤토리 / UI

### 결정 보관
- 별도 **결정 주머니** (가방과 분리)
- 최대 100개
- 종류/성급별 필터

### 장착 UI
- 장비 상세 화면에 결정 슬롯 4칸 표시
- 슬롯 클릭 → 결정 주머니에서 선택 → 장착
- 장착된 결정 클릭 → 탈착 (결정 주머니로 복귀)
- 드래그&드롭 또는 클릭 방식

### 결정 강화 UI
- 대장간(forge) 내 **결정 강화** 탭
- 결정 선택 → 강화 버튼 → 악의 소모 → 수치 증가

## 스탯 적용

### 집계
```
recalcStats() 시:
  모든 장착 장비의 crystals[] 순회 →
  결정 종류별 합산 → P.mhp, P.mst, P.mmp 등에 반영
```

### 적용 위치
| 결정 | 적용 대상 |
|---|---|
| cr_hp | P.mhp 합산 |
| cr_st | P.mst 합산 |
| cr_mp | P.mmp 합산 |
| cr_atk | meleeRef/bowRef/magicRef 합산 |
| cr_def | totalPDef 합산 |
| cr_spd | P.speed 배율 |
| cr_crit | statCrit() 합산 |
| cr_cdmg | statCritDmg() 합산 |
| cr_fire~holy | 해당 속성 추가뎀 (hurtE에서) |
| cr_hpReg | P.hpR 합산 |
| cr_stReg | P.stR 합산 |
| cr_mpReg | P.mpR 합산 |
| cr_drop | statDropBonus() 합산 |

## 세이브/로드

### 아이템 직렬화
```js
item.crystals = [
  {id:'cr_hp', star:3, enh:5},  // HP결정 3성 +5강
  null,                          // 빈 슬롯
  {id:'cr_fire', star:2, enh:0}, // 화염결정 2성 +0강
  null
]
```

### 결정 주머니
```js
G.crystalBag = [
  {id:'cr_hp', star:1, enh:0},
  {id:'cr_st', star:2, enh:3},
  ...
]
```

## 밸런스 메모

- 장비 15슬롯, 소켓 1~4개 랜덤 → 평균 ~1.75개 → 총 **~26소켓** (풀 4티어 시 60개)
- 풀 4티어 5성 +20강 HP결정 60개 = 100 × 10 × 3.0 × 60 = **180,000 HP** (이론 최대)
- 현실적 세팅 (2~3티어 혼합, 3성 +10강) ≈ HP 10,000~20,000 추가
- 기존 장비 HP(bonusHp 200 × 13 = 2,600)에 비해 큰 비중 → 엔드게임 성장축
- 원소결정은 속성 빌드 다양성 제공
- 유틸결정(리젠/드롭)은 파밍 효율 빌드용
