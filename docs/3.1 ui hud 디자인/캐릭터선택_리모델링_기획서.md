# HELL: EXODUSER 캐릭터 선택/생성 화면 리모델링 기획서

## 1. 현재 진단

스크린샷 기준 문제점

- 캐릭터 썸네일이 빈 사각형으로 노출 (시각화 0%)
- 좌우 화면 60% 이상이 검은 빈 공간, 패널만 중앙에 떠있음
- 게임 톤(다크 판타지/베르세르크/소울본)과 UI 톤(일반 웹폼)의 괴리
- 정보 부족: 이름/Lv/위치만, 플레이타임/통계 없음
- 삭제 버튼 상시 노출 - 오클릭 위험
- 캐릭터 생성: 이름만 입력하고 끝, 의식감 0
- LOCAL MODE 표기가 컨텍스트 없이 외롭게 떠있음
- 한국어 드롭다운 셀렉트가 OS 기본 스타일 그대로

## 2. 컨셉 방향

- 레퍼런스: Diablo IV 캐릭터 선택창, Elden Ring 캐릭터 메뉴, Hollow Knight 분위기
- 무드: ritual, 묵직함, 화면 안에 누군가 서있다는 존재감
- 핵심 변화: "리스트에서 데이터 고르기" -> "지옥에 보낼 캐릭터 고르기"
- HELL 로고 톤(주황/적색)을 액센트 컬러로 일관 사용

## 3. 레이아웃 (3분할)

화면을 좌 50% / 중 28% / 우 22% 비율로 분할.

### 좌측 50% - 캐릭터 디스플레이

- 선택된 캐릭터 풀바디 또는 흉상 (게임 내 idle 스프라이트 재활용)
- idle 애니메이션 루프 노출
- 캐릭터 뒤에 그라데이션 배경 + 미세한 안개/먼지 파티클
- 무기/장착 이펙트 그대로 노출
- 좌측 하단에 미세한 viginette

### 중앙 28% - 정보 패널

- 캐릭터 이름 (대형 타이틀, 로고 폰트와 통일)
- Lv. + (있다면) 클래스 태그
- 진행도: 예) 1장 얼음지옥 4층
- 플레이타임: hh:mm
- 누적 처치수
- 누적 사망 횟수
- 보유 골드/주요 재료 요약 (3개까지)
- 마지막 플레이: yyyy-mm-dd hh:mm

### 우측 22% - 슬롯 리스트

- 슬롯 카드 세로 정렬 (스크롤 가능)
- 활성 슬롯: 보더 글로우 + opacity 1.0
- 비활성 슬롯: opacity 0.6
- 호버시 scale 1.02, 200ms ease
- 삭제는 호버시에만 우측 끝 X 아이콘 노출

### 상하단

- 좌상단: HELL 로고 작게
- 우상단: LOCAL/ONLINE 모드 토글 + 언어 선택 (커스텀 셀렉트로 교체)
- 우하단: 빌드 버전 (v0.x.x)
- 좌하단: 종료 버튼
- 중앙 하단: 입장 버튼 (대형, 글로우)

## 4. 슬롯 카드 스펙

- 사이즈: 280 x 72
- 구성: 좌 56x56 흉상 썸네일 + 우측 텍스트 영역
- 텍스트 상단: 이름 (16px bold)
- 텍스트 하단: Lv.000 ㆍ 1장 얼음지옥 4층 (12px, 디밍)
- 보더: 1px solid rgba(255,255,255,0.08)
- 활성시 보더: 1px solid #c44 + box-shadow 0 0 12px rgba(196,68,68,0.4)
- 호버시 transform: scale(1.02)
- 삭제 X 아이콘은 hover시에만 fade-in (opacity 0 -> 0.7)

빈 슬롯

- + 아이콘 + "새 캐릭터" 텍스트
- 클릭시 생성 모달 진입
- 최대 슬롯 수는 정책에 맞게 (8개 추천)

## 5. 캐릭터 생성 플로우 개편

현재: 이름 입력 -> 생성 (단일 단계)

신규: 풀스크린 또는 큰 모달로 전환

1단계 - 이름 입력 (2~5자 제약 유지)
2단계 - (선택) 시작 외형/색조 프리셋 3~4개 중 선택
3단계 - 확인 -> ritual sfx + 페이드 트랜지션 -> 캐릭터 디스플레이로 자연스럽게 전환

외형 프리셋 추가가 부담이면 1, 3단계만 유지하되 트랜지션 강화.

## 6. 삭제 처리 강화

현재: 삭제 버튼 클릭시 즉시 삭제 (위험)

신규:

- 호버시에만 X 버튼 노출
- 클릭시 확인 모달 표시
- 모달 내용: "캐릭터 [이름] 을(를) 영구 삭제합니다. 확인을 위해 이름을 입력하세요."
- 이름 직접 타이핑 입력 일치시에만 삭제 활성
- 또는 대안: 길게 누르기 2초 + 진행 게이지

## 7. 모션/사운드

- 슬롯 전환: 좌측 디스플레이 fade 200ms + 정보 패널 슬라이드인 (translateX 8px -> 0)
- 호버: ui_hover sfx (기존 자산 재활용)
- 클릭: ui_select sfx
- 입장 (게임 시작): 화면 페이드 아웃 + 묵직한 transition sfx
- 생성 완료: ritual sfx + 짧은 글리치 또는 ember 파티클
- 삭제 확인: 낮은 톤 confirm sfx

## 8. 데이터 추가/마이그레이션

캐릭터 데이터에 추가 필드 필요

- totalKills: number
- totalDeaths: number
- playTimeMs: number
- lastPlayedAt: timestamp
- thumbnailFrame: 캐릭터 썸네일용 idle 프레임 캡쳐 (또는 동적 렌더)

기존 캐릭터: 없는 필드는 0 또는 null로 초기화.

## 9. 구현 우선순위

P0 - 이번 스프린트

1. 좌측 캐릭터 디스플레이 영역 (정적 스프라이트라도)
2. 슬롯 카드 시각 디자인 개선 (썸네일 + 보더 + 호버)
3. 정보 패널 (기존 데이터로 표시 가능한 항목부터)
4. 통계 데이터 수집 코드 추가 (다음 캐릭터부터 적용)

P1 - 다음 스프린트

5. 삭제 확인 강화 (이름 입력 또는 길게 누르기)
6. 슬롯 전환/디스플레이 애니메이션
7. 커스텀 언어 셀렉트 (OS 기본 제거)
8. 모드 토글 UI

P2 - 여유 있을 때

9. idle 애니메이션 루프 디스플레이
10. 안개/파티클 분위기 이펙트
11. ritual sfx + 생성 트랜지션
12. 외형 프리셋 시스템

## 10. 기술 메모

- 단일 HTML 파일 구조 유지 (game.html / game_demo.html / game_early.html)
- 좌측 디스플레이는 별도 캔버스 또는 메인 캔버스 영역 분할 사용
- 썸네일은 idle 첫 프레임을 미리 64x64 png로 캡쳐하거나 매 프레임 동적 렌더 (성능 비교 필요)
- 슬롯 카드는 div 기반, 필요시 flex 가로 스크롤 대응
- localStorage 키 변경 없이 캐릭터 데이터 객체에 필드만 추가
