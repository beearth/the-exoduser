# 2. 게임디자인 / 레벨디자인

> 미작성 — 내용 추가 필요

## 콤보 시스템

| 항목 | 값 |
|---|---|
| 타이머 | 일반몹: 최대 20초 (1200틱), 보스: 최대 30초 (1800틱) |
| 충전 규칙 (일반몹) | 킬 1회당 +1초(60틱) 충전, 최대 20초 캡 |
| 충전 규칙 (보스) | 보스 킬 시 +30초(1800틱), 최대 30초 캡 |
| 리셋 조건 | 타이머가 0이 되면 콤보 종료 (사망 시도 종료) |
| 킬 시 | `G.combo++`, 일반: `comboTimer += 60` (max 1200), 보스: `comboTimer += 1800` (max 1800) |
| 최대 기록 | `G.comboMax` (세션 내 최고 콤보) |
| HUD 위치 | 상단 바, cpHud 오른쪽 |
| 콤보 표시 | `comboHud` — 2콤보 이상 시 표시 |
| 타이머 표시 | `comboTimerHud` — 1콤보 이상 시 남은 초 표시 (5초 이하 빨강, 10초 이하 주황, 그 외 노랑) |
| 표시 색상 | 2~9: 노란, 10~19: 주황, 20~49: 진주황, 50+: 빨강 |
| 마일스톤 텍스트 | 5콤보마다 `🔥 N COMBO! +N%보상` 팝업 |
| 종료 텍스트 | 10콤보 이상에서 종료 시 `콤보 종료: N` 팝업 |
| 스테이지 초기화 | `initStage()` 시 combo=0, comboTimer=0 리셋 |

### 콤보 보상 보너스
콤보 보너스 + 난이도 보너스 합산. 캡 없음.
| 항목 | 공식 | 예시 |
|---|---|---|
| 기본 처치 보상 | 모든 적 처치 시 기본 보상 `+2`에 **획득 배율 x2 적용** | 일반몹 1킬 = +4 |
| 콤보 보너스 | `floor(combo/100) × 5%` | 100콤보=+5%, 500=+25%, 2000=+100% |
| 난이도 보너스 | `OPT.diff × 10%` | 0단=0%, 1단=+10%, 10단=+100% |
| 최종 배율 | `1 + 콤보보너스 + 난이도보너스` | 500콤보+5단 = 1+0.25+0.50 = ×1.75 |
| 적용 대상 | 악의(기본 처치 보상, 엘리트/보스 추가 드롭), 경험치(addExp 배율) | 악의 처치 보상은 x2 적용 |
| 일반 난이도(0단) | 보너스 없음 (콤보 보너스만) | |

## 스테이지 타임어택

| 항목 | 값 |
|---|---|
| 변수 | `G.stageTime` (틱 단위, 60fps), `G._totalSpawned`, `G._stageKills`, `G.playTime` (총 플레이타임, 틱) |
| 초기화 | `initStage()` 시 stageTime=0, _stageKills=0. `_totalSpawned`는 맵 생성 시 소환굴 총 스폰량 합산으로 확정 (S:50, M:100, L:100). 벽 타일 스폰 실패 시 분모 차감 (충돌체 위 스폰은 허용). 신성한 땅은 소환굴 제거 불가 |
| 증가 | 매 프레임 `G.stageTime += sp`, **모든 사망 경로**에서 _stageKills++ (분모 _totalSpawned는 고정) |
| 사망 경로 | 직접 킬(hurtE), DOT 사망, 장판 사망, 자폭(화염박쥐/톡주머니), 벽 끼임 즉사, 시간초과 소멸, 오버플로우 컬링 — 전부 카운트 |
| 소환굴 스폰 캡 | `G._aliveEns >= 750` 시 스폰 대기 (30프레임 쿨다운) — 생존 적 수 기준, 하드캡 900 전 여유 |
| 오버플로우 컬링 | `ens > 900` → 거리순 800개 유지, 제거된 생존 몹은 `_stageKills`에 가산 |
| 아레나 전환 보정 | `_enterBossArena` 시 던전 잔여 생존 몹(비보스)을 `_stageKills`에 가산 (소멸 = 킬 처리) |
| 인정 조건 | 스테이지 킬 / 총 스폰 >= 90% + 보스 처치 |
| 기록 저장 | `G._taRecords[stageIndex]` = 초 단위 최단 시간 |
| HUD | `stageTimerHud` — MM:SS 경과시간 + 처치율% + 베스트 기록 + 총 플레이타임(H:MM:SS) |
| HUD 색상 | 90% 이상: #44ddff, 미만: #88aacc |

### 클리어 통계 화면 (`#clearStats`)
스테이지 클리어 시 표시되는 전투 기록:
| 항목 | 변수 | 내용 |
|---|---|---|
| 처치 | `G._stageKills` / `G._totalSpawned` | N/M (처치율%) |
| 클리어 시간 | `G.stageTime` | MM:SS |
| 최대콤보 | `G.comboMax` | 세션 최고 콤보 |
| 사망 | `G._sStats.deaths` | 스테이지 내 사망 횟수 |
| 크리티컬 | `G._sStats.crits` | 크리 적중 횟수 |
| 패링 | `G._sStats.parries` | 패링 성공 횟수 |
| 물약 | `G._sStats.potions` | 물약 사용 횟수 |
| 총 데미지 | `G._sStats.dmgDealt` | 적에게 가한 총 데미지 |
| 받은 데미지 | `G._sStats.dmgTaken` | 플레이어가 받은 총 데미지 |
| 최대한방 | `G._sStats.maxHit` | 단일 최대 데미지 |
| 사슬기동 | `G._sStats.dashCount` / `dashDist` | 발동 횟수 + 이동거리(m, 64px=1m) |
| 도보이동 | `G._sStats.walked` | 도보 이동거리(m) |
| 타임어택 | 처치율 90%+ | ✓ 인정 / ✗ 미달 |
| 베스트 | `G._taRecords[stage]` | 해당 스테이지 최단 기록 |
| 점수 | 공식 계산 | 처치×10 + 콤보×5 + 패링×3 - 시간×2 - 사망×100 |

### 랭킹 점수 공식
`score = max(0, kills×10 + comboMax×5 + parries×3 - time(초)×2 - deaths×100)`
- 처치수와 시간이 핵심 변수
- 패링/콤보 보너스, 사망 페널티

### 뱃지 시스템
클리어 시 조건 달성하면 뱃지 표시:
| 뱃지 | 조건 |
|---|---|
| 🥇 무패 | 사망 0회 |
| 💀 전멸 | 처치율 100% |
| 💀 박멸 | 처치율 90%+ |
| 🔥 콤보마스터 | 최대콤보 50+ |
| 🔥 콤보광 | 최대콤보 20+ |
| 🛡 패링왕 | 패링 30회+ |
| 🛡 패링장인 | 패링 10회+ |
| ⚡ 스피드런 | 3분 이내 클리어 |
| 💪 무약 | 물약 0회 |
| 👻 무피격 | 받은 데미지 0 |
| 🔗 사슬광 | 사슬기동 30회+ |
| 💥 크리장인 | 크리 50회+ |
